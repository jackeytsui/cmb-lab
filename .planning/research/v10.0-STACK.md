# Technology Stack: v10.0 Mastery & Intelligence

**Project:** CantoMando Blueprint LMS
**Researched:** 2026-02-16

## Guiding Principle: Zero New npm Packages

The v9.0 milestone established a zero-new-packages policy. v10.0 continues this. Every feature can be built with existing dependencies plus new Postgres schema, API routes, components, and AI prompts. The one decision worth documenting: implementing FSRS-5 in-house (~100 lines) rather than adding the `ts-fsrs` package.

---

## Recommended Stack

### SRS Scheduling Algorithm

| Technology | Version | Purpose | Why |
|------------|---------|---------|-----|
| Custom FSRS-5 implementation | N/A | Spaced repetition scheduling | ~100 lines of pure TypeScript. Zero dependencies. FSRS-5 outperforms SM-2 by 20-30% fewer reviews for same retention. Reference implementation: femto-fsrs. |

**Decision: Implement FSRS in-house, NOT add ts-fsrs package**

FSRS-5 is well-documented and small enough to implement directly:
- 4 rating levels (again, hard, good, easy)
- 4 card states (new, learning, review, relearning)
- 2 core parameters per card (difficulty, stability)
- 1 core formula: `I = S * (R^(1/w) - 1)` where I=interval, S=stability, R=desired retention
- Total: ~100 lines of pure TypeScript with no external dependencies

This is comparable in complexity to existing lib utilities like `tone-sandhi.ts` (79 lines) and `practice-grading.ts` (206 lines). Adding `ts-fsrs` (22KB minified, external dependency) for what amounts to one pure function is unnecessary.

**Reference implementation:** [femto-fsrs](https://github.com/RickCarlino/femto-fsrs) -- zero-dependency FSRS-5 in ~100 lines of TypeScript.

## Alternatives Considered

| Category | Recommended | Alternative | Why Not |
|----------|-------------|-------------|---------|
| SRS Algorithm | Custom FSRS-5 (~100 lines) | `ts-fsrs` package | Zero-packages policy. Algorithm is small, well-documented, testable in isolation. |
| SRS Algorithm | FSRS-5 | SM-2 | FSRS is 20-30% more efficient. SM-2 is from 1987. FSRS is current state-of-the-art (adopted by Anki in 2023). |
| Tone Audio | Azure Speech (existing) | Web Audio API pitch detection | Azure is trained on millions of Chinese speech samples. Client-side pitch detection unreliable for tonal languages. |
| Tone Visualization | No visualization (v10.0) | Web Audio API F0 contour | Defer pitch contour UI to future milestone. Azure per-character scores are sufficient for v10.0 tone training. |
| Exercise Gen | n8n + OpenAI | Direct OpenAI calls | Breaks "all AI through n8n" constraint. |
| Grammar Editor | TipTap (existing) | MDX/Markdown | TipTap already installed. Rich text better for grammar examples. |
| Study Cache | Upstash Redis (existing) | Postgres materialized view | Redis already configured. 30-min TTL simpler than materialized view refresh. |
| HSK Data | Seed script from GitHub JSON | `hsk-vocabulary` npm | Seed-time data, not runtime dependency. Clone JSON, run seed script, done. |
| Smart Study | Heuristic engine | ML recommendation | <100 users. Heuristic gives 90% value at 1% complexity. |

## Installation

```bash
# No new packages to install.
# Only database migrations:

npm run db:generate   # Generate Drizzle migration for 9 new tables
npm run db:migrate    # Apply migration to Neon Postgres

# Optional: seed grammar/HSK data
npx tsx scripts/seed-grammar.ts
npx tsx scripts/seed-assessments.ts
```
