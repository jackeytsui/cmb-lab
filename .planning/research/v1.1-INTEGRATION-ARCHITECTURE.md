# v1.1 Integration Architecture

**Project:** CantoMando Blueprint LMS
**Scope:** Bulk upload, RAG chatbot, AI prompts dashboard
**Researched:** 2026-01-28
**Confidence:** HIGH (verified via official docs)

## Executive Summary

The v1.1 features integrate cleanly with the existing Next.js 15 + Neon Postgres + Drizzle architecture. Key integration points:

1. **Bulk Mux Upload** - Add MuxUploader React component, new webhook route, bulk assignment UI
2. **RAG Chatbot** - Enable pgvector extension, add knowledge base tables, integrate Vercel AI SDK
3. **AI Prompts Dashboard** - New `ai_prompts` table, CRUD admin UI, inject prompts at runtime

The existing architecture handles video playback (Mux Player), AI grading (n8n webhooks), and voice conversation (OpenAI Realtime). New features extend rather than replace these patterns.

### API Route: Create Upload URL

```typescript
// app/api/mux/upload/route.ts
import { NextRequest, NextResponse } from "next/server";
import { auth } from "@clerk/nextjs/server";
import Mux from "@mux/mux-node";
import { db } from "@/db";
import { muxUploads, users } from "@/db/schema";
import { eq } from "drizzle-orm";

const mux = new Mux({
  tokenId: process.env.MUX_TOKEN_ID!,
  tokenSecret: process.env.MUX_TOKEN_SECRET!,
});

export async function POST(request: NextRequest) {
  // 1. Verify coach/admin role
  const { userId } = await auth();
  if (!userId) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  const dbUser = await db.query.users.findFirst({
    where: eq(users.clerkId, userId),
  });

  if (!dbUser || !["coach", "admin"].includes(dbUser.role)) {
    return NextResponse.json({ error: "Forbidden" }, { status: 403 });
  }

  const body = await request.json();
  const { filename } = body;

  // 2. Create direct upload via Mux API
  const upload = await mux.video.uploads.create({
    new_asset_settings: {
      playback_policy: ["public"],
      encoding_tier: "smart",
    },
    cors_origin: process.env.NEXT_PUBLIC_APP_URL || "*",
  });

  // 3. Store upload record in database
  await db.insert(muxUploads).values({
    uploadId: upload.id,
    status: "pending",
    originalFilename: filename,
    uploadedBy: dbUser.id,
  });

  // 4. Return signed URL to client
  return NextResponse.json({
    uploadUrl: upload.url,
    uploadId: upload.id,
  });
}
```

### Webhook Handler: Asset Events

```typescript
// app/api/webhooks/mux/route.ts
import { NextRequest, NextResponse } from "next/server";
import Mux from "@mux/mux-node";
import { db } from "@/db";
import { muxUploads } from "@/db/schema";
import { eq } from "drizzle-orm";

const mux = new Mux({
  tokenId: process.env.MUX_TOKEN_ID!,
  tokenSecret: process.env.MUX_TOKEN_SECRET!,
});

export async function POST(request: NextRequest) {
  const rawBody = await request.text();
  const signature = request.headers.get("mux-signature");

  // 1. Verify webhook signature
  try {
    mux.webhooks.verifySignature(
      rawBody,
      { "mux-signature": signature! },
      process.env.MUX_WEBHOOK_SECRET!
    );
  } catch {
    return NextResponse.json({ error: "Invalid signature" }, { status: 401 });
  }

  const body = JSON.parse(rawBody);

  // 2. Handle event types
  switch (body.type) {
    case "video.upload.asset_created":
      // Upload completed, asset is being processed
      await db.update(muxUploads)
        .set({
          assetId: body.data.asset_id,
          status: "processing",
        })
        .where(eq(muxUploads.uploadId, body.data.id));
      break;

    case "video.asset.ready":
      // Asset ready for playback
      await db.update(muxUploads)
        .set({
          playbackId: body.data.playback_ids?.[0]?.id,
          durationSeconds: Math.floor(body.data.duration || 0),
          status: "ready",
          readyAt: new Date(),
        })
        .where(eq(muxUploads.assetId, body.data.id));
      break;

    case "video.asset.errored":
      // Processing failed
      await db.update(muxUploads)
        .set({
          status: "failed",
          errorMessage: body.data.errors?.messages?.join(", "),
        })
        .where(eq(muxUploads.assetId, body.data.id));
      break;
  }

  return NextResponse.json({ received: true });
}
```

### Client Component: Bulk Uploader

```typescript
// components/admin/BulkUploader.tsx
"use client";

import { useState, useCallback } from "react";
import { Upload, CheckCircle, XCircle, Loader2 } from "lucide-react";
import { Button } from "@/components/ui/button";

interface UploadState {
  uploadId: string;
  filename: string;
  status: "uploading" | "processing" | "ready" | "failed";
  progress?: number;
}

export function BulkUploader({ onComplete }: { onComplete?: () => void }) {
  const [uploads, setUploads] = useState<UploadState[]>([]);
  const [isUploading, setIsUploading] = useState(false);

  const handleFiles = useCallback(async (files: FileList) => {
    setIsUploading(true);

    for (const file of Array.from(files)) {
      // 1. Get signed URL from our API
      const response = await fetch("/api/mux/upload", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ filename: file.name }),
      });

      if (!response.ok) {
        console.error("Failed to get upload URL");
        continue;
      }

      const { uploadUrl, uploadId } = await response.json();

      // 2. Add to UI state
      setUploads(prev => [...prev, {
        uploadId,
        filename: file.name,
        status: "uploading",
        progress: 0,
      }]);

      // 3. Upload file directly to Mux
      try {
        const xhr = new XMLHttpRequest();
        xhr.open("PUT", uploadUrl);

        xhr.upload.onprogress = (e) => {
          if (e.lengthComputable) {
            const progress = Math.round((e.loaded / e.total) * 100);
            setUploads(prev => prev.map(u =>
              u.uploadId === uploadId ? { ...u, progress } : u
            ));
          }
        };

        xhr.onload = () => {
          if (xhr.status === 200) {
            setUploads(prev => prev.map(u =>
              u.uploadId === uploadId ? { ...u, status: "processing" } : u
            ));
          } else {
            setUploads(prev => prev.map(u =>
              u.uploadId === uploadId ? { ...u, status: "failed" } : u
            ));
          }
        };

        xhr.send(file);
      } catch (error) {
        setUploads(prev => prev.map(u =>
          u.uploadId === uploadId ? { ...u, status: "failed" } : u
        ));
      }
    }

    setIsUploading(false);
  }, []);

  return (
    <div className="space-y-4">
      <div className="border-2 border-dashed border-zinc-600 rounded-lg p-8 text-center">
        <Upload className="w-12 h-12 mx-auto mb-4 text-zinc-400" />
        <p className="text-zinc-300 mb-4">Drag and drop videos or click to select</p>
        <input
          type="file"
          multiple
          accept="video/*"
          onChange={(e) => e.target.files && handleFiles(e.target.files)}
          className="hidden"
          id="video-upload"
        />
        <Button asChild disabled={isUploading}>
          <label htmlFor="video-upload" className="cursor-pointer">
            Select Videos
          </label>
        </Button>
      </div>

      {uploads.length > 0 && (
        <div className="space-y-2">
          {uploads.map((upload) => (
            <div
              key={upload.uploadId}
              className="flex items-center gap-3 p-3 bg-zinc-800 rounded-lg"
            >
              {upload.status === "uploading" && (
                <Loader2 className="w-5 h-5 text-cyan-400 animate-spin" />
              )}
              {upload.status === "processing" && (
                <Loader2 className="w-5 h-5 text-yellow-400 animate-spin" />
              )}
              {upload.status === "ready" && (
                <CheckCircle className="w-5 h-5 text-green-400" />
              )}
              {upload.status === "failed" && (
                <XCircle className="w-5 h-5 text-red-400" />
              )}
              <span className="flex-1 truncate text-sm">{upload.filename}</span>
              {upload.status === "uploading" && upload.progress !== undefined && (
                <span className="text-xs text-zinc-400">{upload.progress}%</span>
              )}
              {upload.status === "processing" && (
                <span className="text-xs text-yellow-400">Processing...</span>
              )}
            </div>
          ))}
        </div>
      )}
    </div>
  );
}
```

```typescript
// db/schema/knowledge.ts
import {
  pgTable,
  uuid,
  text,
  timestamp,
  integer,
  pgEnum,
  index,
} from "drizzle-orm/pg-core";
import { vector } from "drizzle-orm/pg-core";
import { relations } from "drizzle-orm";
import { users } from "./users";

export const sourceTypeEnum = pgEnum("knowledge_source_type", [
  "manual",
  "pdf",
  "doc",
]);

export const knowledgeCategories = pgTable("knowledge_categories", {
  id: uuid("id").defaultRandom().primaryKey(),
  name: text("name").notNull(),           // e.g., "Packages", "Chinese Help", "FAQ"
  description: text("description"),
  sortOrder: integer("sort_order").notNull().default(0),
  createdAt: timestamp("created_at").notNull().defaultNow(),
});

export const knowledgeEntries = pgTable("knowledge_entries", {
  id: uuid("id").defaultRandom().primaryKey(),
  categoryId: uuid("category_id")
    .notNull()
    .references(() => knowledgeCategories.id, { onDelete: "cascade" }),
  title: text("title").notNull(),
  content: text("content").notNull(),      // Full text content
  sourceType: sourceTypeEnum("source_type").notNull(),
  sourceFilename: text("source_filename"), // Original filename if uploaded
  createdBy: uuid("created_by").notNull().references(() => users.id),
  createdAt: timestamp("created_at").notNull().defaultNow(),
  updatedAt: timestamp("updated_at").notNull().defaultNow().$onUpdate(() => new Date()),
});

// Embeddings stored separately for chunking
export const knowledgeEmbeddings = pgTable("knowledge_embeddings", {
  id: uuid("id").defaultRandom().primaryKey(),
  entryId: uuid("entry_id")
    .notNull()
    .references(() => knowledgeEntries.id, { onDelete: "cascade" }),
  chunkIndex: integer("chunk_index").notNull(),
  chunkText: text("chunk_text").notNull(), // The actual text chunk
  embedding: vector("embedding", { dimensions: 1536 }), // OpenAI text-embedding-3-small
  createdAt: timestamp("created_at").notNull().defaultNow(),
}, (table) => [
  index("knowledge_embedding_idx").using("hnsw", table.embedding.op("vector_cosine_ops")),
]);

// Chat sessions for context
export const chatSessions = pgTable("chat_sessions", {
  id: uuid("id").defaultRandom().primaryKey(),
  userId: uuid("user_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  startedAt: timestamp("started_at").notNull().defaultNow(),
  endedAt: timestamp("ended_at"),
});

export const chatMessages = pgTable("chat_messages", {
  id: uuid("id").defaultRandom().primaryKey(),
  sessionId: uuid("session_id").notNull().references(() => chatSessions.id, { onDelete: "cascade" }),
  role: chatRoleEnum("role").notNull(),
  content: text("content").notNull(),
  createdAt: timestamp("created_at").notNull().defaultNow(),
});

export const chatRoleEnum = pgEnum("chat_role", ["user", "assistant"]);

// Relations
export const knowledgeCategoriesRelations = relations(knowledgeCategories, ({ many }) => ({
  entries: many(knowledgeEntries),
}));

export const knowledgeEntriesRelations = relations(knowledgeEntries, ({ one, many }) => ({
  category: one(knowledgeCategories, {
    fields: [knowledgeEntries.categoryId],
    references: [knowledgeCategories.id],
  }),
  embeddings: many(knowledgeEmbeddings),
  createdByUser: one(users, {
    fields: [knowledgeEntries.createdBy],

export const knowledgeEmbeddingsRelations = relations(knowledgeEmbeddings, ({ one }) => ({

export type KnowledgeEntry = typeof knowledgeEntries.$inferSelect;
export type NewKnowledgeEntry = typeof knowledgeEntries.$inferInsert;
```

### Embedding Service

```typescript
// lib/embeddings.ts
import OpenAI from "openai";
import { db } from "@/db";
import { knowledgeEmbeddings } from "@/db/schema";
import { cosineDistance, desc, gt, sql } from "drizzle-orm";

const openai = new OpenAI();

export async function generateEmbedding(text: string): Promise<number[]> {
  const response = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: text,
    dimensions: 1536,
  });
  return response.data[0].embedding;
}

export async function chunkAndEmbed(entryId: string, content: string) {
  // Simple sentence-based chunking
  const sentences = content.split(/[.!?]\s+/).filter(s => s.trim().length > 20);

  // Group ~3 sentences per chunk for better context
  const chunks: string[] = [];
  for (let i = 0; i < sentences.length; i += 3) {
    const chunk = sentences.slice(i, i + 3).join(". ") + ".";
    chunks.push(chunk.trim());
  }

  // Generate embeddings and store
  for (let i = 0; i < chunks.length; i++) {
    const embedding = await generateEmbedding(chunks[i]);
    await db.insert(knowledgeEmbeddings).values({
      entryId,
      chunkIndex: i,
      chunkText: chunks[i],
      embedding,
    });
  }
}

export async function searchKnowledge(query: string, limit = 4): Promise<string[]> {
  const queryEmbedding = await generateEmbedding(query);

  const similarity = sql<number>`1 - (${cosineDistance(knowledgeEmbeddings.embedding, queryEmbedding)})`;

  const results = await db
    .select({
      text: knowledgeEmbeddings.chunkText,
      similarity,
    })
    .from(knowledgeEmbeddings)
    .where(gt(similarity, 0.5))
    .orderBy(desc(similarity))
    .limit(limit);

  return results.map(r => r.text);
}
```

### Chat API Route (Vercel AI SDK)

```typescript
// app/api/chat/route.ts
import { openai } from "@ai-sdk/openai";
import { streamText, tool } from "ai";
import { z } from "zod";
import { auth } from "@clerk/nextjs/server";
import { searchKnowledge } from "@/lib/embeddings";
import { getPrompt } from "@/lib/prompts";

const DEFAULT_SYSTEM_PROMPT = `You are a helpful assistant for CantoMando Blueprint, a language learning platform teaching Mandarin and Cantonese simultaneously.

You can help with:
- Questions about course packages and pricing
- General Chinese language learning tips
- Technical support for the platform
- Information about the CantoMando teaching methodology

When answering questions about Chinese, you can use the appropriate characters and include romanization (Pinyin for Mandarin, Jyutping for Cantonese) when helpful.

Use the getKnowledge tool to search for specific information before answering questions about packages, coaching, or platform features.`;

export async function POST(request: Request) {
  const { userId } = await auth();
  if (!userId) {
    return new Response("Unauthorized", { status: 401 });
  }

  const { messages } = await request.json();

  // Get system prompt from database (or fallback)
  const systemPrompt = await getPrompt("chatbot_system") || DEFAULT_SYSTEM_PROMPT;

  const result = streamText({
    model: openai("gpt-4o-mini"),
    system: systemPrompt,
    messages,
    tools: {
      getKnowledge: tool({
        description: "Search the knowledge base for information about packages, Chinese learning, coaching, or platform features",
        parameters: z.object({
          query: z.string().describe("The search query"),
        }),
        execute: async ({ query }) => {
          const results = await searchKnowledge(query);
          if (results.length === 0) {
            return "No relevant information found in the knowledge base.";
          }
          return results.join("\n\n---\n\n");
        },
      }),
    },
    maxSteps: 3, // Allow multiple tool calls if needed
  });

  return result.toDataStreamResponse();
}
```

### Database Schema

```typescript
// db/schema/prompts.ts
import {
  pgTable,
  uuid,
  text,
  timestamp,
  boolean,
  pgEnum,
} from "drizzle-orm/pg-core";
import { relations } from "drizzle-orm";
import { users } from "./users";

export const promptCategoryEnum = pgEnum("prompt_category", [
  "grading",        // Text and audio grading prompts
  "voice",          // Voice conversation system prompts
  "chatbot",        // Chatbot system prompts
]);

export const aiPrompts = pgTable("ai_prompts", {
  id: uuid("id").defaultRandom().primaryKey(),
  key: text("key").notNull().unique(),           // e.g., "grading_text", "voice_system", "chatbot_system"
  name: text("name").notNull(),                  // Human-readable name
  description: text("description"),              // What this prompt does
  content: text("content").notNull(),            // The actual prompt text
  category: promptCategoryEnum("category").notNull(),
  isActive: boolean("is_active").notNull().default(true),
  lastEditedBy: uuid("last_edited_by").references(() => users.id),
  createdAt: timestamp("created_at").notNull().defaultNow(),
  updatedAt: timestamp("updated_at").notNull().defaultNow().$onUpdate(() => new Date()),
});

// Version history for audit trail
export const aiPromptVersions = pgTable("ai_prompt_versions", {
  id: uuid("id").defaultRandom().primaryKey(),
  promptId: uuid("prompt_id").notNull().references(() => aiPrompts.id, { onDelete: "cascade" }),
  content: text("content").notNull(),
  editedBy: uuid("edited_by").notNull().references(() => users.id),
  createdAt: timestamp("created_at").notNull().defaultNow(),
});

export const aiPromptsRelations = relations(aiPrompts, ({ one, many }) => ({
  lastEditor: one(users, {
    fields: [aiPrompts.lastEditedBy],
    references: [users.id],
  }),
  versions: many(aiPromptVersions),
}));

export const aiPromptVersionsRelations = relations(aiPromptVersions, ({ one }) => ({
  prompt: one(aiPrompts, {
    fields: [aiPromptVersions.promptId],
    references: [aiPrompts.id],
  }),
  editor: one(users, {
    fields: [aiPromptVersions.editedBy],
    references: [users.id],
  }),
}));

export type AiPrompt = typeof aiPrompts.$inferSelect;
export type NewAiPrompt = typeof aiPrompts.$inferInsert;
export type AiPromptVersion = typeof aiPromptVersions.$inferSelect;
```

### Prompt Registry

```typescript
// lib/prompts.ts
import { db } from "@/db";
import { aiPrompts } from "@/db/schema";
import { eq } from "drizzle-orm";

// Fallback prompts used if database entry is missing or inactive
const FALLBACK_PROMPTS: Record<string, string> = {
  grading_text: `You are grading a Chinese language student's text response.
Evaluate accuracy, grammar, and character usage.
Return a score from 0-100 and constructive feedback.`,

  grading_audio: `You are grading a spoken Chinese audio submission.
Evaluate pronunciation, tones, fluency, and vocabulary usage.
Return a score from 0-100 and constructive feedback.`,

  voice_system: `You are an AI language tutor for CantoMando Blueprint.
Help students practice speaking Mandarin and Cantonese.
Focus on the vocabulary and grammar from their current lesson.
Provide gentle corrections and encouragement.`,

  chatbot_system: `You are a helpful assistant for CantoMando Blueprint.
Help with questions about packages, lessons, and Chinese learning.
Use the getKnowledge tool to find specific information.`,
};

export async function getPrompt(key: string): Promise<string> {
  try {
    const prompt = await db.query.aiPrompts.findFirst({
      where: eq(aiPrompts.key, key),
    });

    if (prompt?.isActive && prompt.content) {
      return prompt.content;
    }
  } catch (error) {
    console.error(`Failed to load prompt ${key}:`, error);
  }

  return FALLBACK_PROMPTS[key] || "";
}

export async function getAllPrompts() {
  return db.query.aiPrompts.findMany({
    orderBy: (prompts, { asc }) => [asc(prompts.category), asc(prompts.name)],
  });
}
```

## Anti-Patterns to Avoid

### 1. Storing embeddings in the entry table

**Problem:** Large vector columns (1536 floats) slow down queries on the main content table
**Solution:** Separate `knowledge_embeddings` table with chunked content; JOIN only when searching

### 2. Synchronous Mux upload polling

**Problem:** Blocking while waiting for video processing (can take minutes)
**Solution:** Webhooks + status column + UI polling; never await processing completion

### 3. Hardcoded prompts in code

**Problem:** Requires code deployment to change AI behavior
**Solution:** Database-stored prompts with fallbacks; getPrompt() function

### 4. Chatbot visible without authentication

**Problem:** Public pages expose chatbot to unauthenticated users, potential abuse
**Solution:** Wrap FloatingChat in `<SignedIn>` component from Clerk

### 5. Unbounded knowledge search

**Problem:** Returning too much context overwhelms the model and wastes tokens
**Solution:** Limit to 4 chunks maximum, filter by similarity threshold > 0.5

### 6. No version history for prompts

**Problem:** Can't see what changed or roll back bad prompt edits
**Solution:** `ai_prompt_versions` table stores every edit with timestamp and editor
