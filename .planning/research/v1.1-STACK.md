# Stack Research: v1.1 Features

**Project:** CantoMando Blueprint LMS
**Milestone:** v1.1 Coach Tools & Student AI Assistant
**Researched:** 2026-01-28
**Confidence:** HIGH (verified with npm registry + official docs)

---

## Executive Summary

v1.1 requires stack additions in four areas: bulk video upload, RAG knowledge base, AI chatbot UI, and AI prompts management. The recommendations prioritize integration with the existing Neon Postgres/Drizzle stack to minimize operational complexity.

**Key decisions:**
- **RAG:** pgvector in Neon (NOT external vector DB) - keeps data unified, eliminates sync complexity
- **Chatbot UI:** assistant-ui with Vercel AI SDK - production-ready, shadcn/ui compatible
- **Bulk Upload:** Custom queue with Mux Direct Uploads API (NOT mux-uploader-react for bulk)
- **PDF Processing:** unpdf for serverless-compatible text extraction

**Why NOT use mux-uploader-react for bulk:**

- Workaround (changing `key` prop) is hacky for production bulk workflows
- Better: Create Direct Upload URLs server-side, upload files via fetch/PUT

**Recommended Pattern:**
```typescript
// Server action: Create batch of upload URLs
async function createBulkUploadUrls(count: number) {
  const urls = await Promise.all(
    Array(count).fill(null).map(() =>
      mux.video.uploads.create({
        new_asset_settings: { playback_policies: ['public'] },
        cors_origin: process.env.NEXT_PUBLIC_APP_URL,
      })
    )
  );
  return urls.map(u => ({ uploadId: u.id, url: u.url }));
}

// Client: Upload files in parallel with progress tracking
async function uploadFile(uploadUrl: string, file: File) {
  return fetch(uploadUrl, {
    method: 'PUT',
    body: file,
    headers: { 'Content-Type': file.type },
  });
}
```

### 2. RAG Knowledge Base

#### 2.1 Vector Storage: pgvector in Neon

| Component | Choice | Why |
|-----------|--------|-----|
| Vector DB | pgvector (Neon extension) | Already using Neon; eliminates data sync; HNSW indexes up to 50M vectors |
| NOT | Pinecone, Weaviate, Qdrant | Adds operational complexity, network latency, vendor lock-in |

**Why pgvector over external vector DB:**
1. **Single database** - No sync between Postgres and vector DB
2. **Transactional consistency** - Knowledge base updates atomic with metadata
3. **Cost** - Free with Neon, Pinecone starts at ~$70/month
4. **Scale** - pgvector handles up to 50M vectors; LMS will have <100K
5. **Performance** - HNSW index gives 30ms p99 latency, sufficient for chatbot

**Drizzle Schema for Knowledge Base:**
```typescript
import { pgTable, text, uuid, timestamp, vector, index } from 'drizzle-orm/pg-core';

// Enable pgvector (run in migration)
// CREATE EXTENSION IF NOT EXISTS vector;

export const knowledgeEntries = pgTable('knowledge_entries', {
  id: uuid('id').primaryKey().defaultRandom(),
  title: text('title').notNull(),
  content: text('content').notNull(),
  category: text('category', {
    enum: ['packages', 'coaching', 'chinese_help', 'faq', 'general']
  }).notNull(),
  sourceType: text('source_type', {
    enum: ['manual', 'pdf', 'doc']
  }).notNull(),
  sourceFileName: text('source_file_name'), // Original filename for file uploads
  embedding: vector('embedding', { dimensions: 1536 }),
  metadata: jsonb('metadata'), // Flexible: page numbers, headings, etc.
  createdBy: text('created_by').references(() => users.clerkId),
  createdAt: timestamp('created_at').defaultNow(),
  updatedAt: timestamp('updated_at').defaultNow(),
}, (table) => [
  index('knowledge_embedding_idx').using('hnsw', table.embedding.op('vector_cosine_ops')),
]);

// Chunks for large documents (PDFs)
export const knowledgeChunks = pgTable('knowledge_chunks', {
  id: uuid('id').primaryKey().defaultRandom(),
  entryId: uuid('entry_id').references(() => knowledgeEntries.id, { onDelete: 'cascade' }),
  content: text('content').notNull(),
  chunkIndex: integer('chunk_index').notNull(),
  embedding: vector('embedding', { dimensions: 1536 }),
  metadata: jsonb('metadata'), // Page number, section heading
}, (table) => [
  index('chunk_embedding_idx').using('hnsw', table.embedding.op('vector_cosine_ops')),
]);
```

**Source:** [Drizzle pgvector Guide](https://orm.drizzle.team/docs/guides/vector-similarity-search), [Neon pgvector](https://neon.com/docs/extensions/pgvector)

#### 2.2 Embeddings: OpenAI text-embedding-3-small

| Model | Dimensions | Cost per 1M tokens | Why |
|-------|------------|-------------------|-----|
| text-embedding-3-small | 1536 | $0.02 | 5x cheaper than ada-002, better performance |
| NOT ada-002 | 1536 | $0.10 | Legacy, worse performance, 5x more expensive |
| NOT text-embedding-3-large | 3072 | $0.13 | Overkill for FAQ/help content |

**Why text-embedding-3-small:**
- 62.3% MTEB score (vs 61.0% for ada-002)
- $0.02/1M tokens = ~$0.50 to embed 100K documents
- 1536 dimensions matches common RAG patterns

**Embedding Generation Pattern:**
```typescript
import OpenAI from 'openai';

const openai = new OpenAI();

export async function generateEmbedding(text: string): Promise<number[]> {
  const response = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: text.replaceAll('\n', ' ').trim(),
  });
  return response.data[0].embedding;
}

// Batch embeddings for efficiency
export async function generateBatchEmbeddings(texts: string[]): Promise<number[][]> {
  const response = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: texts.map(t => t.replaceAll('\n', ' ').trim()),
  });
  return response.data.map(d => d.embedding);
}
```

**Source:** [OpenAI Embeddings](https://platform.openai.com/docs/models/text-embedding-3-small)

#### 2.3 Text Chunking

| Library | Version | Purpose | Confidence |
|---------|---------|---------|------------|
| @langchain/textsplitters | 1.0.1 | RecursiveCharacterTextSplitter | HIGH |

**Why use a chunking library:**
- RAG performance depends heavily on chunk quality
- Naive splitting breaks semantic coherence
- RecursiveCharacterTextSplitter is battle-tested

**Recommended Settings for LMS Content:**
```typescript
import { RecursiveCharacterTextSplitter } from '@langchain/textsplitters';

const splitter = new RecursiveCharacterTextSplitter({
  chunkSize: 500,      // ~375 words, good for FAQ/help content
  chunkOverlap: 50,    // 10% overlap for context continuity
  separators: ['\n\n', '\n', '. ', ' ', ''], // Preserve paragraphs
});

const chunks = await splitter.splitText(documentText);
```

**Chunk Size Rationale:**
- 500 chars optimal for FAQ/help content (short, focused answers)
- Larger (1000-2000) for long-form documents
- 10% overlap prevents context loss at boundaries

**Source:** [LangChain Text Splitters](https://js.langchain.com/docs/how_to/recursive_text_splitter/)

#### 2.4 PDF Text Extraction

| Library | Version | Purpose | Confidence |
|---------|---------|---------|------------|
| unpdf | 1.4.0 | Serverless-compatible PDF extraction | HIGH |
| NOT pdf-parse | - | Abandoned, maintenance issues | - |

**Why unpdf:**
- Zero dependencies, serverless-optimized
- Works in Next.js API routes and Edge
- Built on Mozilla PDF.js
- TypeScript-first

**PDF Processing Pattern:**
```typescript
import { getDocumentProxy, extractText } from 'unpdf';

export async function extractPdfContent(fileBuffer: ArrayBuffer): Promise<string> {
  const pdf = await getDocumentProxy(new Uint8Array(fileBuffer));
  const { text } = await extractText(pdf, { mergePages: true });
  return text;
}

// Full pipeline: Upload -> Extract -> Chunk -> Embed -> Store
export async function processKnowledgeFile(file: File, category: string) {
  // 1. Extract text
  const buffer = await file.arrayBuffer();
  const text = await extractPdfContent(buffer);

  // 2. Chunk
  const chunks = await splitter.splitText(text);

  // 3. Embed all chunks
  const embeddings = await generateBatchEmbeddings(chunks);

  // 4. Store in Postgres
  const entry = await db.insert(knowledgeEntries).values({
    title: file.name,
    content: text,
    category,
    sourceType: 'pdf',
    sourceFileName: file.name,
  }).returning();

  await db.insert(knowledgeChunks).values(
    chunks.map((content, i) => ({
      entryId: entry[0].id,
      content,
      chunkIndex: i,
      embedding: embeddings[i],
    }))
  );
}
```

**Source:** [unpdf GitHub](https://github.com/unjs/unpdf)

#### 2.5 Similarity Search Query

```typescript
import { cosineDistance, desc, gt, sql } from 'drizzle-orm';

export async function findRelevantKnowledge(query: string, limit = 5) {
  const queryEmbedding = await generateEmbedding(query);
  const similarity = sql<number>`1 - (${cosineDistance(knowledgeChunks.embedding, queryEmbedding)})`;

  return await db
    .select({
      content: knowledgeChunks.content,
      entryTitle: knowledgeEntries.title,
      category: knowledgeEntries.category,
      similarity,
    })
    .from(knowledgeChunks)
    .innerJoin(knowledgeEntries, eq(knowledgeChunks.entryId, knowledgeEntries.id))
    .where(gt(similarity, 0.5)) // Threshold for relevance
    .orderBy(desc(similarity))
    .limit(limit);
}
```

```typescript
// lib/prompts.ts
import { db } from '@/db';
import { aiPrompts } from '@/db/schema';
import { eq } from 'drizzle-orm';

const promptCache = new Map<string, string>();

export async function getPrompt(name: string): Promise<string> {
  if (promptCache.has(name)) {
    return promptCache.get(name)!;
  }

  const result = await db.query.aiPrompts.findFirst({
    where: eq(aiPrompts.name, name),
  });

  if (!result) {
    throw new Error(`Prompt not found: ${name}`);
  }

  promptCache.set(name, result.prompt);
  return result.prompt;
}

// Invalidate cache on update
export function invalidatePromptCache(name?: string) {
  if (name) {
    promptCache.delete(name);
  } else {
    promptCache.clear();
  }
}
```

### 5. Batch Content Operations

**No new libraries needed.** Server actions with transaction support.

**Pattern for Batch Operations:**
```typescript
// lib/actions/batch-content.ts
'use server';

import { db } from '@/db';
import { lessons, modules } from '@/db/schema';
import { eq, inArray } from 'drizzle-orm';

export async function batchUpdateLessons(
  lessonIds: string[],
  updates: { title?: string; description?: string }
) {
  return await db.transaction(async (tx) => {
    const results = await Promise.all(
      lessonIds.map(id =>
        tx.update(lessons)
          .set({ ...updates, updatedAt: new Date() })
          .where(eq(lessons.id, id))
          .returning()
      )
    );
    return results.flat();
  });
}

export async function batchReorderLessons(
  lessonOrders: { id: string; order: number }[]
) {
  return await db.transaction(async (tx) => {
    for (const { id, order } of lessonOrders) {
      await tx.update(lessons)
        .set({ order })
        .where(eq(lessons.id, id));
    }
  });
}

export async function batchMoveLessons(
  lessonIds: string[],
  targetModuleId: string
) {
  return await db.transaction(async (tx) => {
    // Get max order in target module
    const maxOrder = await tx.query.lessons.findFirst({
      where: eq(lessons.moduleId, targetModuleId),
      orderBy: desc(lessons.order),
    });

    let nextOrder = (maxOrder?.order ?? 0) + 1;

    for (const lessonId of lessonIds) {
      await tx.update(lessons)
        .set({ moduleId: targetModuleId, order: nextOrder++ })
        .where(eq(lessons.id, lessonId));
    }
  });
}
```
