# Milestone v6.0: Reading & Dictionary

**Status:** SHIPPED 2026-02-09
**Phases:** 44-49
**Total Plans:** 16

## Overview

Give students a Chinese text reader with instant character-level dictionary lookup, dual Cantonese/Mandarin pronunciation with Azure TTS audio, radical/component breakdown, traditional/simplified conversion, and lesson integration. Think Pleco + Chinese reader apps but with full Cantonese support — a market first.

## Milestone Summary

**Key Decisions:**
- CC-CEDICT + CC-Canto combined dictionary with language-source flagging (cedict/canto/both)
- Intl.Segmenter for zero-dependency Chinese word segmentation
- Single shared Floating UI popup (not per-character Radix Popover instances)
- opencc-js for T/S conversion with HK variant for Cantonese context
- Azure TTS REST API with Redis cache-aside pattern and tiered TTL
- pinyin-pro for tone sandhi with right-to-left third-tone pairing
- Content-shaped loading skeletons over generic spinners
- CC-CEDICT context message for missing entries over AI fallback button
- Raw SQL batch UPDATE for CC-Canto merging (200x faster than individual Drizzle ORM updates)
- Truncate-and-reseed idempotency for dictionary seeding

**Issues Resolved:**
- Pending migrations (0007-0010) applied to Neon database
- Practice tables created from v4.0/v5.0 schema
- hanzi-writer TypeScript types resolved via dist/types path

**Issues Deferred:**
- Custom font files not yet provided by user (sans-serif fallback active)
- Azure Speech credentials not yet configured for production

**Technical Debt Incurred:**
- Dictionary data files (CC-CEDICT, CC-Canto, Make Me a Hanzi) stored in repo data/ directory (~30MB)
- opencc-js loaded via dynamic import to avoid 2MB eager bundle
