<task type="auto">
  <name>Task 1: Create knowledge base schema</name>
  <files>src/db/schema/knowledge.ts, src/db/schema/index.ts</files>
  <action>
Create `src/db/schema/knowledge.ts` with:

1. **kbEntryStatusEnum** pgEnum with values: "draft", "published"

2. **kbCategories** table:
   - id: uuid, defaultRandom, primaryKey
   - name: text, notNull (e.g., "Packages", "Chinese Language Help")
   - slug: text, notNull, unique (URL-safe identifier)
   - description: text (nullable)
   - sortOrder: integer, notNull, default(0)
   - createdAt: timestamp, notNull, defaultNow
   - updatedAt: timestamp, notNull, defaultNow, $onUpdate(() => new Date())

3. **kbEntries** table (manual knowledge entries with rich text):
   - id: uuid, defaultRandom, primaryKey
   - title: text, notNull
   - content: text, notNull (rich text / markdown content)
   - categoryId: uuid, nullable, references kbCategories.id with onDelete set null
   - status: kbEntryStatusEnum, notNull, default("published")
   - createdBy: text, notNull, references users.clerkId with onDelete cascade
   - updatedBy: text, nullable, references users.clerkId with onDelete set null
   - createdAt: timestamp, notNull, defaultNow
   - updatedAt: timestamp, notNull, defaultNow, $onUpdate(() => new Date())

4. **kbFileSources** table (uploaded PDF/doc files):
   - id: uuid, defaultRandom, primaryKey
   - entryId: uuid, notNull, references kbEntries.id with onDelete cascade
   - filename: text, notNull
   - mimeType: text, notNull
   - fileSize: integer, notNull (bytes)
   - storageKey: text, notNull (path/key for file storage, e.g., uploads/kb/uuid.pdf)
   - processedAt: timestamp (nullable, set when chunking completes)
   - chunkCount: integer, default(0)
   - createdAt: timestamp, notNull, defaultNow

5. **kbChunks** table (searchable text chunks from entries and files):
   - id: uuid, defaultRandom, primaryKey
   - entryId: uuid, notNull, references kbEntries.id with onDelete cascade
   - fileSourceId: uuid, nullable, references kbFileSources.id with onDelete cascade
   - content: text, notNull (chunk text content)
   - chunkIndex: integer, notNull (order within source)
   - metadata: text (nullable, JSON string for extra context like page number)
   - createdAt: timestamp, notNull, defaultNow

6. **Relations:**
   - kbCategoriesRelations: kbCategories has many kbEntries
   - kbEntriesRelations: kbEntries belongs to kbCategories, has many kbFileSources, has many kbChunks
   - kbFileSourcesRelations: kbFileSources belongs to kbEntries, has many kbChunks
   - kbChunksRelations: kbChunks belongs to kbEntries, belongs to kbFileSources

7. Export types: KbCategory, NewKbCategory, KbEntry, NewKbEntry, KbFileSource, NewKbFileSource, KbChunk, NewKbChunk

Then update `src/db/schema/index.ts`:
- Add: `// Knowledge Base` comment
- Add: `export * from "./knowledge";`

Follow existing patterns from prompts.ts and courses.ts. Use `users.clerkId` (text) for createdBy/updatedBy since that's the pattern used in uploads.ts.
  </action>
