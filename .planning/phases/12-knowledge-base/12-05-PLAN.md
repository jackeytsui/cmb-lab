  <action>
Create `src/app/api/knowledge/search/route.ts`:

**GET /api/knowledge/search?q=keyword&limit=10** — Search knowledge base
- Require authentication (auth() must return userId — not role-specific, as chatbot will use this too)
- Query params:
  - q: search query string (required, min 2 chars)
  - limit: max results (optional, default 10, max 50)
  - categoryId: optional category filter

- Search strategy (keyword-based for v1, semantic search can be added in v2):
  1. Search kbChunks.content using ilike for the query term
  2. Also search kbEntries.title and kbEntries.content using ilike
  3. Only include entries with status = "published"
  4. Join results to get entry title, category name

- Implementation:
  ```
  // Search chunks for matching content
  const chunkResults = await db
    .select({
      chunkId: kbChunks.id,
      chunkContent: kbChunks.content,
      chunkIndex: kbChunks.chunkIndex,
      entryId: kbEntries.id,
      entryTitle: kbEntries.title,
      categoryName: kbCategories.name,
    })
    .from(kbChunks)
    .innerJoin(kbEntries, eq(kbChunks.entryId, kbEntries.id))
    .leftJoin(kbCategories, eq(kbEntries.categoryId, kbCategories.id))
    .where(
      and(
        ilike(kbChunks.content, `%${query}%`),
        eq(kbEntries.status, "published"),
        ...(categoryId ? [eq(kbEntries.categoryId, categoryId)] : [])
      )
    )
    .limit(limit);
  ```

- Deduplicate: Group results by entryId, aggregate matching chunks per entry
- Return format:
  ```json
  {
    "results": [
      {
        "entryId": "uuid",
        "entryTitle": "Entry Title",
        "categoryName": "FAQ",
        "matchingChunks": [
          { "chunkId": "uuid", "content": "...matching text...", "chunkIndex": 0 }
        ],
        "matchCount": 3
      }
    ],
    "query": "search term",
    "totalResults": 5
  }
  ```
- Order results by matchCount descending (most relevant first)

- Sanitize the search query: strip SQL-special chars, trim whitespace, escape % and _ for ilike safety. Replace % with \% and _ with \_ in the query string before using in ilike.

Error handling: return 400 if q is missing or too short, 500 on internal errors.
