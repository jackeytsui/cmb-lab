   **GET /api/admin/knowledge/categories** — List all categories
   - Require hasMinimumRole("coach")
   - Query: db.select().from(kbCategories).orderBy(asc(kbCategories.sortOrder))
   - Return: { categories: [...] }

   **POST /api/admin/knowledge/categories** — Create category
   - Require hasMinimumRole("coach")
   - Body: { name: string, slug: string, description?: string }
   - Validate: name required, slug required (lowercase, hyphen-separated)
   - Auto-generate slug from name if not provided: name.toLowerCase().replace(/\s+/g, "-").replace(/[^a-z0-9-]/g, "")
   - Set sortOrder to max existing sortOrder + 1
   - Insert into kbCategories
   - Return: { category: {...} } with 201 status

2. Create `src/app/api/admin/knowledge/categories/[categoryId]/route.ts`:

   **PATCH /api/admin/knowledge/categories/[categoryId]** — Update category
   - Require hasMinimumRole("coach")
   - Body: { name?: string, slug?: string, description?: string, sortOrder?: number }
   - Update only provided fields using spread
   - Return: { category: {...} }

   **DELETE /api/admin/knowledge/categories/[categoryId]** — Delete category
   - Require hasMinimumRole("admin") (only admins can delete categories)
   - Delete from kbCategories where id = categoryId
   - Entries in this category will have categoryId set to null (onDelete: set null)
   - Return: { success: true }

Follow the pattern from admin/prompts/route.ts and admin/courses/route.ts for error handling and auth checks.

<task type="auto">
  <name>Task 2: Create entry CRUD API routes with auto-chunking</name>
  <files>src/app/api/admin/knowledge/entries/route.ts, src/app/api/admin/knowledge/entries/[entryId]/route.ts</files>
  <action>
1. Create `src/app/api/admin/knowledge/entries/route.ts`:

   **GET /api/admin/knowledge/entries** — List entries with optional filters
   - Require hasMinimumRole("coach")
   - Query params: ?categoryId=uuid&status=published&search=keyword
   - Build where conditions array:
     - If categoryId: eq(kbEntries.categoryId, categoryId)
     - If status: eq(kbEntries.status, status)
     - If search: or(ilike(kbEntries.title, `%${search}%`), ilike(kbEntries.content, `%${search}%`))
   - Join with kbCategories to include category name
   - Order by updatedAt desc
   - Return: { entries: [...] } where each entry includes categoryName

   **POST /api/admin/knowledge/entries** — Create entry
   - Require hasMinimumRole("coach")
   - Get userId from auth()
   - Body: { title: string, content: string, categoryId?: string, status?: "draft"|"published" }
   - Validate: title required, content required
   - Insert into kbEntries with createdBy = userId
   - After insert, auto-chunk the content:
     - Split content by double newline (\n\n) to get paragraphs
     - Filter out empty paragraphs
     - For each paragraph, create a chunk in kbChunks with:
       - entryId: new entry id
       - fileSourceId: null (manual entry, not from file)
       - content: paragraph text (trim whitespace)
       - chunkIndex: sequential index (0, 1, 2...)
     - Use db.insert(kbChunks).values(chunksArray) for batch insert
   - Return: { entry: {...}, chunkCount: N } with 201 status

2. Create `src/app/api/admin/knowledge/entries/[entryId]/route.ts`:

   **GET /api/admin/knowledge/entries/[entryId]** — Get single entry with details
   - Require hasMinimumRole("coach")
   - Query entry with category name, chunk count, file source count
   - Use db.query.kbEntries.findFirst with relations or manual joins
   - Return: { entry: {..., categoryName, chunkCount, fileSourceCount} }

   **PATCH /api/admin/knowledge/entries/[entryId]** — Update entry
   - Require hasMinimumRole("coach")
   - Get userId from auth()
   - Body: { title?: string, content?: string, categoryId?: string, status?: "draft"|"published" }
   - Update entry with updatedBy = userId
   - If content changed: delete existing chunks where fileSourceId is null (manual chunks only), re-chunk new content
   - Return: { entry: {...} }

   **DELETE /api/admin/knowledge/entries/[entryId]** — Delete entry
   - Require hasMinimumRole("coach")
   - Delete from kbEntries where id = entryId
   - Cascade deletes chunks and file sources automatically
   - Return: { success: true }

Import ilike from drizzle-orm for case-insensitive search. Use `and()` and `or()` to compose where conditions.
  </action>
  <verify>npm run build compiles without errors</verify>
