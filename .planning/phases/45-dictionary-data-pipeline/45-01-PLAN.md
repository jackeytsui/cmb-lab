**Step C: Create parser library at `src/lib/dictionary-parsers.ts`**

Export the following functions:

1. **`normalizeCedictPinyin(raw: string): string`** - Replace `u:` with `v` for pinyin-pro compatibility. Example: `"nu:3 ren2"` -> `"nv3 ren2"`.

2. **`parseCedictLine(line: string): ParsedCedictEntry | null`** - Parse CC-CEDICT format: `Traditional Simplified [pinyin] /def1/def2/`. Use regex `/^(\S+)\s+(\S+)\s+\[([^\]]+)\]\s+\/(.+)\/\s*$/`. Skip lines starting with `#` or empty. Return `{ traditional, simplified, pinyin, definitions }`.

3. **`parseCantoLine(line: string): ParsedCantoEntry | null`** - Parse CC-Canto format (same as CEDICT but with optional `{jyutping}` block): regex `/^(\S+)\s+(\S+)\s+\[([^\]]+)\]\s+(?:\{([^}]+)\}\s+)?\/(.+)\/\s*$/`. Return `{ traditional, simplified, pinyin, jyutping, definitions }`.

4. **`parseMakeHanziDictionary(filePath: string): MakeHanziDict[]`** - Read file, split by newline, JSON.parse each line. Use optional chaining for etymology fields. Return array of `{ character, definition, pinyin, decomposition, radical, etymology? }`.

5. **`parseMakeHanziGraphics(filePath: string): MakeHanziGraphics[]`** - Read file, split by newline, JSON.parse each line. Return array of `{ character, strokes, medians }`.

Define TypeScript interfaces:
- `ParsedCedictEntry { traditional, simplified, pinyin, definitions }`
- `ParsedCantoEntry extends ParsedCedictEntry { jyutping: string | null }`
- `MakeHanziDict { character, definition, pinyin, decomposition, radical, matches?, etymology? }`
- `MakeHanziGraphics { character, strokes, medians }`

All parse functions reading files should use `readFileSync(filePath, "utf-8")`.

Do NOT use `convert` from pinyin-pro in this file (that goes in the seed script). This file is pure parsing.

**Step B: Create seed script at `scripts/seed-dictionary.ts`**

Follow the existing `src/db/seed.ts` pattern (dotenv + neon + drizzle). The script must:

1. **Setup:**
   ```typescript
   import { drizzle } from "drizzle-orm/neon-http";
   import { neon } from "@neondatabase/serverless";
   import { config } from "dotenv";
   import { convert } from "pinyin-pro";
   import ToJyutping from "to-jyutping";
   import { readFileSync } from "fs";
   import { and, eq } from "drizzle-orm";
   import { dictionaryEntries, characterData } from "../src/db/schema";
   import { parseCedictLine, parseCantoLine, parseMakeHanziDictionary, parseMakeHanziGraphics, normalizeCedictPinyin } from "../src/lib/dictionary-parsers";

   config({ path: ".env.local" });
   // ... validate DATABASE_URL, create db instance
   ```

2. **`batchInsert(table, rows, label, batchSize=500)`** — Generic batch inserter with progress logging every 5000 rows. Use `onConflictDoNothing()`. Wrap each batch in try/catch; on failure, retry row-by-row.

3. **Phase A: Seed CC-CEDICT (~124K entries)**
   - Read and parse the CC-CEDICT file using `parseCedictLine`
   - For each entry, compute:
     - `pinyinDisplay` via `convert(normalizeCedictPinyin(pinyin))` — this converts numbered pinyin with u: handling to tone marks
     - `isSingleChar` as `traditional.length === 1`
     - `source` as `"cedict"`
   - Batch insert all entries into dictionary_entries
   - Log count of entries inserted

4. **Phase B: Merge CC-Canto data (two sub-steps)**

   **B1: CC-Canto readings file** (jyutping for existing CEDICT entries)
   - Read and parse the readings file (same CC-CEDICT format with `{jyutping}` blocks) using `parseCantoLine`
   - Build a Map keyed on `traditional+simplified+pinyin` -> jyutping
   - Batch UPDATE existing dictionary_entries: for each reading with a matching (traditional, simplified, pinyin) triple, SET `jyutping` and `source='both'`
   - Since individual UPDATEs are slow (~22K), batch them: load all readings into memory, then iterate with UPDATE queries. Accept that this takes 1-3 minutes for a one-time seed.

   **B2: CC-Canto dictionary file** (Cantonese-only entries)
   - Read and parse using `parseCantoLine`
   - For entries NOT already in dictionary_entries (check by traditional+simplified+pinyin), INSERT with `source='canto'`
   - For entries that DO exist, UPDATE `jyutping` and set `source='both'` (same as B1)
   - Compute `pinyinDisplay` and `isSingleChar` for new entries

5. **Phase C: Seed Make Me a Hanzi (~9K characters)**
   - Parse `dictionary.txt` using `parseMakeHanziDictionary` -> Map by character
   - Parse `graphics.txt` using `parseMakeHanziGraphics` -> Map by character
   - For each character, merge dict + graphics data into a character_data row:
     - `pinyin`: from dict.pinyin array
     - `jyutping`: use `ToJyutping.getJyutpingList(character)` to generate jyutping array. The library returns an array of `[char, jyutping]` tuples; extract the jyutping strings. If the result is empty or null, set to null.
     - `radical`: from dict.radical
     - `radicalMeaning`: look up from a static KANGXI_RADICALS map (define a const object mapping ~214 radicals to their English meanings, e.g., `{ "人": "person", "口": "mouth", "水": "water", ... }`). Research says Make Me a Hanzi does NOT include radical meanings, so we must provide them.
     - `strokeCount`: from graphics.strokes.length
     - `decomposition`: from dict.decomposition
     - `etymologyType`: from dict.etymology?.type
     - `etymologyHint`: from dict.etymology?.hint
     - `etymologyPhonetic`: from dict.etymology?.phonetic
     - `etymologySemantic`: from dict.etymology?.semantic
     - `definition`: from dict.definition
     - `strokePaths`: from graphics.strokes (array of SVG path strings)
     - `strokeMedians`: from graphics.medians (array of point arrays)
   - Batch insert into character_data using `onConflictDoNothing()` (natural PK = character)
   - Log count of characters inserted

6. **Final summary:** Log total counts for each table.

**Important implementation details:**
- The `convert` function from `pinyin-pro` converts numbered pinyin to tone marks: `convert("zhong1 guo2")` returns `"zhōng guó"`. Always call `normalizeCedictPinyin` first to handle `u:` -> `v`.
- Handle tone 5 (neutral): `convert("ma5")` should produce `"ma"` (no mark). Test this.
- For the Kangxi radicals map, include at least the 50 most common radicals. It is acceptable to have a partial map — set radicalMeaning to null for unmapped radicals.
- The `to-jyutping` import may need adjustment based on ESM/CJS: try `import ToJyutping from "to-jyutping"` and if that fails try `const ToJyutping = require("to-jyutping")` or named import. Check how it's used in existing `src/lib/search-utils.ts` for the correct import pattern.
- CC-Canto readings file format: Inspect the actual downloaded file before parsing. It may be plain CEDICT-format with `{jyutping}` or it may be a simpler format. Adjust the parser if needed.
