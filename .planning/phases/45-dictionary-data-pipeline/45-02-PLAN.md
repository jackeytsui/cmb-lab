  <action>
Create `src/app/api/dictionary/lookup/route.ts` following the project convention established in `src/app/api/search/route.ts`:

```typescript
// GET /api/dictionary/lookup?word=X
// Returns dictionary entries matching the word (traditional or simplified)
```

**Implementation:**

1. **Auth check:** Use `auth()` from `@clerk/nextjs/server`. Return 401 if no userId.

2. **Parameter validation:** Extract `word` from searchParams. Return 400 with `{ error: "Missing word parameter" }` if empty or missing.

3. **Query:** Use Drizzle `db.select()` from `dictionaryEntries` with `or(eq(traditional, word), eq(simplified, word))`.

4. **Response shape:**
   ```json
   {
     "entries": [
       {
         "id": "uuid",
         "traditional": "中國",
         "simplified": "中国",
         "pinyin": "Zhong1 guo2",
         "pinyinDisplay": "Zhōng guó",
         "jyutping": "zung1 gwok3",
         "definitions": ["China", "Middle Kingdom"],
         "source": "both",
         "isSingleChar": false
       }
     ]
   }
   ```

5. **Error handling:** Wrap query in try/catch. On error, log with `console.error` and return 500 with `{ error: "Dictionary lookup failed" }`.

6. **No pagination needed** — a word typically has 1-5 entries, never more than ~20.

Imports needed: `NextRequest`, `NextResponse`, `auth`, `db`, `dictionaryEntries`, `or`, `eq`.

<task type="auto">
  <name>Task 2: Create character detail API endpoint</name>
  <files>src/app/api/dictionary/character/route.ts</files>
  <action>
Create `src/app/api/dictionary/character/route.ts` following project convention:

```typescript
// GET /api/dictionary/character?char=X
// Returns character data (radical, etymology, strokes) plus example words
```

**Implementation:**

1. **Auth check:** Use `auth()` from `@clerk/nextjs/server`. Return 401 if no userId.

2. **Parameter validation:** Extract `char` from searchParams. Validate that it is exactly 1 character (use spread operator `[...char].length === 1` to handle multi-byte characters correctly). Return 400 with `{ error: "Single character required" }` if invalid.

3. **Character data query:** Use Drizzle `db.select()` from `characterData` with `eq(characterData.character, char)`. This uses the varchar(1) natural PK.

4. **Example words query:** Use Drizzle `db.select()` from `dictionaryEntries` with:
   ```typescript
   or(
     sql`${dictionaryEntries.traditional} LIKE ${'%' + char + '%'}`,
     sql`${dictionaryEntries.simplified} LIKE ${'%' + char + '%'}`
   )
   ```
   Limit to 20 results. Filter to only multi-character entries (where the entry is NOT just the single character itself) to show actual example words.

   To make examples more useful, prefer:
   - Exclude single-character entries that are just the character itself (where `traditional === char` and `isSingleChar === true`)
   - Order by `frequencyRank` ASC NULLS LAST to show common words first (frequent words have lower rank numbers)

5. **Response shape:**
   ```json
   {
     "character": {
       "character": "人",
       "pinyin": ["ren2"],
       "jyutping": ["jan4"],
       "radical": "人",
       "radicalMeaning": "person",
       "strokeCount": 2,
       "decomposition": "人",
       "etymologyType": "pictographic",
       "etymologyHint": "Pictograph of a person",
       "etymologyPhonetic": null,
       "etymologySemantic": null,
       "definition": "man, person",
       "frequencyRank": null,
       "strokePaths": ["M ..."],
       "strokeMedians": [[[x,y],...]],
       "createdAt": "..."
     },
     "examples": [
       {
         "traditional": "人民",
         "simplified": "人民",
         "pinyin": "ren2 min2",
         "pinyinDisplay": "ren min",
         "definitions": ["the people"],
         "source": "cedict"
       }
     ]
   }
   ```

6. **Handle missing character:** If no character data found, return `{ character: null, examples: [] }` with status 200 (not 404 — the character may simply not be in Make Me a Hanzi).

7. **Error handling:** Wrap in try/catch. On error, log and return 500 with `{ error: "Character lookup failed" }`.

Imports needed: `NextRequest`, `NextResponse`, `auth`, `db`, `characterData`, `dictionaryEntries`, `eq`, `or`, `sql`, `asc`.
