  <name>Task 1: Assignment CRUD + resolution library</name>
  <files>src/lib/assignments.ts</files>
  <action>
Create `src/lib/assignments.ts` following the exact pattern of `src/lib/practice.ts`.

**CRUD functions:**

1. `createAssignment(data: { practiceSetId, targetType, targetId, assignedBy, dueDate? })`:
   - Verify practice set exists, is published, and not soft-deleted (query practiceSets where status='published' AND deletedAt IS NULL)
   - Validate targetId exists in the correct table based on targetType:
     - 'course' -> courses table (where deletedAt IS NULL)
     - 'module' -> modules table (where deletedAt IS NULL)
     - 'lesson' -> lessons table (where deletedAt IS NULL)
     - 'student' -> users table
     - 'tag' -> tags table
   - Insert into practiceSetAssignments. Catch unique constraint violation (code '23505') and return a friendly error ("Assignment already exists") instead of 500.
   - Return the created assignment.

2. `deleteAssignment(id: string)`:
   - Hard delete from practiceSetAssignments (no soft delete — assignments are lightweight join records).
   - Return deleted row or null.

3. `updateAssignmentDueDate(id: string, dueDate: Date | null)`:
   - Update the dueDate column. Return updated row or null.

4. `listAssignmentsForSet(practiceSetId: string)`:
   - Select all assignments for a given practice set, ordered by createdAt desc.
   - Return array of assignments.

**Resolution function:**

5. `getStudentAssignments(userId: string)`:
   - This is the core query. Given a student's internal user ID, resolve ALL assigned practice sets from 5 paths.
   - Step-by-step approach (consistent with analytics patterns in the codebase):

   Step 1: Collect all valid target entries for this student:
   - Direct: `{ type: 'student', id: userId }`
   - Tags: Query studentTags where userId matches, get tagIds -> `{ type: 'tag', id: tagId }` for each
   - Course enrollment: Query courseAccess where userId matches (and not expired: expiresAt IS NULL OR expiresAt > now), get courseIds -> `{ type: 'course', id: courseId }` for each
   - Modules: For enrolled courseIds, query modules where courseId IN (...) AND deletedAt IS NULL, get moduleIds -> `{ type: 'module', id: moduleId }` for each
   - Lessons: For those moduleIds, query lessons where moduleId IN (...) AND deletedAt IS NULL, get lessonIds -> `{ type: 'lesson', id: lessonId }` for each

   Step 2: Build OR conditions for practiceSetAssignments query:
   - Group target entries by type, build conditions like: `(targetType = 'student' AND targetId IN [...]) OR (targetType = 'tag' AND targetId IN [...]) OR ...`
   - If a type has no entries (e.g., no tags), skip that condition.
   - If NO valid target entries exist at all, return empty array.

   Step 3: Query practiceSetAssignments with the OR conditions, JOIN with practiceSets for title/description/status:
   - Filter: practiceSets.status = 'published' AND practiceSets.deletedAt IS NULL
   - Select: assignment.id, assignment.practiceSetId, assignment.targetType, assignment.targetId, assignment.dueDate, assignment.createdAt, practiceSet.title, practiceSet.description

   Step 4: LEFT JOIN with practiceAttempts for completion status:
   - For each assignment's practiceSetId, check if user has a completed attempt (completedAt IS NOT NULL).
   - Get bestScore (MAX score) and attemptCount (COUNT).

   Step 5: Deduplicate by practiceSetId:
   - If the same practice set is assigned through multiple paths (e.g., to both a course AND a lesson within that course), keep only one entry.
   - Priority: lesson > module > course > tag > student (most specific wins).
   - Use a Map keyed by practiceSetId, overwrite if current targetType has higher priority.

   Step 6: Return array of `ResolvedAssignment` objects:
   ```typescript
   interface ResolvedAssignment {
     assignmentId: string;
     practiceSetId: string;
     practiceSetTitle: string;
     practiceSetDescription: string | null;
     targetType: string;
     targetId: string;
     dueDate: Date | null;
     assignedAt: Date;
     status: "pending" | "completed";
     bestScore: number | null;
     attemptCount: number;
   }
   ```

   Export the `ResolvedAssignment` interface for use by the dashboard component.

**Import pattern:** Use `import { db } from "@/db"` and import tables from `@/db/schema`.
