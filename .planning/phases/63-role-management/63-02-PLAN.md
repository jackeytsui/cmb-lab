**Step 1: Create admin roles page**
Create `src/app/(dashboard)/admin/roles/page.tsx` as a `"use client"` component following the PromptList client-side search pattern:

**State:**
- `roles: RoleWithCount[]` (fetched from API), `search: string`, `dialogOpen: boolean`, `editingRole: Role | null`, `deleteConfirmRole: Role | null`, `isLoading: boolean`, `isSeeding: boolean`

**Type:**
- `RoleWithCount = { id, name, description, color, sortOrder, studentCount, createdAt }`

**Functions:**
- `fetchRoles()`: GET `/api/admin/roles?search=${search}` (or no search param for initial load). Set roles state. Use `useCallback` with `search` dependency.
- `handleDelete(roleId)`: DELETE `/api/admin/roles/${roleId}`. If 409, show `toast.error(data.error)`. If success, `toast.success("Role deleted")`, close confirm dialog, refetch.
- `handleSeedTemplates()`: POST `/api/admin/roles/templates`. Show toast with created/skipped counts. Refetch.

**Initial load:** `useEffect` calls `fetchRoles()` on mount and when search changes (with 300ms debounce via `setTimeout`/cleanup pattern).

**Layout:**
- Page header: "Roles" title with Shield icon, description text
- Toolbar row: Search Input (with Search icon, placeholder "Search roles...", value bound to `search` state), "Seed Templates" button (visible only to admin -- conditionally render based on user role, or always show since the API guards it), "Create Role" Plus button
- Role list: Map over filtered roles, each row showing:
  - Badge component with `style={{ backgroundColor: role.color + "20", color: role.color }}` and role name text
  - Description text (truncated, text-muted-foreground)
  - Student count: `{role.studentCount} student(s)` with Users icon
  - Created date: `formatDistanceToNow(new Date(role.createdAt), { addSuffix: true })` from date-fns
  - Edit button (Pencil icon) -> opens dialog with `editingRole = role`
  - Delete button (Trash2 icon) -> opens confirmation dialog
- Empty state when no roles: "No roles found" message with create prompt
- Dialog for create/edit: `<Dialog open={dialogOpen}>` containing `<RoleForm>` component
- Delete confirmation dialog: Show role name, if role has students show warning "This role is assigned to X students. Remove all assignments before deleting." with only a Close button (no delete action). If zero students, show "Are you sure?" with Cancel and Delete buttons.
