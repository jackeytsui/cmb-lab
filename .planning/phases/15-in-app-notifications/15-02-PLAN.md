  <name>Task 1: Create notification list and count endpoints</name>
  <files>src/app/api/notifications/route.ts, src/app/api/notifications/count/route.ts</files>
  <action>
    Create `src/app/api/notifications/route.ts`:
    - GET handler: Requires authenticated user via getCurrentUser()
    - Query notifications table WHERE userId = current user's DB id AND deletedAt IS NULL
    - Order by createdAt DESC
    - Limit to 50 most recent (hardcoded pagination -- sufficient for MVP)
    - Return JSON array of notifications

    Create `src/app/api/notifications/count/route.ts`:
    - GET handler: Requires authenticated user
    - Query COUNT(*) WHERE userId = current user AND read = false AND deletedAt IS NULL
    - Return `{ unreadCount: number }`
    - This is the lightweight endpoint called every 30 seconds by the polling hook

    Both routes must:
    - Use getCurrentUser() from "@/lib/auth" for auth (returns user with DB id)
    - Return 401 if not authenticated
    - Use Drizzle ORM for queries
    - Follow existing API route patterns (see submissions/route.ts for style)
  </action>
  <verify>
    - `npm run build` compiles without errors
    - Both route files export GET handlers
  </verify>
  <done>GET /api/notifications returns notification list, GET /api/notifications/count returns unread count integer</done>
</task>

<task type="auto">
  <name>Task 2: Create mark-read, mark-all-read, and preferences endpoints</name>
  <files>src/app/api/notifications/[notificationId]/route.ts, src/app/api/notifications/read-all/route.ts, src/app/api/notifications/preferences/route.ts</files>
  <action>
    Create `src/app/api/notifications/[notificationId]/route.ts`:
    - PATCH handler: Requires authenticated user
    - Update notification SET read = true, readAt = new Date() WHERE id = notificationId AND userId = current user
    - Return 404 if notification not found or doesn't belong to user
    - Return updated notification

    Create `src/app/api/notifications/read-all/route.ts`:
    - POST handler: Requires authenticated user
    - Accept JSON body with `{ before: string }` (ISO timestamp) -- this is the timestamp boundary from Pitfall 2
    - Update all notifications SET read = true, readAt = new Date() WHERE userId = current user AND read = false AND createdAt <= before AND deletedAt IS NULL
    - Return `{ updated: number }` with count of affected rows

    Create `src/app/api/notifications/preferences/route.ts`:
    - GET handler: Returns all preference rows for current user. If none exist, return default preferences (all categories unmuted).
    - PATCH handler: Accept `{ category: string, muted: boolean }`. Upsert the preference row for this user + category.
    - Use Drizzle's onConflictDoUpdate or check-then-insert pattern for upsert.

    All routes:
    - Authenticate via getCurrentUser()
    - Validate inputs with basic checks (no need for zod on simple inputs)
    - Return appropriate HTTP status codes
  </action>
  <verify>
    - `npm run build` compiles without errors
    - All route files export their respective handlers
  </verify>
  <done>PATCH mark-read returns updated notification, POST read-all uses timestamp boundary, GET/PATCH preferences support category mute toggling</done>
</task>
