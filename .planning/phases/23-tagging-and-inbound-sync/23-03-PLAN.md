**Create `src/components/tags/TagBadge.tsx`**:
- A small inline badge component that renders a tag with its color.
- Props: `{ name: string, color: string, type: "coach" | "system", onRemove?: () => void }`.
- Styling: rounded-full pill shape, background from hex color at 20% opacity, text in the tag's color.
- System tags have a dashed border and a small "S" icon or "GHL" label to distinguish from coach tags.
- Coach tags have a solid style.
- If `onRemove` provided, show a tiny X button on hover.
- Use the TAG_COLORS palette from research for the predefined color options.

**Create `src/components/tags/TagManager.tsx`**:
- A popover/dialog component for managing tags on a student.
- Props: `{ studentId: string, currentTags: Tag[], allTags: Tag[], onTagsChange: () => void }`.
- Shows a list of all available tags with checkboxes — checked = assigned to this student.
- Toggling a checkbox calls POST or DELETE to `/api/students/${studentId}/tags`.
- Has a "Create Tag" button that expands an inline form: name input + color grid (10 predefined swatches from TAG_COLORS) + type select (coach/system, default coach).
- Creating a new tag calls POST to `/api/admin/tags`, then auto-assigns it to the student.
- Calls `onTagsChange()` after any mutation to refresh parent data.
- Use Radix Popover for the container (already in project).

**Create `src/components/tags/TagFilter.tsx`**:
- A filter bar component for the student list page.
- Fetches all tags via GET `/api/admin/tags` on mount.
- Renders tags as clickable pills — clicking toggles filter on/off.
- Active filters are visually highlighted (solid background).
- Props: `{ selectedTagIds: string[], onFilterChange: (tagIds: string[]) => void }`.
- Supports "Any of" logic: student matches if they have ANY selected tag.

**Modify student list API endpoint** (find the existing student list API route, likely `src/app/api/admin/students/route.ts` or the server-side data fetch in the students page):
- Add support for `?tagIds=id1,id2,id3` query parameter.
- When `tagIds` is present, parse as comma-separated UUID array.
- Filter students using a Drizzle subquery or join: select students whose userId exists in `studentTags` table where `tagId` is in the provided array (`inArray(studentTags.tagId, tagIds)`).
- This implements "Any of" filter logic: a student matches if they have ANY of the selected tags.
- If no `tagIds` param, return all students as before (backward compatible).
- Also include each student's tags in the response by joining with `studentTags` + `tags` tables so the frontend can display TagBadges without a separate API call per student.

**Modify `src/app/(dashboard)/coach/students/StudentList.tsx`**:
- Add a `tags` column (or inline display) showing TagBadge components for each student's tags.
- Fetch student tags alongside student data (either via a joined query in the server component or an additional API call).
- Add a small "+" button next to each student's tags that opens the TagManager popover.
- Integrate tag filtering: accept `selectedTagIds` from parent, append `?tagIds=id1,id2` to the student list API query. The backend handles the filtering via `studentTags` join (server-side, not client-side).

**Modify `src/app/(dashboard)/coach/students/page.tsx`**:
- Add TagFilter component above the student list.
- Manage `selectedTagIds` state and pass to StudentList.
- When filter changes, refetch or re-filter student list.
