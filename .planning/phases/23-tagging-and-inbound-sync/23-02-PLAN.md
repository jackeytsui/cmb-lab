**Create `src/lib/ghl/tag-sync.ts`** with three functions:

1. `syncTagToGhl(userId: string, tagName: string, action: "add" | "remove")`:
   - Get ghlContactId via `getGhlContactId(userId)`. If not linked, return silently.
   - Call `markOutboundChange(ghlContactId, "tag", tagName)` BEFORE the API call.
   - For "add": `ghlClient.post(\`/contacts/${ghlContactId}/tags\`, { tags: [tagName] })`.
   - For "remove": `ghlClient.delete(\`/contacts/${ghlContactId}/tags\`, { tags: [tagName] })`.
   - Call `logSyncEvent({ eventType: \`tag.${action}\`, direction: "outbound", entityType: "tag", entityId: tagName, ghlContactId })`.
   - Fire-and-forget pattern: caller uses `.catch(console.error)`, this function itself throws on failure for logging.
   - Tag naming: When syncing coach tags to GHL, prefix with `lms:` (e.g., tag "VIP" becomes "lms:VIP" in GHL). System tags sync without prefix.

2. `syncTagRemovalFromGhl(userId: string, tagName: string)`:
   - Convenience wrapper that calls `syncTagToGhl(userId, tagName, "remove")`.

3. `processInboundTagUpdate(ghlContactId: string, currentGhlTags: string[])`:
   - Find the LMS user linked to this ghlContactId via `ghlContacts` table lookup.
   - If no linked user, log and return.
   - Get the cached GHL tags from `ghlContacts.cachedData.tags` (or empty array if no cache).
   - Diff `currentGhlTags` against cached tags to find additions and removals.
   - For each added tag: check `isEchoWebhook(ghlContactId, "tag", tagName)`. If echo, skip. Otherwise:
     - Strip `lms:` prefix if present (these are our own tags echoing back — but echo detection should catch them).
     - Check if a matching tag exists in the tags table (by name). If not, create a new system-type tag.
     - Call `assignTag(userId, tagId, null, { source: "webhook" })` — the `source: "webhook"` flag ensures this does NOT trigger outbound sync back to GHL, preventing echo loops.
   - For each removed tag: check echo detection. If not echo:
     - Only remove if the tag type is "system" (GHL-originated). Do NOT auto-remove coach tags.
     - Call `removeTag(userId, tagId, { source: "webhook" })` — same `source: "webhook"` flag to prevent echo.
   - Update `ghlContacts.cachedData.tags` with `currentGhlTags` to keep the diff baseline fresh.
   - Log sync event for each processed tag change.

**Create `src/app/api/webhooks/ghl/route.ts`**:
- Follow the exact pattern from `src/app/api/webhooks/enroll/route.ts` for structure.
- Rate limit using existing `webhookLimiter` (10/min per IP).
- Verify shared secret: `req.headers.get("x-webhook-secret") === process.env.GHL_INBOUND_WEBHOOK_SECRET`. Return 401 if mismatch.
- Parse body as JSON. Validate with Zod schema for `ContactTagUpdate`:
  ```
  { type: "ContactTagUpdate", id: string, tags: string[], locationId: string, ... }
  ```
- Route by `body.type`:
  - `"ContactTagUpdate"`: call `processInboundTagUpdate(body.id, body.tags)`.
  - Other types: log and return 200 (future extensibility).
- Always return `{ received: true }` with 200 status (GHL expects quick acknowledgment).
- Wrap processing in try/catch — never return 5xx to GHL (causes retries and noise).

**Modify `src/app/api/students/[studentId]/tags/route.ts`** (created in Plan 01):
- In the `POST` handler: after `assignTag()` returns successfully, check if `source` is `"api"` (it will be, since this is a UI-initiated action). If so, fire-and-forget call `syncTagToGhl(studentId, tag.name, "add").catch(console.error)`. Import `syncTagToGhl` from `@/lib/ghl/tag-sync`.
- In the `DELETE` handler: after `removeTag()` returns successfully, fire-and-forget call `syncTagToGhl(studentId, tag.name, "remove").catch(console.error)`.
- This is the key wiring that connects "assign tag in LMS" to "sync to GHL". The `assignTag` function itself does NOT trigger sync — the API route does, based on the source context.

**Echo loop prevention architecture:**
- **LMS UI -> API route** (`source: "api"`): `assignTag` + `syncTagToGhl` (outbound sync triggered)
- **GHL webhook -> processInboundTagUpdate** (`source: "webhook"`): `assignTag` only (NO outbound sync)
- **Cron/system** (`source: "system"`): `assignTag` + `syncTagToGhl` (outbound sync triggered)
- Echo detection (`markOutboundChange` / `isEchoWebhook`) serves as a secondary safety net on top of the source-based routing.

Add `GHL_INBOUND_WEBHOOK_SECRET` to the env vars that the app expects (document in the summary).

**Create `src/lib/ghl/contact-fields.ts`**:

1. `fetchGhlContactData(userId: string)`:
   - Look up ghlContacts row for userId. If not linked, return `{ data: null, lastFetchedAt: null }`.
   - Check cache: if `cachedData` exists and `lastFetchedAt` is less than 5 minutes ago, return cached.
   - Otherwise, fetch from GHL: `ghlClient.get(\`/contacts/${contact.ghlContactId}\`)`.
   - Extract: `tags`, `customFields`, `timezone`, `firstName`, `lastName` from response.
   - Update `ghlContacts` row with `cachedData = freshData, lastFetchedAt = new Date()`.
   - On fetch failure, return stale cache if available (graceful degradation).
   - Return `{ data, lastFetchedAt }`.

2. `refreshGhlContactData(userId: string)`:
   - Force refresh: set `lastFetchedAt = null` on the ghlContacts row, then call `fetchGhlContactData(userId)`.

3. `resolveCustomFields(customFields: Array<{id, value}>, fieldMappings: GhlFieldMapping[])`:
   - Maps opaque GHL field IDs to human-readable labels using ghl_field_mappings table.
   - Returns array of `{ lmsConcept, label, value }` for display.
   - Field mappings are fetched from `ghlFieldMappings` table where `isActive = true`.

**Create `src/app/api/students/[studentId]/ghl-profile/route.ts`**:
- `GET` handler:
  - Requires admin/coach role.
  - Calls `fetchGhlContactData(studentId)`.
  - If data exists, also calls `resolveCustomFields` with active field mappings.
  - Returns JSON: `{ data, lastFetchedAt, fields: resolvedFields, ghlContactId, ghlDeepLink }`.
  - `ghlDeepLink`: `https://app.gohighlevel.com/v2/location/${process.env.GHL_LOCATION_ID}/contacts/detail/${ghlContactId}`.
  - Accepts optional `?refresh=true` query param that calls `refreshGhlContactData` instead.
