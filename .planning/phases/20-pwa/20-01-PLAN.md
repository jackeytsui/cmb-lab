<task type="auto">
  <name>Task 2: Create service worker with smart caching strategies</name>
  <files>
    public/sw.js
    next.config.ts
  </files>
  <action>
    Create `public/sw.js` with three caching strategies:

    1. NETWORK-ONLY (never cache):
       - /api/* (all API routes)
       - /sign-in, /sign-up (Clerk auth)
       - /dashboard, /courses/*, /lessons/* (dynamic content)
       - Cross-origin requests (Mux video from stream.mux.com, Clerk from clerk.*.com)
       - Only handle same-origin requests

    2. CACHE-FIRST (cache after first load):
       - request.destination === "style" (CSS)
       - request.destination === "script" (JS bundles)
       - request.destination === "image" (images)
       - request.destination === "font" (fonts)
       - On cache miss: fetch from network, clone response, store in cache, return

    3. NETWORK-FIRST (default for HTML navigations):
       - All other same-origin requests
       - Try network first, fall back to cache on network failure

    Include install event: precache icon files (icon-192x192.png, icon-512x512.png). Call self.skipWaiting().
    Include activate event: delete old caches (any cache name !== current CACHE_NAME). Call self.clients.claim().
    Use CACHE_NAME = "cantomando-v1".

    Update `next.config.ts` to add headers:
    - For /sw.js: Content-Type "application/javascript; charset=utf-8", Cache-Control "no-cache, no-store, must-revalidate"
    - For all routes (/(.*) ): X-Content-Type-Options "nosniff", X-Frame-Options "DENY", Referrer-Policy "strict-origin-when-cross-origin"
    - Keep any existing config options
  </action>
  <verify>
    Run `npm run build` and confirm no errors. Verify public/sw.js exists and contains fetch event listener with all three strategies. Verify next.config.ts has headers function.
  </verify>
  <done>
    Service worker handles fetch events with network-only for API/auth/lesson routes, cache-first for static assets, and network-first for HTML. Cross-origin requests (Mux, Clerk) are not intercepted. next.config.ts serves sw.js with no-cache headers.
  </done>
