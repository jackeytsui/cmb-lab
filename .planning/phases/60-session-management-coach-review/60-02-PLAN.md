  **Submissions list page (`thread-reviews/page.tsx`):**

  Server component with `hasMinimumRole('coach')` guard (redirect to `/dashboard` if not coach). Query all `videoThreadSessions` joined with:
  - `users` table (via `studentId`) for student name and email
  - `videoThreads` table (via `threadId`) for thread title

  Use Drizzle relational query: `db.query.videoThreadSessions.findMany({ with: { student: true, thread: true }, orderBy: [desc(videoThreadSessions.startedAt)] })`.

  Render a page titled "Thread Reviews" with a back link to `/coach`. Display a table/card list with columns:
  - **Student**: `student.name || student.email.split('@')[0]`
  - **Thread**: `thread.title`
  - **Status**: Badge showing `in_progress` (yellow), `completed` (green), or `abandoned` (gray)
  - **Started**: formatted date from `startedAt`
  - **Completed**: formatted date from `completedAt` or "--"
  - Each row links to `/coach/thread-reviews/[session.id]`

  Follow the existing coach page styling patterns: `bg-zinc-900/50 border border-zinc-800 rounded-lg`, `text-zinc-400` for secondary text, cyan accent color for icons. If no sessions exist, show an empty state: "No thread submissions yet."

  **Session detail page (`thread-reviews/[sessionId]/page.tsx`):**

  Server component with `hasMinimumRole('coach')` guard. Accepts `params: { sessionId: string }`.

  Query the session with nested relations:
  ```
  db.query.videoThreadSessions.findFirst({
    where: eq(videoThreadSessions.id, sessionId),
    with: {
      student: true,
      thread: true,
      responses: {
        orderBy: [asc(videoThreadResponses.createdAt)],
        with: { step: true }
      }
    }
  })
  ```

  If not found, call `notFound()`.

  Render:
  1. **Header**: Back link to `/coach/thread-reviews`. Title: "{thread.title}" with student name and status badge below.

  2. **Response Timeline**: A vertical timeline (left border line pattern) showing each response in chronological order. For each response:
     - **Step title**: `response.step.promptText || 'Step'` in bold
     - **Response type badge**: styled pill (text=cyan, audio=purple, video=rose, button=amber)
     - **Response content** based on type:
       - `text` or `button`: Display `response.content` in a `bg-zinc-800/50 rounded-lg p-4` block
       - `audio`: If `response.content` looks like a Mux playback ID (not empty, no spaces), render a `<VideoPlayer playbackId={response.content} />` component (Mux handles audio-only playback too). Import from `@/components/video/VideoPlayer`. Wrap in `max-w-md`.
       - `video`: Same as audio -- render `<VideoPlayer playbackId={response.content} />` in an `aspect-video` container with `max-w-2xl`.
       - If `response.metadata?.muxPlaybackId` exists, prefer that over `response.content` for the playback ID.
     - **Timestamp**: `response.createdAt` formatted as relative or absolute time, in `text-zinc-500 text-xs`

  3. **Session summary card** at the bottom: Total responses count, session duration (startedAt to completedAt or "In Progress"), completion status.

  Follow the existing submission detail page pattern (`coach/submissions/[submissionId]/page.tsx`) for layout conventions: container with padding, zinc-900/50 cards, lucide icons, back navigation with ChevronLeft.
