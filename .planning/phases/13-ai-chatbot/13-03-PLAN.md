    **Create `src/hooks/useChatbot.ts`** as a client hook ('use client'):

    - Export function `useChatbot()`:
      - Get `{ preference }` from `useLanguagePreference()`
      - Call `useChat` with:
        - `transport: new DefaultChatTransport({ api: '/api/chat', body: { languagePreference: preference } })`
      - Return the full `useChat` return value: `{ messages, sendMessage, status, stop, setMessages, error }`

    **Update `src/components/chat/ChatPanel.tsx`** — replace the stub state with real AI SDK integration:
    - Remove local message/input/status state stubs
    - Import `useChatbot` from '@/hooks/useChatbot'
    - Add local `input` state with `useState('')` (AI SDK 6 manages messages, not input)
    - Destructure from `useChatbot()`: `{ messages, sendMessage, status, setMessages, error }`
    - Update `handleSubmit`:
      1. `e.preventDefault()`
      2. Guard: `if (!input.trim() || status !== 'ready') return;`
      3. Call `sendMessage({ text: input })`
      4. Clear input: `setInput('')`
    - Update `handleClear`: call `setMessages([])` to reset conversation
    - Auto-scroll: add a `useRef<HTMLDivElement>(null)` for the messages container end, and a `useEffect` that scrolls to bottom when `messages` changes: `messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' })`
    - Streaming indicator: after rendering all messages, if `status === 'submitted'`, show a small "Thinking..." text with `animate-pulse` class
    - If `status === 'streaming'`, the last assistant message is already being streamed by useChat — no extra indicator needed
    - Error display: if `error`, show a small red text above the input: `text-red-400 text-xs px-3`
    - Keep all existing styling from Plan 02
  </action>
