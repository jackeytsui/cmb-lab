<task type="auto">
  <name>Task 2: Create KB search utility and streaming chat API route</name>
  <files>src/lib/chat-utils.ts, src/app/api/chat/route.ts</files>
  <action>
    **Create `src/lib/chat-utils.ts`:**
    - Export `searchKnowledgeBase(query: string): Promise<string>` function
    - Reuse the same ilike search pattern from the existing `/api/knowledge/search` route (NOT calling the HTTP endpoint — do the DB query directly server-side)
    - Sanitize query (escape % and _ wildcards)
    - Search `kbChunks.content` joined with `kbEntries` where `kbEntries.status = 'published'`
    - Limit to 5 results
    - Return formatted string: each result as `[entryTitle]: chunkContent` separated by double newlines
    - Return "No relevant information found in knowledge base." if no results

    **Create `src/app/api/chat/route.ts`:**
    - `export const maxDuration = 30;` (prevent timeout on multi-step LLM calls)
    - Import: `convertToModelMessages`, `streamText`, `tool`, `UIMessage` from 'ai'; `openai` from '@ai-sdk/openai'; `z` from 'zod'; `auth` from '@clerk/nextjs/server'; `getPrompt` from '@/lib/prompts'; `searchKnowledgeBase` from '@/lib/chat-utils'
    - POST handler:
      1. Auth check via `await auth()` — return 401 if no userId
      2. Parse request body: `{ messages: UIMessage[], languagePreference?: string }`
      3. Load system prompt via `getPrompt('chatbot-system', DEFAULT_SYSTEM_PROMPT)` where DEFAULT_SYSTEM_PROMPT says: "You are a helpful learning assistant for CantoMando Blueprint, an LMS teaching Mandarin and Cantonese simultaneously. Always search the knowledge base before answering factual questions about the platform, courses, or Chinese language. Be encouraging and supportive."
      4. Append language preference to system prompt: `\n\nStudent language preference: ${languagePreference || 'both'}. Respond in the language the student prefers. If they prefer Cantonese, respond in Traditional Chinese with Jyutping. If they prefer Mandarin, respond in Simplified Chinese with Pinyin. If both, include both annotations.`
      5. Call `streamText` with:
         - `model: openai('gpt-4o')`
         - `system: systemPrompt` (constructed above)
         - `messages: await convertToModelMessages(messages.slice(-20))` (trim to last 20 messages to avoid unbounded costs)
         - `tools: { searchKnowledgeBase: tool({ description, inputSchema: z.object({ query: z.string() }), execute: async ({ query }) => searchKnowledgeBase(query) }) }`
         - `maxSteps: 3` (allow multi-step: search KB then answer)
      6. Return `result.toUIMessageStreamResponse()`

    **Important:** Use AI SDK 6 APIs only:
    - `convertToModelMessages` (NOT passing raw UIMessage to model)
    - `toUIMessageStreamResponse()` (NOT toDataStreamResponse)
    - `tool()` helper with Zod schema
  </action>

  </verify>
