<task type="auto">
  <name>Task 3: Configure Drizzle client and generate migrations</name>
  <files>src/db/index.ts, drizzle.config.ts</files>
  <action>
    Create `drizzle.config.ts` at project root:
    ```typescript
    import { defineConfig } from "drizzle-kit";

    export default defineConfig({
      dialect: "postgresql",
      schema: "./src/db/schema/index.ts",
      out: "./src/db/migrations",
      dbCredentials: {
        url: process.env.DATABASE_URL!,
      },
    });
    ```

    Create `src/db/index.ts`:
    ```typescript
    import { drizzle } from "drizzle-orm/neon-http";
    import { neon } from "@neondatabase/serverless";
    import * as schema from "./schema";

    const sql = neon(process.env.DATABASE_URL!);
    export const db = drizzle(sql, { schema });
    ```

    **Migration Generation vs Execution:**

    1. Generate migration SQL files (schema -> SQL):
       ```bash
       npx drizzle-kit generate
       ```
       This creates SQL files in `src/db/migrations/` based on schema definitions.
       Requires DATABASE_URL to be set (Drizzle needs to introspect schema).

    2. Apply migrations to database (SQL -> database):
       ```bash
       npx drizzle-kit migrate
       ```
       OR for development (push schema directly without migration files):
       ```bash
       npx drizzle-kit push
       ```
       Both require DATABASE_URL to connect to the actual database.

    **Execution Flow:**
    - If DATABASE_URL is NOT set: Log instructions for user to configure Neon
    - If DATABASE_URL IS set: Run both generate AND migrate/push to apply schema

    The user_setup section documents that DATABASE_URL must come from Neon Console.
    After user provides DATABASE_URL, re-run this task to apply migrations.
  </action>

      - `npm run db:push` succeeds and tables are created in Neon (verify via Neon Console or `npx drizzle-kit studio`)
    - If DATABASE_URL NOT available:

      - Note that migrations will be applied when DATABASE_URL is configured
  </verify>
