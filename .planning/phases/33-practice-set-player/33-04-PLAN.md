  <files>src/components/practice/player/renderers/MatchingRenderer.tsx</files>
  <action>
Create `src/components/practice/player/renderers/MatchingRenderer.tsx` as a "use client" component.

**Props interface:**
```typescript
interface MatchingRendererProps {
  definition: MatchingDefinition;
  language: "cantonese" | "mandarin" | "both";
  onSubmit: (response: { pairs: { leftId: string; rightId: string }[] }) => void;
  disabled?: boolean;
}
```

**Research recommendation: Use click-to-pair** (not DnD). DnD lines between columns is complex and fragile on mobile. Click-to-pair is cleaner.

**State management:**
1. `matches: Map<string, string>` — maps leftId -> rightId (the student's pairings)
2. `selectedLeft: string | null` — currently selected left item awaiting a right-side click
3. Shuffle left and right columns independently using the deterministic `shuffleArray` function (copy from ExercisePreview.tsx). Use different seeds for left vs right. Memoize with useMemo.

**UX flow:**
1. Student clicks a left-column item -> it highlights (blue border)
2. Student clicks a right-column item -> creates a match (leftId -> rightId), removes any existing match for that leftId AND any existing match pointing to that rightId (bidirectional dedup — research Pitfall 1)
3. Matched items show matching color indicators (use a palette of 5-6 colors for visual distinction, e.g., blue, emerald, purple, amber, rose, cyan). Each pair gets a color based on match index.
4. Clicking an already-matched left item allows reassignment (clears its current match, selects it)
5. Clicking an already-matched right item clears that match

**Rendering:**
- Two columns side by side (`flex gap-4`)
- Left column: items showing `pair.left` text with PhoneticText
- Right column: items showing `pair.right` text with PhoneticText
- Each item is a button: unmatched = `bg-zinc-800`, selected = `ring-2 ring-blue-500`, matched = colored left border/indicator matching its pair color
- Small colored dot or bar on the left side of left items and right side of right items shows the match color

**Submit button:**
- Disabled when not all pairs matched (matches.size < definition.pairs.length) OR disabled prop
- On click: convert Map to array of `{ leftId, rightId }` and call onSubmit
- Label: "Submit Matches"

**Instructional text:** "Click a left item, then a right item to match them" in `text-zinc-400`

```typescript
interface OrderingRendererProps {

}
```

**State management:**
1. `orderedItems` — array of item objects in the student's current order, initialized as a shuffled copy of `definition.items` using the deterministic `shuffleArray` (copy from ExercisePreview). Memoize initial shuffle.

**DnD implementation using @dnd-kit/react:**
Follow the pattern from the builder (Phase 32) but simpler — single column sortable list.

Use `DragDropProvider` from `@dnd-kit/react` wrapping the list, and `useSortable` from `@dnd-kit/react/sortable` on each item.

```tsx
import { DragDropProvider } from "@dnd-kit/react";
import { useSortable } from "@dnd-kit/react/sortable";
```

Each sortable item:
- Shows a `GripVertical` icon on the left (from lucide-react)
- Shows item text with PhoneticText
- Shows position number (1, 2, 3...) on the left
- Styled: `bg-zinc-800 rounded-lg px-4 py-3 flex items-center gap-3`
- Has `cursor-grab` when not disabled, `cursor-not-allowed` when disabled

On drag end (`onDragOver` event from DragDropProvider):
- Reorder the `orderedItems` array: move item from source.index to the new position
- Use the same reorder logic pattern as the builder canvas

**Submit button:**
- Always enabled (order is always defined) unless disabled prop
- On click: extract item IDs from orderedItems in current order, call `onSubmit({ orderedIds })`
- Label: "Submit Order"

**Instructional text:** "Drag items to arrange in the correct order" in `text-zinc-400`
