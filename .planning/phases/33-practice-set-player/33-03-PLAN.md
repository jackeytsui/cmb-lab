Create `src/components/practice/player/renderers/FillInBlankRenderer.tsx` as a "use client" component.

**Props interface:**
```typescript
interface FillInBlankRendererProps {
  definition: FillInBlankDefinition;
  language: "cantonese" | "mandarin" | "both";
  onSubmit: (response: { answers: string[] }) => void;
  disabled?: boolean;
}
```

**Component behavior:**
1. State: `answers: string[]` initialized to an array of empty strings matching `definition.blanks.length`
2. Parse the sentence using `parseBlankSentence(definition.sentence)` from `@/lib/practice` (memoize with useMemo)
3. Render the sentence with segments:
   - Text segments: wrapped in `<PhoneticText>` with forceLanguage
   - Blank segments: render an `<input>` element inline:
     - `type="text"`, `placeholder="..."`, `autoComplete="off"`
     - Value bound to `answers[seg.index]`
     - onChange updates the answers array at the correct index
     - Styled: `mx-1 inline-block min-w-[80px] max-w-[160px] border-b-2 border-zinc-500 bg-transparent text-center text-white focus:border-blue-500 focus:outline-none transition`
     - Disabled when `disabled` prop is true
4. "Submit Answer" button:
   - Disabled when any blank is empty (all answers must be non-empty after trim) OR disabled prop
   - On click: call `onSubmit({ answers })`
   - Same button styling as MCQ renderer
5. Instructional text "Fill in the blanks:" above the sentence in `text-zinc-400`

Match the visual pattern from `FillInBlankPreview` but replace the static "___" underline with actual input fields.
