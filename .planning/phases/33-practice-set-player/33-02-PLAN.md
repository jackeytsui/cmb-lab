<task type="auto">
  <name>Task 2: Create practice grade API route for AI exercises</name>
  <files>src/app/api/practice/grade/route.ts</files>
  <action>
Create `src/app/api/practice/grade/route.ts` with a single POST handler that delegates to the existing n8n grading webhooks.

This route handles BOTH free_text and audio_recording exercise types. It determines which based on the Content-Type header:
- `application/json` -> free_text grading
- `multipart/form-data` -> audio_recording grading

**Free text grading flow (JSON body):**
1. Auth check with `auth()`
2. Parse JSON body: `{ exerciseId, type: "free_text", studentResponse, definition }`
3. Rate limit check
4. Check `N8N_GRADING_WEBHOOK_URL` env var. If not configured, return mock GradeResult: `{ isCorrect: true, score: 85, feedback: "Mock: Configure N8N_GRADING_WEBHOOK_URL for real AI grading." }`
5. Load grading prompt from DB: `getPrompt("grading-text-prompt", DEFAULT_PROMPT)`
6. Build n8n payload: `{ interactionId: exerciseId, userId, studentResponse, expectedAnswer: definition.sampleAnswer || "", language: "both", prompt }`
   - NOTE: Reuse `interactionId` field name in the n8n payload even though this is a practice exercise. The n8n workflow uses this for logging, not LMS lookups. This avoids needing to modify the n8n workflow.
7. Call n8n webhook with 15s timeout (same pattern as `/api/grade`)
8. Parse response as `GradingResponse` and return it, mapping to `GradeResult` shape: `{ isCorrect, score, feedback: response.feedback, explanation: undefined }`

**Audio grading flow (FormData):**
1. Auth check
2. Parse FormData: `audio` (File), `exerciseId`, `type: "audio_recording"`, `targetPhrase`, `language`
3. Rate limit check
4. Check `N8N_AUDIO_GRADING_WEBHOOK_URL`. If not configured, return mock.
5. Build FormData for n8n: append audio file, interactionId (= exerciseId), userId, expectedAnswer (= targetPhrase), language, prompt
6. Call n8n webhook with 15s timeout (same pattern as `/api/grade-audio`)
7. Parse response as `AudioGradingResponse` and return as `GradeResult` shape

**Important differences from /api/grade and /api/grade-audio:**
- Does NOT require a real `interactionId` from the interactions table
- Does NOT capture to `submissions` table (practice attempts are stored separately via the attempts API)
- Uses `exerciseId` as the identifier passed to n8n

Content-Type detection: check if `request.headers.get("content-type")?.includes("multipart/form-data")` to branch between text and audio.
