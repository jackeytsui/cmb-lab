Create `src/components/practice/player/renderers/AudioRecordingRenderer.tsx` as a "use client" component.

**Props interface:**
```typescript
interface AudioRecordingRendererProps {
  definition: AudioRecordingDefinition;
  language: "cantonese" | "mandarin" | "both";
  onSubmit: (response: { audioBlob: Blob }) => void;
  disabled?: boolean;
}
```

**Component behavior using `useAudioRecorder` hook:**
1. Import and call `useAudioRecorder()` to get: `{ status, audioBlob, audioUrl, recordingTime, startRecording, stopRecording, reset, error }`
2. Display the target phrase prominently in the center, wrapped in `<PhoneticText>` with forceLanguage
3. If `definition.referenceText` exists, show it below as context in `text-zinc-400`

**Recording UI (state machine):**
- `idle`: Show "Record" button (red circle + Mic icon). `onClick={startRecording}`
- `recording`: Show "Stop" button with recording indicator (pulsing red dot) + elapsed time display (`recordingTime` seconds formatted as mm:ss). `onClick={stopRecording}`
- `stopped`: Show audio playback controls:
  - `<audio src={audioUrl} controls />` styled with dark theme
  - "Re-record" button (calls `reset()`) - secondary styling
  - "Submit Recording" button (calls `onSubmit({ audioBlob })`) - primary styling
- `error`: Show error message from `error` state in red, with "Try Again" button (calls `reset()`)

**Styling:**
- Center-aligned layout matching AudioRecordingPreview
- Record button: `rounded-full bg-red-600 hover:bg-red-700 px-6 py-3 text-white font-medium`
- Pulsing indicator: use Tailwind `animate-pulse` on a small red dot during recording
- Recording time: `text-zinc-400 font-mono`
- Audio element: standard HTML audio with dark background styling
- All buttons disabled when `disabled` prop is true

  <action>
Create `src/components/practice/player/renderers/FreeTextRenderer.tsx` as a "use client" component.

**Props interface:**
```typescript
interface FreeTextRendererProps {
  definition: FreeTextDefinition;
  language: "cantonese" | "mandarin" | "both";
  onSubmit: (response: { text: string }) => void;
  disabled?: boolean;
}
```

**Component behavior:**
1. State: `text: string` for the textarea value
2. Display the prompt text wrapped in `<PhoneticText>` with forceLanguage
3. Render a `<textarea>`:
   - Placeholder: "Type your answer here..."
   - Value bound to `text` state
   - `rows={5}`, resizable vertically
   - Styled: `w-full rounded-lg border border-zinc-600 bg-zinc-800 px-4 py-3 text-white placeholder:text-zinc-500 focus:border-blue-500 focus:ring-1 focus:ring-blue-500 focus:outline-none transition`
   - Disabled when `disabled` prop is true
4. Character count display below textarea:
   - Show current character count
   - If `definition.minLength` or `definition.maxLength` defined, show as "X / max" or "X (min Y)" or "X / Y-Z"
   - Color: green when within range, red when below min or above max
5. "Submit Answer" button:
   - Disabled when:
     - `text.trim()` is empty
     - `text.trim().length` < definition.minLength (if defined)
     - `text.trim().length` > definition.maxLength (if defined)
     - `disabled` prop is true
   - On click: call `onSubmit({ text: text.trim() })`
   - Same primary button styling

**Instructional text:** None needed beyond the prompt itself.
