**PracticePlayer.tsx** — main player shell ("use client"):

Props:
```typescript
interface PracticePlayerProps {
  practiceSet: PracticeSet;
  exercises: PracticeExercise[];
  userId: string; // internal DB user ID
}
```

**Layout structure:**
1. **Header bar** — practice set title + progress indicator
   - Title: `practiceSet.title` in `text-xl font-bold text-white`
   - Progress bar: shadcn `<Progress>` component showing `player.progress` (0-100)
   - Exercise counter: "Exercise {currentIndex + 1} of {total}" in `text-zinc-400`

2. **Main content area** — conditionally renders:
   - **Start screen** (status === "not_started"):
     - Practice set title and description
     - Exercise count and estimated time
     - "Start Practice" button (calls `player.start()` then creates attempt via API)
   - **Exercise view** (status === "in_progress"):
     - Language badge (teal/amber/zinc matching existing colors)
     - `<ExerciseRenderer>` for current exercise
     - `<PracticeFeedback>` shown below renderer after grading (when `hasResult` is true)
     - Navigation buttons at bottom
   - **Results screen** (status === "completed"):
     - `<PracticeResults>` component

3. **Navigation bar** (during exercise view):
   - "Previous" button (disabled on first exercise or while grading)
   - "Next" button (disabled on last exercise or while grading or if current exercise not yet answered)
   - On last exercise with result: "View Results" button instead of "Next"
   - Exercise dot indicators (small circles below the nav showing which exercises are answered/current)

**Attempt persistence:**
- On START: POST to `/api/practice/{setId}/attempts` with `{ totalExercises: exercises.length }`. Store returned `attemptId`.
- On COMPLETE (entering results screen): POST to `/api/practice/{setId}/attempts` with `{ attemptId, score, correctCount, results, completedAt }` to update the attempt record.
- On individual exercise retry: after re-grading, POST update to sync the results JSONB.

**Animation:** Use Framer Motion `AnimatePresence` + `motion.div` for exercise transitions (slide left/right on next/prev).

**PracticeResults.tsx** — results summary screen ("use client"):

Props:
```typescript
interface PracticeResultsProps {
  practiceSet: PracticeSet;
  exercises: PracticeExercise[];
  results: Record<string, GradeResult>;
  responses: Record<string, unknown>;
  startedAt: Date | null;
  completedAt: Date | null;
  onRetryAll: () => void;
  onRetryExercise: (exerciseId: string) => void;
}
```

**Layout:**
1. **Score card** (center, prominent):
   - Large score number: overall percentage `Math.round(correctCount/total * 100)`
   - Score ring or circle visualization (use CSS or simple fraction)
   - Color: green >= 80, yellow >= 60, red < 60
   - "X of Y correct"

2. **Time taken:**
   - Calculate from startedAt to completedAt using `differenceInSeconds` from date-fns
   - Format as "Xm Ys" using `formatDuration` or manual formatting

3. **Per-question breakdown** (scrollable list):
   - For each exercise, show:
     - Exercise number and type badge
     - Question text (truncated to ~80 chars)
     - Score badge: green check or red X + score
     - "Retry" button (only for incorrect exercises) — calls `onRetryExercise(exerciseId)`

4. **Action buttons at bottom:**
   - "Retry All" button (secondary) — calls `onRetryAll()`
   - "Done" button (primary) — navigates to `/dashboard` or back using `router.push()`

Use `date-fns` for time formatting. Use `lucide-react` icons for check/x indicators.

**Server component logic:**
1. Call `auth()` from Clerk. If no userId, redirect to `/sign-in`.
2. Get `setId` from `await params`.
3. Query practice set directly from DB (NOT self-fetch):
   ```typescript
   const practiceSet = await db.query.practiceSets.findFirst({
     where: and(
       eq(practiceSets.id, setId),
       eq(practiceSets.status, "published"),
       isNull(practiceSets.deletedAt)
     ),
   });
   ```
   If not found: call `notFound()` (returns 404 page).
4. Query exercises:
   ```typescript
   const exercises = await db
     .select()
     .from(practiceExercises)
     .where(and(
       eq(practiceExercises.practiceSetId, setId),
       isNull(practiceExercises.deletedAt)
     ))
     .orderBy(asc(practiceExercises.sortOrder));
   ```
5. If exercises is empty, show a message: "This practice set has no exercises yet."
6. Get internal user ID:
   ```typescript
   const dbUser = await db.query.users.findFirst({
     where: eq(users.clerkId, clerkId),
     columns: { id: true },
   });
   ```
7. Render `<PracticePlayer practiceSet={practiceSet} exercises={exercises} userId={dbUser?.id ?? ""} />`

**Page metadata:**
```typescript
export async function generateMetadata({ params }) {
  const { setId } = await params;
  const practiceSet = await db.query.practiceSets.findFirst({
    where: and(eq(practiceSets.id, setId), isNull(practiceSets.deletedAt)),
    columns: { title: true },
  });
  return { title: practiceSet?.title ? `${practiceSet.title} - Practice` : "Practice" };
}
```

**Important:** Follow the documented anti-pattern avoidance — query DB directly, don't self-fetch API routes. This is the same pattern as the builder page in Phase 32.
