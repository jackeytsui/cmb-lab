Create `src/app/api/admin/students/bulk/route.ts`:

**Auth**: Require coach+ role (same pattern as other admin routes).

**Request validation** using zod:
```typescript
const bulkSchema = z.object({
  operation: z.enum(["assign_course", "remove_course", "add_tag", "remove_tag"]),
  studentIds: z.array(z.string().uuid()).min(1).max(500),
  targetId: z.string().uuid(),
});
```

**Processing logic** (POST handler):
1. Parse and validate request body with bulkSchema
2. Get the current user's DB id (using getCurrentUser from @/lib/auth)
3. Process each studentId independently in a loop (NOT Promise.all to avoid overwhelming the DB):

For `assign_course`:
- Check if courseAccess already exists (userId + courseId)
- If exists, report success with note "Already enrolled"
- If not, insert into courseAccess with grantedBy: "coach"
- Use try/catch per student

For `remove_course`:
- Delete from courseAccess where userId + courseId
- If no row deleted, report error "Not enrolled"

For `add_tag`:
- Call `assignTag(studentId, targetId, currentUser.id)` from @/lib/tags
- This already handles ON CONFLICT DO NOTHING

For `remove_tag`:
- Call `removeTag(studentId, targetId)` from @/lib/tags

4. Build results array:
```typescript
results: { studentId: string; success: boolean; error?: string }[]
```

5. Log the operation to bulk_operations table:
```typescript
const succeededIds = results.filter(r => r.success).map(r => r.studentId);
const [op] = await db.insert(bulkOperations).values({
  operationType: operation,
  targetId,
  studentIds,
  succeededIds,
  performedBy: currentUser.id,
  expiresAt: new Date(Date.now() + 5 * 60 * 1000), // 5 minutes
}).returning();
```

6. Return response:
```json
{
  "operationId": "uuid",
  "results": [...],
  "summary": { "total": 10, "succeeded": 9, "failed": 1 }
}
```

**Error handling**: If zod validation fails, return 400 with error details. If auth fails, return 401/403. Wrap the entire handler in try/catch for 500s.

Create `src/app/api/admin/students/bulk/undo/route.ts`:

**Auth**: Require coach+ role.

**Request validation**:
```typescript
const undoSchema = z.object({
  operationId: z.string().uuid(),
});
```

**Undo logic** (POST handler):
1. Parse and validate request body
2. Get current user's DB id
3. Fetch the bulk operation record:
```typescript
const [op] = await db.select().from(bulkOperations)
  .where(and(
    eq(bulkOperations.id, operationId),
    eq(bulkOperations.performedBy, currentUser.id)
  ));
```
4. Validate:
   - If not found: return 404 "Operation not found"
   - If `op.undoneAt` is not null: return 409 "Operation already undone"
   - If `op.expiresAt < new Date()`: return 410 "Undo window expired (5 minutes)"

5. Reverse the operation using the `succeededIds` array (only undo what actually succeeded):
   - `assign_course` -> delete courseAccess for each succeededId + targetId
   - `remove_course` -> insert courseAccess for each succeededId + targetId (grantedBy: "coach")
   - `add_tag` -> call removeTag for each succeededId + targetId
   - `remove_tag` -> call assignTag for each succeededId + targetId

6. Mark the operation as undone:
```typescript
await db.update(bulkOperations)
  .set({ undoneAt: new Date() })
  .where(eq(bulkOperations.id, operationId));
```

7. Return:
```json
{
  "undone": true,
  "operationId": "uuid",
  "studentsAffected": 9
}
```

**Edge case**: If reversing individual students fails (e.g., course was already re-assigned by someone else), log the failure but still mark the operation as undone. Return partial results.
