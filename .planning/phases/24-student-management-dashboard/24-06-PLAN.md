**Filter Presets API** (`src/app/api/admin/students/filter-presets/route.ts`):

Auth: coach+ role on all handlers.

GET handler:
- Fetch all filter presets for the current user: `WHERE createdBy = currentUser.id ORDER BY isDefault DESC, name ASC`
- Return `{ presets: FilterPreset[] }`

POST handler:
- Body: `{ name: string, filters: object, isDefault?: boolean }`
- Validate with zod
- If isDefault is true, unset any existing default for this user first (`UPDATE filter_presets SET is_default = false WHERE created_by = userId AND is_default = true`)
- Insert into filter_presets
- Return 201 with the created preset

**Filter Preset Detail** (`src/app/api/admin/students/filter-presets/[presetId]/route.ts`):

PATCH handler:
- Body: `{ name?: string, filters?: object, isDefault?: boolean }`
- Validate preset belongs to current user
- Update only provided fields
- Handle isDefault toggle same as POST
- Return updated preset

DELETE handler:
- Validate preset belongs to current user
- Delete the preset
- Return 204

**CSV Export** (`src/app/api/admin/students/export/route.ts`):

GET handler:
- Auth: coach+ role
- Parse same filter params as the main students API (search, tagIds, courseId, atRisk, sortBy, sortOrder)
- Also parse `columns` param: comma-separated list of columns to include (default: "name,email,coursesEnrolled,completionPercent,lastActive,tags,createdAt")
- Call `getStudentsPageData` with pageSize set to 10000 and page 1 (fetch all matching, not paginated) - or implement a streaming variant if concerned about size
- Build CSV using `formatCsvRow` from `@/lib/analytics`:
  - Header row from selected columns
  - Data rows mapping each student to the selected columns
  - Tags formatted as comma-separated names within the cell
  - Dates formatted as ISO strings
- Return response with headers:
  ```
  Content-Type: text/csv
  Content-Disposition: attachment; filename="students-export-YYYY-MM-DD.csv"
  ```
- Use the current date in the filename
