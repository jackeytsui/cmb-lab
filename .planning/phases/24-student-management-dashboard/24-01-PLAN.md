Create `src/lib/student-queries.ts` with a `getStudentsPageData` function:

```typescript
interface StudentQueryParams {
  page: number;
  pageSize: number;
  sortBy: string; // "name" | "email" | "createdAt" | "lastActive" | "completionPercent"
  sortOrder: "asc" | "desc";
  search?: string;
  tagIds?: string[];
  courseId?: string;
  atRisk?: boolean; // no activity in last 7 days
}

interface StudentRow {
  id: string;
  clerkId: string;
  email: string;
  name: string | null;
  createdAt: string; // ISO
  coursesEnrolled: number;
  completionPercent: number;
  lastActive: string | null; // ISO
  tags: { id: string; name: string; color: string; type: "coach" | "system" }[];
}

interface StudentPageResult {
  students: StudentRow[];
  total: number;
}
```

Implementation details:
1. Build WHERE conditions: always `role = 'student'`, optionally add search (ILIKE on name/email), tagIds (subquery on student_tags), courseId (subquery on course_access), atRisk filter using: `AND users.id NOT IN (SELECT DISTINCT user_id FROM lesson_progress WHERE last_accessed_at > NOW() - INTERVAL '7 days')` — this identifies students with zero lesson activity in the past 7 days as at-risk
2. Execute COUNT and data queries in parallel via Promise.all
3. For the data query: select from users, ORDER BY the sortBy column (map "lastActive" and "completionPercent" to subquery columns), LIMIT/OFFSET for pagination
4. After fetching the page of students, batch-fetch in parallel: tags (same pattern as existing route.ts), course access counts (GROUP BY userId), lesson progress aggregates (completed count / total count per user, max lastAccessedAt)
5. Merge the enrichment data into each student row
6. For "lastActive" sorting: use a correlated subquery `(SELECT MAX(last_accessed_at) FROM lesson_progress WHERE user_id = users.id)` as the sort expression
7. For "completionPercent" sorting: this is trickier with server-side sort - compute in a subquery or sort after enrichment for the current page only (acceptable since we're sorting a page of 25-50 rows)

Export the function and the `StudentRow` / `StudentPageResult` types.

- Call `getStudentsPageData(params)` and return `{ students, total }`
- Keep the existing auth check (coach+ role)
- Preserve backward compatibility: if no page/pageSize params, use defaults that match the old behavior (limit=50 maps to pageSize=50)
