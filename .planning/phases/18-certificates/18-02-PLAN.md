  <files>

  <action>
    1. Create `src/app/api/certificates/generate/route.ts`:
       - POST handler
       - Requires authentication (use `auth()` from Clerk, return 401 if no userId)
       - Look up internal user by clerkId
       - Accept JSON body: `{ courseId: string }`
       - Validate courseId is provided (400 if missing)
       - Call `checkCourseCompletion(user.id, courseId)` -- return 400 with message "Course not completed" if false
       - Call `createCertificate(user.id, courseId)` -- returns certificate record (idempotent)
       - Return 200 with `{ certificate: { verificationId, completedAt } }`
       - Wrap in try/catch, return 500 on unexpected errors

    2. Create `src/app/api/certificates/[certificateId]/download/route.ts`:
       - GET handler -- the certificateId param is the verificationId
       - Call `getCertificateByVerificationId(params.certificateId)`
       - Return 404 if not found
       - Import `renderToBuffer` from `@react-pdf/renderer` (use: `import { renderToBuffer } from '@react-pdf/renderer'`)
       - Import `CertificateDocument` component
       - Call `renderToBuffer(<CertificateDocument studentName={cert.studentName} courseTitle={cert.courseTitle} completedAt={cert.completedAt} verificationId={cert.verificationId} />)`
       - Return new Response with the buffer, headers:
         - `Content-Type: application/pdf`
         - `Content-Disposition: attachment; filename="certificate-{verificationId}.pdf"`
       - NO authentication required -- anyone with the verification ID can download
       - Wrap in try/catch, return 500 on render errors

    IMPORTANT: The `renderToBuffer` import from `@react-pdf/renderer` works server-side in API routes. Do NOT use `renderToStream` (buffer is simpler for Response).
  </action>
  <verify>
    - `npm run build` passes with no type errors
    - Both route files export the correct HTTP method handlers (POST, GET)
  </verify>
