<task type="auto">
  <name>Task 2: Create XState video player machine</name>
  <files>
    src/machines/videoPlayerMachine.ts
  </files>
  <action>
    Create `src/machines/videoPlayerMachine.ts` with XState v5 state machine:

    **States:**
    - `idle`: Initial state, transitions to `playing` on PLAY
    - `playing`: Active playback, transitions to:
      - `paused` on PAUSE
      - `pausedForInteraction` on CUE_POINT_REACHED (with guard: not already completed)
    - `paused`: User-paused, transitions to `playing` on PLAY
    - `pausedForInteraction`: System-paused for interaction, ONLY transitions to `playing` on INTERACTION_COMPLETE

    **Context:**
    - currentTime, duration, volume, cuePoints[], activeCuePoint, savedVolume

    **Actions:**
    - `updateTime`: assigns currentTime from TIME_UPDATE event
    - `setActiveCuePoint`: assigns activeCuePoint from CUE_POINT_REACHED event
    - `clearActiveCuePoint`: clears activeCuePoint on INTERACTION_COMPLETE
    - `saveVolume`: saves current volume before fade
    - `restoreVolume`: restores volume after interaction complete

    **Guards:**
    - `cuePointNotCompleted`: check cuePoint.completed !== true before transitioning to pausedForInteraction

    Use XState v5 `createMachine` with `types` inference. Do NOT use v4 `Machine()` function.

    Pattern from RESEARCH.md Pattern 1 with modifications for volume saving.
  </action>
  <verify>
    - `npx tsc --noEmit src/machines/videoPlayerMachine.ts` passes
    - Machine imports from 'xstate', not '@xstate/core'
    - No v4 patterns (Machine, interpret)
  </verify>
  <done>XState machine with 4 states and proper transitions exported</done>

  <action>
    **Part A: Create `src/hooks/useInteractiveVideo.ts`**

    Hook that wraps XState machine and Mux Player ref:

    1. Accept `cuePoints: CuePoint[]` as parameter
    2. Use `useMachine` from @xstate/react with videoPlayerMachine
    3. Return:
       - `state`: current state value ("idle" | "playing" | etc.)
       - `context`: machine context
       - `playerRef`: React.RefObject<MuxPlayerRefAttributes> for Mux Player
       - `play()`: sends PLAY event AND calls playerRef.current?.play()
       - `pause()`: sends PAUSE event AND calls playerRef.current?.pause()
       - `updateTime(time)`: sends TIME_UPDATE event
       - `handleCuePointReached(cuePoint)`: fades volume (500ms), pauses player, sends CUE_POINT_REACHED
       - `completeInteraction()`: restores volume, sends INTERACTION_COMPLETE, calls play()
       - `isInteractionPending`: boolean derived from state === "pausedForInteraction"

    4. Include volume fade utility (from RESEARCH.md Pattern 3):
       - Gradually fade from current volume to 0 over 500ms
       - Save original volume before fading
       - Restore on completeInteraction

    **Part B: Create `src/components/video/InteractiveVideoPlayer.tsx`**

    Component using the hook:

    1. Props:
       - `playbackId: string`
       - `cuePoints: CuePoint[]`
       - `onInteractionRequired: (cuePoint: CuePoint) => void` - called when pausing for interaction
       - `onInteractionComplete?: () => void` - optional callback after interaction done
       - `title?: string`
       - `className?: string`

    2. Use MuxPlayer component with ref from hook
    3. On `durationchange` event: add cue points via `player.addCuePoints()` (from RESEARCH.md Pattern 2)
    4. On `cuepointchange` event: detect active cue point, call handleCuePointReached if not completed
    5. On `timeupdate` event: send TIME_UPDATE to machine
    6. Wire play/pause buttons to hook's play/pause methods
    7. Expose `completeInteraction` method for parent to call after interaction graded

    **Cue point markers on progress bar:**
    - Add visual markers using absolute positioning on a wrapper div
    - Each marker positioned at (timestamp / duration) * 100%
    - Green if completed, yellow if pending

    **Do NOT include overlay in this component** - that's plan 02-02.
