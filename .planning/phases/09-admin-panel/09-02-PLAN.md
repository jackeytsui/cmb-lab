**interactions/route.ts:**
- GET: List interactions for lessonId (query param), ordered by timestamp ASC
  - Return full interaction objects including id, lessonId, timestamp, type, language, prompt, expectedAnswer, correctThreshold, sortOrder
- POST: Create interaction with validation
  - Required: {lessonId, timestamp, type, language, prompt}
  - Optional: {expectedAnswer, correctThreshold (default 80), sortOrder}
  - Validate timestamp is positive integer
  - Validate type is "text" or "audio" (matches interactionTypeEnum)
  - Validate language is "cantonese", "mandarin", or "both" (matches interactionLanguageEnum)

**interactions/[interactionId]/route.ts:**
- GET: Single interaction by ID
- PUT: Update interaction fields (partial update)
  - Can update: timestamp, type, language, prompt, expectedAnswer, correctThreshold, sortOrder
  - Validate timestamp doesn't conflict with another interaction (within 2 seconds of existing)
- DELETE: Soft delete by setting deletedAt = new Date()

Use Drizzle queries with proper error handling.
Return 400 for validation errors with message field.
Return 404 if interaction not found.

Create components for the interaction editor interface.

**VideoPreviewPlayer.tsx:**
- Props: {playbackId: string, currentTime: number, onTimeUpdate: (time: number) => void, onTimeSelect: (time: number) => void, interactions: Interaction[]}
- Use @mux/mux-player-react like existing VideoPlayer.tsx
- Display current timestamp prominently (MM:SS format)
- On click/tap on player progress bar, call onTimeSelect with that timestamp
- Visual markers on progress bar for existing interactions (small dots or lines)
- Pause button to freeze at current time for precision
- Keyboard shortcuts: Space=play/pause, Left/Right=seek 1s, Shift+Left/Right=seek 5s
- No interaction overlay (this is admin preview mode)

**InteractionTimeline.tsx:**
- Props: {interactions: Interaction[], currentTime: number, onSelect: (interaction: Interaction) => void, onAddAtTime: (time: number) => void}
- Horizontal timeline bar spanning video duration
- Visual markers (circles) at each interaction timestamp
- Color code by type: cyan for text, purple for audio (matches SubmissionCard)
- Tooltip on hover showing interaction prompt preview
- Click marker to select interaction (calls onSelect)
- Click empty area to call onAddAtTime with that timestamp
- Current time indicator (vertical line) that moves with video

**InteractionForm.tsx:**
- Props: {lessonId: string, interaction?: Interaction, timestamp: number, onSave: (interaction: Interaction) => void, onCancel: () => void, onDelete?: () => void}
- **Zod schema validation:** Define schema matching API enum requirements:
  ```typescript
  const interactionSchema = z.object({
    timestamp: z.number().int().min(0, "Timestamp must be positive"),
    type: z.enum(["text", "audio"], { errorMap: () => ({ message: "Type must be text or audio" }) }),
    language: z.enum(["cantonese", "mandarin", "both"], { errorMap: () => ({ message: "Invalid language selection" }) }),
    prompt: z.string().min(5, "Prompt must be at least 5 characters"),
    expectedAnswer: z.string().optional(),
    correctThreshold: z.number().int().min(0).max(100).default(80),
  });
  ```
- Fields:
  - timestamp: number input, read from prop but editable (MM:SS format display)
  - type: select with options "text" / "audio"
  - language: select with options "Cantonese only" / "Mandarin only" / "Both languages"
  - prompt: textarea (required, min 5 chars) - what student sees
  - expectedAnswer: textarea (optional) - context for AI grading
  - correctThreshold: number input (0-100, default 80) - minimum score to pass
- If interaction prop exists, it's edit mode (PUT), else create mode (POST)
- **API error handling:** Display 400 error messages from API response in form (use toast or inline error display). Handle network errors gracefully.
- Delete button only in edit mode, with confirmation dialog
- Dark theme styling matching other admin forms
  </action>
