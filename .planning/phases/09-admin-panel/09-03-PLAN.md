Create API routes for viewing students and their progress. All routes require admin role.

**students/route.ts:**
- GET: List all students (users with role="student")
  - Query params: search (optional, matches name or email), limit (default 50), offset (default 0)
  - Return: {students: User[], total: number}
  - Include: id, clerkId, email, name, createdAt
  - Order by createdAt DESC (newest first)
  - Use ILIKE for case-insensitive search on name and email

**students/[studentId]/route.ts:**
- GET: Single student details
  - Return user info + summary stats:
    - coursesEnrolled: number (count of courseAccess records)
    - lessonsCompleted: number (count of lessonProgress with completedAt not null)
    - lastActive: timestamp (most recent lessonProgress.lastAccessedAt)

**students/[studentId]/progress/route.ts:**
- GET: Detailed progress for a student
  - Return array of courses with nested progress:
    ```typescript
    {
      courses: {
        id: string;
        title: string;
        progress: {
          lessonsTotal: number;
          lessonsCompleted: number;
          percentComplete: number;
        };
        modules: {
          id: string;
          title: string;
          lessons: {
            id: string;
            title: string;
            videoWatchedPercent: number;
            interactionsCompleted: number;
            interactionsTotal: number;
            completedAt: string | null;
          }[];
        }[];
      }[];
    }
    ```
  - Only include courses student has access to (via courseAccess)
  - Use Drizzle relations for efficient nested query

Use existing courseAccess, lessonProgress tables.

Create API route for viewing AI grading/feedback logs. Requires admin role.

**ai-logs/route.ts:**
- GET: List AI feedback events from multiple sources
  - Query params:
    - studentId (optional): filter by student
    - type (optional): "text" | "audio" | "all" (default "all")
    - startDate (optional): ISO date string
    - endDate (optional): ISO date string
    - limit (default 50)
    - offset (default 0)
  - Sources to aggregate:
    1. interactionAttempts table (score, feedback, isCorrect, response, createdAt)
    2. submissions table (score, aiFeedback, transcription for audio)
  - Return unified shape:
    ```typescript
    {
      logs: {
        id: string;
        type: "text" | "audio";
        studentId: string;
        studentName: string;
        studentEmail: string;
        lessonTitle: string;
        interactionPrompt: string;
        studentResponse: string;
        transcription?: string; // For audio
        score: number;
        aiFeedback: string;
        passed: boolean;
        createdAt: string;
        source: "attempt" | "submission";
      }[];
      total: number;
    }
    ```
  - Order by createdAt DESC
  - Use UNION query or merge in code to combine sources

Note: interactionAttempts captures all grading attempts during video watching.
submissions captures items sent to coach queue (failing text, all audio).
Both have AI feedback worth reviewing.
  </action>
