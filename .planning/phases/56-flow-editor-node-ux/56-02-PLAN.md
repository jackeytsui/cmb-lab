**FlowEditor.tsx -- Update connection logic for new LogicNode handle IDs:**

The FlowEditor's `onConnect` callback currently handles logic node connections by matching `sourceHandle` to rule IDs or 'fallback'. With the new True/False fixed outputs, the handle IDs are `'true-output'` and `'false-output'`. Update:

1. **Update edge creation in `edges` useMemo**: For logic nodes, instead of creating edges per-rule (matching `rule.id` as sourceHandle), create exactly two edges:
   - If `step.logicRules` has rules: the first matching rule's `nextStepId` maps to `'true-output'` handle
   - `step.fallbackStepId` maps to `'false-output'` handle

   Replace the logicRules edge creation block with:
   ```tsx
   if (step.logicRules && step.logicRules.length > 0 && isLogicStep(step)) {
     // True output: first rule's destination (all rules evaluate to the same "true" path)
     const trueDestination = step.logicRules[0]?.nextStepId;
     if (trueDestination) {
       newEdges.push({
         id: `e-${step.id}-true-${trueDestination}`,
         source: step.id,
         target: trueDestination,
         sourceHandle: 'true-output',
         type: 'deletable',
         animated: true,
         markerEnd: { type: MarkerType.ArrowClosed },
         style: { stroke: '#10b981', strokeWidth: 2 },
       });
     }
   }
   ```

   And update the fallback edge block to use 'false-output' for logic nodes:
   ```tsx
   if (step.fallbackStepId && isLogicStep(step)) {
     newEdges.push({
       id: `e-${step.id}-false-${step.fallbackStepId}`,
       source: step.id,
       target: step.fallbackStepId,
       sourceHandle: 'false-output',
       type: 'deletable',
       animated: true,
       markerEnd: { type: MarkerType.ArrowClosed },
       style: { stroke: '#f87171', strokeWidth: 2, strokeDasharray: '5,5' },
       label: '',
     });
   }
   ```

   Keep the non-logic fallback edge creation for video nodes (they use sourceHandle 'fallback').

2. **Update `onConnect` for logic nodes**: When a new connection is dragged from a logic node:
   ```tsx
   if (isLogicStep(sourceStep)) {
     if (sourceHandle === 'true-output') {
       // Set the first rule's nextStepId (or create one if empty)
       if (sourceStep.logicRules && sourceStep.logicRules.length > 0) {
         const newRules = sourceStep.logicRules.map((r, idx) =>
           idx === 0 ? { ...r, nextStepId: targetStepId } : r
         );
         onUpdateStep(sourceStepId, { logicRules: newRules });
       } else {
         // Create a default rule pointing to the target
         const defaultRule = { id: crypto.randomUUID(), field: '', operator: 'equals' as const, value: '', nextStepId: targetStepId };
         onUpdateStep(sourceStepId, { logicRules: [defaultRule] });
       }
     } else if (sourceHandle === 'false-output') {
       onUpdateStep(sourceStepId, { fallbackStepId: targetStepId });
     }
     return;
   }
   ```

3. **Update `onEdgesDelete` for logic nodes**: When edges from logic nodes are deleted:
   ```tsx
   if (isLogicStep(sourceStep)) {
     if (sourceHandle === 'true-output') {
       if (sourceStep.logicRules && sourceStep.logicRules.length > 0) {
         const newRules = sourceStep.logicRules.map((r, idx) =>
           idx === 0 ? { ...r, nextStepId: '' } : r
         );
         onUpdateStep(sourceStepId, { logicRules: newRules });
       }
     } else if (sourceHandle === 'false-output') {
       onUpdateStep(sourceStepId, { fallbackStepId: null } as any);
     }
     return;
   }
   ```

**IMPORTANT**: These FlowEditor changes to `onConnect` and `onEdgesDelete` replace the old logic-node handling that used per-rule sourceHandle matching. The old code that matched `sourceHandle` to `rule.id` or `'fallback'` must be replaced, not duplicated. The video node (non-logic) connection handling stays the same.

**Double-click behavior (FLOW-06)**: Already implemented in FlowEditor's `onNodeDoubleClick` -- it calls `onEditLogic(node.id)` for logicStep and `onEditStep(node.id)` for others. The builder page's `handleEditLogic` opens the logic modal, and `handleEditStep` switches to build view. This is correct and needs no changes. Single-click select/drag is React Flow's default behavior.
