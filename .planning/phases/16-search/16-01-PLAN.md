  <action>
1. Create `src/lib/search-utils.ts` with two exported functions:

   **sanitizeSearchQuery(query: string): string**
   - Trim the input
   - Escape `%` to `\%` and `_` to `\_` (same pattern as `/api/knowledge/search/route.ts`)
   - Return sanitized string

   **generateSearchFields(title: string, description?: string | null): { searchPinyin: string | null; searchJyutping: string | null }**
   - Combine title and description into a single text string (filter out null/undefined)
   - Extract only Chinese characters using regex: `/[\u4e00-\u9fff]/g`
   - If no Chinese characters found, return `{ searchPinyin: null, searchJyutping: null }`
   - Convert Chinese chars to Pinyin using `pinyin-pro`:
     ```typescript
     import { pinyin } from "pinyin-pro";
     const pinyinText = pinyin(chineseChars, { toneType: "none", type: "array" }).join(" ");
     ```
   - Convert Chinese chars to Jyutping using `to-jyutping`:
     ```typescript
     import ToJyutping from "to-jyutping";
     const jyutpingText = ToJyutping.getJyutpingText(chineseChars) ?? "";
     ```
   - Strip tone numbers from jyutping: `.replace(/[0-9]/g, "")`
   - Return both values lowercased
   - NOTE: `to-jyutping` is ESM-only. If import issues, use dynamic import or check if the default export works. The library exports a default object with `getJyutpingText` method.

2. Generate Drizzle migration:
   ```bash
   npm run db:generate
   ```
   This should create migration 0003 with ALTER TABLE statements adding search_pinyin and search_jyutping to both courses and lessons.

3. Run the migration:
   ```bash
   npm run db:migrate
   ```
  </action>
