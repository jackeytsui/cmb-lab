**src/app/(dashboard)/dashboard/reader/ReaderClient.tsx** — Main orchestrator (`"use client"`):
- State management:
  - `rawText: string` — the original imported/pasted text (unmodified source of truth)
  - `displayText: string` — text after T/S conversion applied (derived from rawText + scriptMode)
  - `segments: WordSegment[]` — segmented displayText for rendering
  - `importDialogOpen: boolean`
  - `isConverting: boolean` — loading state during T/S conversion (async operation)
- Import and use `useReaderPreferences()` for annotationMode, scriptMode, fontSize and their setters
- Import `segmentText` from `@/lib/segmenter`
- Import `convertScript` from `@/lib/chinese-convert`
- Import all reader components: `ImportDialog`, `ReaderToolbar`, `ReaderTextArea`

- **Text import flow:**
  - When `onImport(text)` fires from ImportDialog: set `rawText = text`, close dialog
  - Use `useEffect` watching `[rawText, scriptMode]` to derive displayText:
    - If scriptMode is 'original': `displayText = rawText`
    - If scriptMode is 'simplified' or 'traditional': call `convertScript(rawText, 'original', scriptMode)` (async), set isConverting true during conversion, set displayText when done
  - Use `useMemo` watching `[displayText]` to derive segments: `segmentText(displayText)`

- **T/S conversion flow:**
  - When user toggles scriptMode via toolbar: `setScriptMode(newMode)`
  - The useEffect above re-derives displayText from rawText + new scriptMode
  - Show a subtle loading indicator (via isConverting state) during first conversion (opencc-js lazy loads its dictionary)

- **Layout:**
  ```
  <div className="container mx-auto px-4 py-6 space-y-4">
    <h1 className="text-2xl font-bold text-white">Chinese Reader</h1>
    <ReaderToolbar
      annotationMode={annotationMode}
      scriptMode={scriptMode}
      fontSize={fontSize}
      onAnnotationModeChange={setAnnotationMode}
      onScriptModeChange={setScriptMode}
      onFontSizeChange={setFontSize}
      onImportClick={() => setImportDialogOpen(true)}
    />
    {isConverting && <div className="text-sm text-zinc-400">Converting...</div>}
    <ReaderTextArea
      segments={segments}
      annotationMode={annotationMode}
      fontSize={fontSize}
    />
    <ImportDialog
      open={importDialogOpen}
      onOpenChange={setImportDialogOpen}
      onImport={handleImport}
    />
  </div>
  ```

- **Edge cases:**
  - Empty rawText: ReaderTextArea shows empty state (handled in component)
  - Very long text (>20K chars already truncated by API): just render what we have
  - T/S conversion error: catch in useEffect, show toast or inline error, keep displayText as rawText
  - Segmentation runs synchronously and is fast (Intl.Segmenter is native) — no loading state needed
