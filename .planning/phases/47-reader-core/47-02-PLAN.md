**src/components/reader/WordSpan.tsx** — Individual annotated word span:
- Props: `{ text: string; index: number; isWordLike: boolean; annotationMode: "pinyin" | "jyutping" | "plain"; showSandhi?: boolean }`
- Import `html as pinyinHtml` from `pinyin-pro` for Mandarin annotations
- Import `ToJyutping` from `to-jyutping` for Cantonese annotations
- Import `applyThirdToneSandhi` from `@/lib/tone-sandhi` for sandhi-applied pinyin
- Rendering logic:
  - If `!isWordLike` (punctuation, whitespace): render plain `<span>` with no data attributes and no annotation
  - If `annotationMode === 'plain'`: render `<span data-word={text} data-index={index} className="cursor-pointer hover:bg-cyan-500/20 rounded px-0.5 transition-colors">{text}</span>`
  - If `annotationMode === 'pinyin'`:
    - When `showSandhi` is true (default true): use `applyThirdToneSandhi(text)` to get sandhi-applied pinyin syllables, then build `<ruby>` HTML manually per character: `<ruby><span>{char}</span><rp>(</rp><rt>{sandhiPinyin}</rt><rp>)</rp></ruby>`
    - When `showSandhi` is false: use `pinyinHtml(text)` from pinyin-pro for standard pinyin annotations
    - Wrap in `<span data-word={text} data-index={index}>` with hover styles
  - If `annotationMode === 'jyutping'`:
    - Use `ToJyutping.getJyutpingList(text)` to get per-character jyutping
    - Build `<ruby>` HTML: for each `[char, jp]` pair, if `jp` exists render `<ruby><span>{char}</span><rp>(</rp><rt>{jp}</rt><rp>)</rp></ruby>`, else render char plain
    - Wrap in `<span data-word={text} data-index={index}>` with hover styles
- Use `dangerouslySetInnerHTML` for the ruby HTML output (pinyin-pro html() returns raw HTML strings)
- OR: build the ruby JSX directly (preferred for type safety) — construct React elements instead of HTML strings. Choosing direct JSX construction is strongly preferred: map characters to `<ruby>` React elements.
- Memoize with `React.memo` since this component renders thousands of times
- Add `rt` CSS: `text-align: center; font-size: 0.6em; color: rgb(161 161 170)` (zinc-400) via className on the rt element or a parent class

**src/components/reader/ReaderTextArea.tsx** — Segmented text display with event delegation:
- Props: `{ segments: WordSegment[]; annotationMode: "pinyin" | "jyutping" | "plain"; fontSize: number; onWordHover?: (word: string, index: number, element: HTMLElement) => void; onWordClick?: (word: string, index: number, element: HTMLElement) => void }`
- Import `WordSegment` from `@/lib/segmenter`
- Import `WordSpan` from `./WordSpan`
- Event delegation pattern on the container div (ONE handler, NOT per-span):
  - `onMouseOver`: find closest `[data-word]` ancestor, extract `data-word` and `data-index`, call `onWordHover` if the word changed from last hovered
  - `onClick`: find closest `[data-word]` ancestor, extract attributes, call `onWordClick`
  - `onMouseLeave` on container: clear hover state
- Track last hovered word index in a ref to avoid re-triggering for same word
- Container styling: `leading-relaxed` (or `leading-loose` when annotations shown for ruby space), dynamic `fontSize` via style prop, `text-zinc-100`, overflow-y-auto, max-height with scrolling
- When `annotationMode !== 'plain'`, add extra line-height (e.g., `leading-[2.5]` or `leading-[3]`) to accommodate ruby annotations above characters
- If segments is empty, show an empty state: centered text "Paste or import Chinese text to begin reading" with FileText icon, muted styling
- Render each segment as `<WordSpan key={index} ... />`
