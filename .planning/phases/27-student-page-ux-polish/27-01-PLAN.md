  <action>
Wrap DB queries in try/catch blocks and render inline ErrorAlert when queries fail, providing page-specific error messages. The error.tsx boundaries remain as a safety net, but these inline catches give better UX.

**Dashboard page.tsx:**
- Add import: `import { ErrorAlert } from "@/components/ui/error-alert";`
- Wrap the two DB query blocks (userCourses query and certificates query) in a single try/catch
- The auth() and currentUser() calls stay OUTSIDE the try/catch (auth failures should redirect, not show error)
- In the catch block: `console.error("Dashboard data failed to load:", error);` then return the same page shell (min-h-screen, AppHeader, container) but with `<ErrorAlert variant="block" message="Unable to load your courses. Please try refreshing the page." />` instead of the course grid
- The greeting section (displayName) should still render in the error case since it comes from Clerk, not from the failing DB query

**Course detail page.tsx:**
- Add import: `import { ErrorAlert } from "@/components/ui/error-alert";`
- The auth check (steps 1-3: auth, user lookup, access check) stays OUTSIDE try/catch - those redirect on failure
- Wrap steps 4-6 (course fetch, progress fetch, unlock computation) in try/catch
- In the catch block: `console.error("Course detail failed to load:", error);` then return page shell with back link to /dashboard and `<ErrorAlert variant="block" message="Unable to load course details. Please try refreshing the page." />`

**Lesson player page.tsx:**
- Add import: `import { ErrorAlert } from "@/components/ui/error-alert";`
- The auth check (steps 1-5: auth, user lookup, lesson fetch, access check, unlock check) already handles failures via redirect/notFound - keep those as-is
- Wrap step 6 (interactions fetch) and step 7 (cue points mapping) in try/catch
- In the catch block: `console.error("Failed to load lesson interactions:", error);` then render the full page but with cuePoints as an empty array `[]` and an inline ErrorAlert above the video player: `<ErrorAlert message="Some lesson interactions could not be loaded. The video will play without interactive checkpoints." className="mb-4" />`
- This is a graceful degradation: the video still plays, just without interactions. The error is shown inline (not block variant) since the page is still partially functional.
  </action>
