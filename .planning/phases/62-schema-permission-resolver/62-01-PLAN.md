Create `src/db/schema/roles.ts` with four tables, their relations, and inferred types. Follow the `tags.ts` pattern exactly (same import style, same table structure conventions).

**Tables to create:**

1. **roles** (`pgTable("roles")`):
   - `id`: uuid, defaultRandom, primaryKey
   - `name`: text, notNull, unique (TEXT not pgEnum per Decision 6)
   - `description`: text, nullable
   - `color`: text, notNull, default "#6b7280"
   - `allCourses`: boolean("all_courses"), notNull, default false
   - `isDefault`: boolean("is_default"), notNull, default false
   - `sortOrder`: integer("sort_order"), notNull, default 0
   - `createdBy`: uuid("created_by"), references users.id (nullable, no cascade)
   - `createdAt`: timestamp, notNull, defaultNow
   - `updatedAt`: timestamp, notNull, defaultNow, $onUpdate(() => new Date())
   - `deletedAt`: timestamp, nullable (soft delete)

2. **roleCourses** (`pgTable("role_courses")`):
   - `id`: uuid, defaultRandom, primaryKey
   - `roleId`: uuid("role_id"), notNull, references roles.id with onDelete cascade
   - `courseId`: uuid("course_id"), notNull, references courses.id with onDelete cascade
   - `accessTier`: accessTierEnum("access_tier"), notNull, default "full" -- IMPORT from `./access`, do NOT recreate
   - `createdAt`: timestamp, notNull, defaultNow
   - Indexes: uniqueIndex on (roleId, courseId), index on roleId

3. **roleFeatures** (`pgTable("role_features")`):
   - `id`: uuid, defaultRandom, primaryKey
   - `roleId`: uuid("role_id"), notNull, references roles.id with onDelete cascade
   - `featureKey`: text("feature_key"), notNull -- TEXT column, NOT pgEnum (Decision 6). Validated by Zod at API boundary in Plan 02.
   - `createdAt`: timestamp, notNull, defaultNow
   - Indexes: uniqueIndex on (roleId, featureKey), index on roleId

4. **userRoles** (`pgTable("user_roles")`):
   - `id`: uuid, defaultRandom, primaryKey
   - `userId`: uuid("user_id"), notNull, references users.id with onDelete cascade
   - `roleId`: uuid("role_id"), notNull, references roles.id with onDelete cascade
   - `assignedBy`: uuid("assigned_by"), references users.id (nullable, no cascade)
   - `expiresAt`: timestamp("expires_at"), nullable (null = permanent)
   - `createdAt`: timestamp, notNull, defaultNow
   - Indexes: uniqueIndex on (userId, roleId), index on userId, index on roleId

**Relations to define:**

- `rolesRelations`: many(roleCourses), many(roleFeatures), many(userRoles), one(users) for createdBy
- `roleCoursesRelations`: one(roles), one(courses)
- `roleFeaturesRelations`: one(roles)
- `userRolesRelations`: one(users) with relationName "userRoleUser", one(users) with relationName "userRoleAssigner" for assignedBy, one(roles)
  -- CRITICAL: Two FKs to users table requires relationName disambiguation. Follow the `studentTags` pattern exactly.

**Types to export:**

- `Role`, `NewRole` (inferred from roles table)
- `RoleCourse`, `NewRoleCourse`
- `RoleFeature`, `NewRoleFeature`
- `UserRole`, `NewUserRole`

**Imports:**
