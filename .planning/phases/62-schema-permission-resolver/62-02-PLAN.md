**2. PermissionSet Interface:**

```typescript
export interface PermissionSet {
  courseIds: Set<string>;
  features: Set<string>;
  accessTiers: Map<string, "preview" | "full">;
  hasWildcardAccess: boolean;
  canAccessCourse(courseId: string): boolean;
  canUseFeature(featureKey: string): boolean;
  getAccessTier(courseId: string): "preview" | "full" | null;
}
```

- `canAccessCourse()` is SYNC: returns `hasWildcardAccess || courseIds.has(id)`
- `canUseFeature()` is SYNC: returns `features.has(key)`
- `getAccessTier()` is SYNC: returns `hasWildcardAccess ? "full" : accessTiers.get(id) ?? null`

**3. Internal _resolvePermissions(userId) Function:**

Algorithm (3 parallel DB queries):

Step 1 - Load active role assignments:
```sql
SELECT userRoles.roleId, roles.allCourses
FROM user_roles
INNER JOIN roles ON userRoles.roleId = roles.id
WHERE userRoles.userId = :userId
  AND roles.deletedAt IS NULL
  AND (userRoles.expiresAt IS NULL OR userRoles.expiresAt > NOW())
```
Use the EXACT expiration pattern from the codebase: `or(isNull(userRoles.expiresAt), gt(userRoles.expiresAt, new Date()))`.
Extract roleIds array and check if any role has allCourses=true.

Step 2 - Three parallel queries (Promise.all):
- If roleIds is non-empty AND no wildcard access: query `roleCourses` WHERE roleId IN roleIds, select courseId + accessTier
- If roleIds is non-empty: query `roleFeatures` WHERE roleId IN roleIds, select featureKey
- Always: query `courseAccess` WHERE userId = userId AND (expiresAt IS NULL OR expiresAt > NOW()), select courseId + accessTier

If roleIds is empty, skip the role-based queries (return empty arrays).
If hasWildcardAccess is true, skip roleCourses query (not needed).

Step 3 - Build PermissionSet (additive union):
- Initialize empty `Set<string>` for courseIds and features
- Initialize empty `Map<string, "preview" | "full">` for accessTiers
- Iterate roleCourseRows: add courseId to set, set accessTier in map (upgrade to "full" if any source grants full)
- Iterate roleFeatureRows: add featureKey to set
- Iterate directGrants (courseAccess): add courseId to set, upgrade accessTier if "full"
- Access tier merge rule: if current is null or "preview" and new is "full", upgrade to "full". Never downgrade.
- Return object with the sets/maps and the three sync methods

**4. Exported cache()-wrapped resolver:**

```typescript
import { cache } from "react";
export const resolvePermissions = cache(_resolvePermissions);
```

CRITICAL: Import `cache` from "react" (NOT from a custom utility). This provides per-request deduplication in React 19 server components. The cache key is the userId string argument, compared by `Object.is()`.

Do NOT use a module-level Map or global variable (Pitfall 3: would cache across requests in serverless).

**5. Async Helper Functions (standalone, not class methods):**

`canAccessModule(permissions: PermissionSet, moduleId: string): Promise<boolean>`:
- Query `db.query.modules.findFirst({ where: eq(modules.id, moduleId), columns: { courseId: true } })`
- If not found, return false
- Return `permissions.canAccessCourse(mod.courseId)`

`canAccessLesson(permissions: PermissionSet, lessonId: string): Promise<boolean>`:
- Query `db.query.lessons.findFirst({ where: eq(lessons.id, lessonId), with: { module: { columns: { courseId: true } } } })`
- If not found, return false
- Return `permissions.canAccessCourse(lesson.module.courseId)`

These are ASYNC (they hit the database). They are standalone functions, NOT methods on PermissionSet (Pitfall 6). This keeps PermissionSet as a plain data object.
