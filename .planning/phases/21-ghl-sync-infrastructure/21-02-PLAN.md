Create `src/lib/ghl/contacts.ts` -- Contact linking service:

Functions to implement:
1. `findOrLinkContact(userId: string, email: string): Promise<{ ghlContactId: string; isNewLink: boolean }>`:
   - First checks ghl_contacts table for existing mapping by userId
   - If found and syncStatus is "active", returns existing ghlContactId
   - If not found, calls GHL API: `GET /contacts/search/duplicate?locationId={locationId}&email={email}` via ghlClient
   - If GHL returns a contact, stores mapping in ghl_contacts table (upsert on userId)
   - If GHL returns no contact, throws descriptive error "No GHL contact found for email {email}"
   - Logs a sync event via sync-logger (direction: "outbound", eventType: "contact.linked" or "contact.lookup")
   - Returns the ghlContactId and whether this was a new link

2. `getGhlContactId(userId: string): Promise<string | null>`:
   - Simple lookup from ghl_contacts table by userId
   - Returns ghlContactId or null if not linked

3. `unlinkContact(userId: string): Promise<void>`:
   - Sets syncStatus to "disconnected" on ghl_contacts row (soft unlink)
   - Logs sync event

4. `getContactMapping(userId: string): Promise<GhlContact | null>`:
   - Returns full ghl_contacts row for a user

5. `updateGhlContactEmail(userId: string, newEmail: string): Promise<void>`:
   - Looks up ghlContactId from ghl_contacts table by userId (use getGhlContactId)
   - If no mapping exists (null), return silently -- user is not linked to GHL, nothing to sync
   - If mapping exists, call GHL API: `PUT /contacts/{ghlContactId}` with body `{ email: newEmail, locationId }` via ghlClient
   - Update the ghl_contacts row: set lastSyncedAt to new Date()
   - Log a sync event: eventType "contact.email_updated", direction "outbound", entityType "contact", entityId userId, ghlContactId, payload { newEmail }
   - Wrap in try/catch: if the GHL API call fails, log a failed sync event with the error message but do NOT throw -- email sync failure should not break the Clerk webhook

IMPORTANT: Use `eq` from drizzle-orm for where clauses. Import db from "@/db". Import schema types from "@/db/schema".

Create `src/lib/ghl/sync-logger.ts` -- Sync event logging service:

Functions to implement:
1. `logSyncEvent(event: { eventType: string; direction: "inbound" | "outbound"; entityType: string; entityId?: string; ghlContactId?: string; payload?: Record<string, unknown>; status?: "pending" | "processing" | "completed" | "failed" }): Promise<string>`:
   - Inserts row into sync_events table
   - Returns the event ID
   - Default status is "completed" for simple log entries, "pending" for operations that need processing

2. `markEventCompleted(eventId: string): Promise<void>`:
   - Updates sync_events row: status = "completed", processedAt = new Date()

3. `markEventFailed(eventId: string, errorMessage: string): Promise<void>`:
   - Updates sync_events row: status = "failed", errorMessage, retryCount increment

4. `getRecentSyncEvents(options: { limit?: number; direction?: "inbound" | "outbound"; status?: string }): Promise<SyncEvent[]>`:
   - Queries sync_events with optional filters, ordered by createdAt DESC
   - Default limit: 50

5. `getSyncEventStats(): Promise<{ total: number; pending: number; failed: number; completed: number }>`:
   - Aggregation query for dashboard display
