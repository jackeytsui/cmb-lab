1. **ghl_contacts** table (contact linking -- maps LMS users to GHL contacts):
   - `id`: uuid PK defaultRandom
   - `userId`: uuid NOT NULL, unique, references users.id with onDelete cascade
   - `ghlContactId`: text NOT NULL, unique (the persistent GHL contact ID)
   - `ghlLocationId`: text NOT NULL (GHL sub-account ID)

   - `syncStatus`: text NOT NULL default "active" (values: "active", "error", "disconnected")

   - `updatedAt`: timestamp NOT NULL defaultNow with $onUpdate

2. **sync_events** table (audit log and processing queue):
   - `id`: uuid PK defaultRandom
   - `eventType`: text NOT NULL (e.g., "contact.linked", "tag.added", "field.synced", "contact.updated")
   - `direction`: syncDirectionEnum NOT NULL (values: "inbound", "outbound")
   - `status`: syncStatusEnum NOT NULL default "pending" (values: "pending", "processing", "completed", "failed")

   - `retryCount`: integer NOT NULL default 0

Create `src/lib/ghl/client.ts` -- a typed GHL API client class:

The client uses Private Integration Token (env var GHL_API_TOKEN) and GHL_LOCATION_ID. No OAuth complexity needed.

```typescript
// Key design decisions:
// - Uses Upstash Ratelimit (already in project) for burst limiting, NOT p-queue (avoid new dependency)
// - All GHL API calls MUST go through this client
// - Base URL: https://services.leadconnectorhq.com
// - Required headers: Authorization: Bearer {token}, Version: 2021-07-28, Content-Type: application/json
```

Implementation:
- Create a `GhlClient` class with methods: `get(path)`, `post(path, body)`, `put(path, body)`, `delete(path)`
- Each method calls a private `request()` method that:
  1. Checks Upstash rate limiter (ghlBurstLimiter: 80 requests per 10 seconds sliding window)
  2. If rate limited, waits and retries (exponential backoff, max 3 retries)
  3. Adds Authorization header with PIT token from process.env.GHL_API_TOKEN
  4. Adds Version header "2021-07-28"
  5. Tracks X-RateLimit-Remaining and X-RateLimit-Daily-Remaining from response headers
  6. Logs warnings when burst remaining < 20 or daily remaining < 10000
  7. On 429 response, reads Retry-After header, waits, and retries
  8. On 401, throws descriptive error (token may be invalid)
  9. Returns typed Response object
- Export a singleton `ghlClient` instance
- Export a `getGhlLocationId()` helper that reads from process.env.GHL_LOCATION_ID

Add to `src/lib/rate-limit.ts`:
```typescript
// GHL outbound API: 80/10s (leaves 20% headroom from 100/10s burst limit)
export const ghlBurstLimiter = new Ratelimit({
  redis,
  limiter: Ratelimit.slidingWindow(80, "10 s"),
  prefix: "ratelimit:ghl:burst",
  analytics: true,
});
```

Create `src/lib/ghl/echo-detection.ts`:
- Uses Redis.fromEnv() (same Upstash Redis already in project)
- `markOutboundChange(contactId: string, changeType: string, changeValue: string): Promise<void>` -- sets Redis key `ghl:echo:{contactId}:{changeType}:{changeValue}` with 60s TTL
- `isEchoWebhook(contactId: string, changeType: string, changeValue: string): Promise<boolean>` -- checks if key exists, if so deletes it and returns true (this was our own change)
- Export both functions

IMPORTANT: Do NOT add `@gohighlevel/api-client` as a dependency. The official SDK is not needed since we are using PIT with simple REST calls. The typed client wrapper we build is sufficient and avoids adding a heavy dependency. Use native `fetch()` inside the client.
