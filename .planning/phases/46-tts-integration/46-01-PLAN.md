<task type="auto">
  <name>Task 1: Create TTS core library and rate limiters</name>
  <files>src/lib/tts.ts, src/lib/rate-limit.ts</files>
  <action>
Create `src/lib/tts.ts` with the following exports:

1. **`escapeXml(text: string): string`** -- Escape 5 XML special characters (`&`, `<`, `>`, `"`, `'`). Order matters: escape `&` first to avoid double-escaping.

2. **`resolveVoice(language: string): { voiceName: string; lang: string }`** -- Map language to voice:
   - `"zh-CN"` or `"mandarin"` -> `{ voiceName: "zh-CN-XiaoxiaoNeural", lang: "zh-CN" }`
   - `"zh-HK"` or `"cantonese"` -> `{ voiceName: "zh-HK-HiuMaanNeural", lang: "zh-HK" }`
   - Default to Mandarin for any other value.
   - Type: `type TTSLanguage = "zh-CN" | "zh-HK" | "mandarin" | "cantonese";`
   - Type: `type TTSRate = "x-slow" | "slow" | "medium" | "fast";`

3. **`buildSSML(text: string, voiceName: string, lang: string, rate: string): string`** -- Build SSML string with `<speak>` root, `<voice>` element, and `<prosody rate="...">` wrapper around escaped text. Use SSML version 1.0 and namespace `http://www.w3.org/2001/10/synthesis`. Rate defaults to `"medium"` if not provided.

4. **`buildPhonemeSSML(text: string, phoneme: string, voiceName: string, lang: string, rate: string): string`** -- Same as `buildSSML` but wraps text in `<phoneme alphabet="sapi" ph="...">` inside the prosody tag. The phoneme parameter accepts pinyin tone numbers (Mandarin) or jyutping (Cantonese).

5. **`buildCacheKey(text: string, language: string, voice: string, rate: string): string`** -- Build deterministic cache key: `tts:${language}:${voice}:${rate}:${md5(text)}`. Use Node.js `crypto.createHash("md5")`.

6. **`getCacheTTL(textLength: number): number`** -- Return TTL in seconds:
   - 1-2 chars: 604800 (7 days)
   - 3-6 chars: 259200 (3 days)
   - 7+ chars: 86400 (24 hours)

7. **`synthesizeSpeech(ssml: string): Promise<Buffer>`** -- Call Azure TTS REST API:
   - Endpoint: `https://${region}.tts.speech.microsoft.com/cognitiveservices/v1`
   - Headers: `Ocp-Apim-Subscription-Key`, `Content-Type: application/ssml+xml`, `X-Microsoft-OutputFormat: audio-24khz-48kbitrate-mono-mp3`, `User-Agent: CantoMandoBlueprint`
   - Read `AZURE_SPEECH_KEY` and `AZURE_SPEECH_REGION` from `process.env`
   - Throw `"Azure Speech credentials not configured"` if missing
   - Use AbortController with 5-second timeout (NOT 20s like pronunciation -- TTS is faster)
   - On timeout, throw `"TTS request timed out"`
   - On non-ok response, log error text and throw `"Azure TTS error: ${status}"`
   - Return `Buffer.from(await response.arrayBuffer())`
   - Follow the exact error handling pattern from `src/lib/pronunciation.ts`

8. **Export types:** `TTSLanguage`, `TTSRate`, and an interface `TTSRequest` with fields: `text: string`, `language: TTSLanguage`, `rate?: TTSRate`, `phoneme?: string`.

Then add TTS rate limiters to `src/lib/rate-limit.ts`:

- Add `ttsLimiter`: sliding window 30/min, prefix `"ratelimit:tts"`
- Add `ttsLimiterElevated`: sliding window 90/min, prefix `"ratelimit:tts:elevated"`
- Export both alongside existing limiters
- Use the same `redis` instance already created at the top of the file (do NOT create a new `Redis.fromEnv()`)

<task type="auto">
  <name>Task 2: Create POST /api/tts endpoint with cache-aside pattern</name>
  <files>src/app/api/tts/route.ts</files>
  <action>
Create `src/app/api/tts/route.ts` as a POST handler following the exact pattern from `src/app/api/grade-audio/route.ts` for auth, rate limiting, and error handling.

**Request body** (JSON):
```typescript
{ text: string; language: "zh-CN" | "zh-HK" | "mandarin" | "cantonese"; rate?: "x-slow" | "slow" | "medium" | "fast"; phoneme?: string; }
```

**Flow:**

1. **Auth**: `await auth()` from `@clerk/nextjs/server`. Return 401 if no userId.

2. **Rate limit**: Extract role from `sessionClaims?.metadata`, use `selectLimiter(role, ttsLimiter, ttsLimiterElevated)`. Return 429 via `rateLimitResponse(rl)` if over limit.

3. **Parse and validate**: `await request.json()`. Validate:
   - `text` is a non-empty string, max 500 characters. Return 400 with `"Text is required (max 500 characters)"` if invalid.
   - `language` is one of the 4 accepted values. Default to `"zh-CN"` if missing.
   - `rate` is one of the 4 accepted values. Default to `"medium"` if missing or invalid.

4. **Resolve voice**: Call `resolveVoice(language)` from `@/lib/tts`.

5. **Build cache key**: Call `buildCacheKey(text, voice.lang, voice.voiceName, rate)`.

6. **Check Redis cache**: `const cached = await redis.get<string>(cacheKey)`. If hit, decode from base64 and return as `new NextResponse(Buffer.from(cached, "base64"))` with headers:
   - `Content-Type: audio/mpeg`
   - `Cache-Control: public, max-age=86400`
   - `X-Cache: HIT`

7. **Build SSML**: If `phoneme` parameter provided, use `buildPhonemeSSML(text, phoneme, voice.voiceName, voice.lang, rate)`. Otherwise use `buildSSML(text, voice.voiceName, voice.lang, rate)`.

8. **Call Azure**: `const audioBuffer = await synthesizeSpeech(ssml)`.

9. **Cache in Redis**: `const ttl = getCacheTTL(text.length)`. Store as `audioBuffer.toString("base64")` with `{ ex: ttl }`.

10. **Return**: `new NextResponse(audioBuffer)` with headers:
    - `Content-Type: audio/mpeg`
    - `Cache-Control: public, max-age=86400`
    - `X-Cache: MISS`

**Error handling** (wrap steps 3-9 in try/catch):
- `"Azure Speech credentials not configured"` -> 503 with `{ error: "TTS service not configured" }`
- `"TTS request timed out"` -> 504 with `{ error: "TTS request timed out" }`
- `"Azure TTS error:"` prefix -> 502 with `{ error: "TTS service unavailable" }`
- Any other error -> 500 with `{ error: "Failed to generate speech" }`
- Log all errors with `console.error("TTS API error:", error)`

Import `Redis` from `@upstash/redis` and create a module-level `const redis = Redis.fromEnv()`.
