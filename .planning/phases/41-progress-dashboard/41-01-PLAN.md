Create `src/lib/progress-dashboard.ts` with 5 exported async functions. All use direct DB queries via Drizzle (NO self-fetch API routes). Import from `@/db`, `@/db/schema`, `drizzle-orm`, `date-fns`, `@date-fns/tz`.

**1. `getXPTimeline(userId: string, timezone: string, range: "daily" | "weekly" | "monthly")`**
Returns `{ date: string; xp: number }[]`.
- Query `dailyActivity` for the user within the date range (30 days for daily, 90 for weekly, 365 for monthly).
- Use `TZDate.tz(timezone)` for `now`, then `subDays(now, daysBack)` for start.
- For daily: return one data point per day, fill gaps with 0 using `eachDayOfInterval`.
- For weekly: group by ISO week using `startOfWeek` with `weekStartsOn: 1`, sum XP per week.
- For monthly: group by month using `startOfMonth`, sum XP per month.
- Format dates as `"yyyy-MM-dd"` for daily, `"MMM d"` for weekly (start of week), `"MMM yyyy"` for monthly.

**2. `getActivityHeatmap(userId: string)`**
Returns `{ date: string; count: number; level: 0|1|2|3|4 }[]`.
- Query past 365 days from `dailyActivity`.
- Build a Map of `activityDate -> totalXp`.
- Generate contiguous 365-day array using `eachDayOfInterval`.
- For each day, lookup totalXp from map (default 0).
- Compute level using thresholds: 0 xp = 0, <25 = 1, <75 = 2, <150 = 3, >=150 = 4.
- Return contiguous array (react-activity-calendar requires this).

**3. `getMasteryData(userId: string)`**
Returns `{ courseId: string; courseTitle: string; modules: { moduleId: string; moduleTitle: string; totalLessons: number; completedLessons: number }[]; totalLessons: number; completedLessons: number }[]`.
- Single SQL query using JOINs: `courseAccess` -> `courses` -> `modules` -> `lessons` LEFT JOIN `lessonProgress`.
- Filter: `courseAccess.userId = userId`, `courses.deletedAt IS NULL`, `modules.deletedAt IS NULL`, `lessons.deletedAt IS NULL`.
- GROUP BY course + module to get per-module total and completed lesson counts.
- Use `sql<number>` template for COUNT(DISTINCT lessons.id) and COUNT(DISTINCT CASE WHEN lessonProgress.completedAt IS NOT NULL THEN lessonProgress.lessonId END).
- Reshape flat rows into nested course -> modules structure in TypeScript after query.
- Compute course-level totals by summing module totals.

**4. `getBadgeStats(userId: string)`**
Returns `UserStats` (from `@/lib/badges`).
- Run parallel queries using `Promise.all`:
  - Total XP: `SUM(dailyActivity.totalXp)` for the user
  - Longest streak: `users.longestStreak`
  - Current streak: compute from recent `dailyActivity` rows (walk backwards from today checking consecutive days; stop at first gap unless streakFreezeUsed)
  - Lessons completed: `COUNT(lessonProgress)` WHERE `completedAt IS NOT NULL`
  - Practice sets completed: `COUNT(DISTINCT practiceAttempts)` WHERE `completedAt IS NOT NULL`
  - Practice sets perfect: `COUNT(DISTINCT practiceAttempts)` WHERE `score >= 95`
  - Conversation count: `COUNT(conversations)` (or 0 if table not relevant — use daily_activity conversationCount SUM)
  - Days active: `COUNT(dailyActivity)` for the user
- Assemble into UserStats object.

**5. `getWeeklySummary(userId: string, timezone: string)`**
Returns `{ thisWeek: WeekStats; lastWeek: WeekStats; dayOfWeek: number }` where `WeekStats = { totalXp: number; lessonsCompleted: number; practiceCompleted: number; daysActive: number; goalsMet: number }`.
- Calculate week boundaries using user's timezone:
  - This week: `startOfWeek(now, { weekStartsOn: 1 })` to today
  - Last week: 7 days before this week start to day before this week start
- Use a shared `aggregateWeek(userId, start, end)` helper that queries dailyActivity with SUM/COUNT.
- Return `dayOfWeek` (1=Monday to 7=Sunday) so the UI can compute goal hit rate correctly (goalsMet / dayOfWeek for current week, goalsMet / 7 for last week).
- Guard against division by zero in the data layer: if all values are 0, return 0s cleanly.
