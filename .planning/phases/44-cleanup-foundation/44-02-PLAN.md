Output: Two new schema files, updated barrel export, and a generated migration SQL file ready for Phase 45 to seed data into.
</objective>

  <name>Task 1: Create dictionary schema (dictionary_entries + character_data)</name>
  <files>src/db/schema/dictionary.ts</files>
  <action>
    Create `src/db/schema/dictionary.ts` following the exact pattern from `src/db/schema/xp.ts` (imports, section comments, tables, relations, type inference).

    **Enum:**
    ```typescript
    export const dictionarySourceEnum = pgEnum("dictionary_source", ["cedict", "canto", "both"]);
    ```

    **Table 1: dictionary_entries**
    Columns:
    - `id`: uuid, defaultRandom, primaryKey (codebase convention — NOT serial)
    - `traditional`: varchar("traditional", { length: 50 }).notNull()
    - `simplified`: varchar("simplified", { length: 50 }).notNull()
    - `pinyin`: varchar("pinyin", { length: 200 }).notNull() — numbered tones from CEDICT ("zhong1 guo2")
    - `pinyinDisplay`: varchar("pinyin_display", { length: 200 }) — nullable, tone marks version
    - `jyutping`: varchar("jyutping", { length: 200 }) — nullable (not all entries have Cantonese)
    - `definitions`: text("definitions").array().notNull() — PostgreSQL text[] with default `sql\`'{}'::text[]\`` (use `import { sql } from "drizzle-orm"`)
    - `source`: dictionarySourceEnum("source").notNull().default("cedict")
    - `isSingleChar`: boolean("is_single_char").notNull().default(false) — populated during seeding, NOT a generated column
    - `frequencyRank`: integer("frequency_rank") — nullable
    - `createdAt`: timestamp("created_at").notNull().defaultNow()

    Indexes (3rd arg callback):
    - `index("dictionary_entries_traditional_idx").on(table.traditional)`
    - `index("dictionary_entries_simplified_idx").on(table.simplified)`
    - `index("dictionary_entries_pinyin_idx").on(table.pinyin)`

    **Table 2: character_data**
    Uses the character itself as natural primary key (NOT uuid). This is the ONE exception to the uuid convention because each character IS its identity and lookups are always by character.

    Columns:
    - `character`: varchar("character", { length: 1 }).primaryKey() — the Chinese character itself
    - `pinyin`: text("pinyin").array() — nullable, array of readings from Make Me a Hanzi
    - `jyutping`: text("jyutping").array() — nullable, from CC-Canto / to-jyutping
    - `radical`: varchar("radical", { length: 4 }) — nullable
    - `radicalMeaning`: varchar("radical_meaning", { length: 50 }) — nullable
    - `strokeCount`: smallint("stroke_count") — nullable (import `smallint` from drizzle-orm/pg-core)
    - `decomposition`: varchar("decomposition", { length: 100 }) — IDS string like "⿰亻尔"
    - `etymologyType`: varchar("etymology_type", { length: 20 }) — pictographic/ideographic/pictophonetic
    - `etymologyHint`: text("etymology_hint") — learner-facing hint
    - `etymologyPhonetic`: varchar("etymology_phonetic", { length: 10 }) — phonetic component
    - `etymologySemantic`: varchar("etymology_semantic", { length: 10 }) — semantic component
    - `definition`: text("definition") — brief learner-friendly definition
    - `frequencyRank`: integer("frequency_rank") — nullable
    - `strokePaths`: jsonb("stroke_paths") — SVG path data from Make Me a Hanzi graphics.txt
    - `strokeMedians`: jsonb("stroke_medians") — animation median points
    - `createdAt`: timestamp("created_at").notNull().defaultNow()

    Indexes:
    - `index("character_data_radical_idx").on(table.radical)`
    - `index("character_data_stroke_count_idx").on(table.strokeCount)`
    - `index("character_data_frequency_idx").on(table.frequencyRank)`

    **Relations:** dictionary_entries has no FK relations (standalone lookup table). character_data has no FK relations (standalone lookup table). Define empty relations exports if desired for consistency, but they are optional for tables without FKs.

    **Type inference (bottom of file):**
    ```typescript
    export type DictionaryEntry = typeof dictionaryEntries.$inferSelect;
    export type NewDictionaryEntry = typeof dictionaryEntries.$inferInsert;
    export type CharacterData = typeof characterData.$inferSelect;
    export type NewCharacterData = typeof characterData.$inferInsert;
    export type DictionarySource = (typeof dictionarySourceEnum.enumValues)[number];
    ```

    Follow exact formatting from xp.ts: section comment separators (`// ============`), blank lines between sections, no trailing semicolons on type exports.
  </action>
  <verify>Run `npx tsc --noEmit src/db/schema/dictionary.ts` or ensure no TypeScript errors are reported when checking the file. Verify the file exports dictionaryEntries, characterData, DictionaryEntry, CharacterData, DictionarySource types.</verify>
  <done>src/db/schema/dictionary.ts exists with dictionaryEntries (uuid PK, varchar traditional/simplified/pinyin, text[] definitions, source enum, 3 indexes) and characterData (varchar(1) natural PK, radical/stroke/etymology fields, jsonb stroke data, 3 indexes) tables plus type inference exports.</done>

    **Part A: Create `src/db/schema/vocabulary.ts`**

    Follow the exact pattern from `src/db/schema/xp.ts` for a user-linked table.

    **Table: saved_vocabulary**
    Columns:
    - `id`: uuid("id").defaultRandom().primaryKey()
    - `userId`: uuid("user_id").notNull().references(() => users.id, { onDelete: "cascade" })
    - `traditional`: varchar("traditional", { length: 50 }).notNull()
    - `simplified`: varchar("simplified", { length: 50 }).notNull()
    - `pinyin`: varchar("pinyin", { length: 200 }) — nullable
    - `jyutping`: varchar("jyutping", { length: 200 }) — nullable
    - `definitions`: text("definitions").array().notNull() — same pattern as dictionary_entries
    - `notes`: text("notes") — user's personal notes, nullable
    - `createdAt`: timestamp("created_at").notNull().defaultNow()

    Indexes:
    - `index("saved_vocabulary_user_id_idx").on(table.userId)` — FK index (codebase convention)
    - `index("saved_vocabulary_user_traditional_idx").on(table.userId, table.traditional)` — composite for "has user saved this word?" queries

    **Relations:**
    ```typescript
    export const savedVocabularyRelations = relations(savedVocabulary, ({ one }) => ({
      user: one(users, {
        fields: [savedVocabulary.userId],
        references: [users.id],
      }),
    }));
    ```

    **Type inference:**
    ```typescript
    export type SavedVocabulary = typeof savedVocabulary.$inferSelect;
    export type NewSavedVocabulary = typeof savedVocabulary.$inferInsert;
    ```

    Import `users` from `"./users"` and `relations` from `"drizzle-orm"`. Import `sql` from `"drizzle-orm"` if using the text array default.

    Place these after the existing `// XP System` block.
