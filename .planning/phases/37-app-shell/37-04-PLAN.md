    The current pattern is:
    ```tsx
    <div className="min-h-screen bg-zinc-900 text-white">
      <AppHeader title="..." />
    ```

    ```tsx
    <div className="container mx-auto px-4 py-8">
      {/* content */}
    </div>
    ```

    **Step 3: Handle error states that also have min-h-screen wrappers.**
    Some pages have try/catch where the catch block also renders `<div className="min-h-screen ..."><AppHeader /><ErrorAlert /></div>`. Apply the same migration to the error rendering path.

    **Step 4: Handle loading.tsx files.**
    Loading skeletons currently render `<div className="min-h-screen bg-zinc-900 text-white"><AppHeader title="..." /><div className="container">skeleton</div></div>`. Remove the outer wrapper and AppHeader. The loading skeleton should just be the container with skeleton content.

    - `lessons/[lessonId]/page.tsx`: The lesson player may have a full-width layout (no container). Keep whatever inner layout it has, just remove the min-h-screen + AppHeader wrapper.
    - `practice/[setId]/page.tsx`: Same — may be full-width. Remove wrapper, keep inner layout.
    - `coach/submissions/[submissionId]/page.tsx`: Has complex layout. Just remove the outermost min-h-screen div and AppHeader.

    **IMPORTANT:** Do NOT change the inner content, styling, or logic of any page. Only remove the outer shell (min-h-screen div + AppHeader). The sidebar layout provides these now.
