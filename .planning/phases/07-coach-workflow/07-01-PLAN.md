Create `src/db/schema/submissions.ts`:
- `submissions` table:
  - id: uuid primary key
  - userId: uuid FK to users.id (cascade delete)
  - interactionId: uuid FK to interactions.id (cascade delete)
  - lessonId: uuid FK to lessons.id (cascade delete) - denormalized for easy querying
  - type: pgEnum "submission_type" ["text", "audio"]
  - response: text - the student's text response (for text submissions)
  - audioData: text - base64-encoded audio data (for audio submissions, nullable) - stored inline for v1 simplicity
  - score: integer - AI grading score 0-100
  - aiFeedback: text - AI feedback text
  - transcription: text (nullable) - AI transcription for audio submissions
  - status: pgEnum "submission_status" ["pending_review", "reviewed", "archived"]
  - reviewedAt: timestamp (nullable) - when coach reviewed
  - reviewedBy: uuid FK to users.id (nullable) - which coach reviewed
  - createdAt, updatedAt timestamps

- `coachFeedback` table:
  - id: uuid primary key
  - submissionId: uuid FK to submissions.id (cascade delete, unique - one feedback per submission)
  - coachId: uuid FK to users.id (cascade delete)
  - loomUrl: text (nullable) - Loom video link
  - feedbackText: text (nullable) - written feedback
  - createdAt, updatedAt timestamps

Add relations for submissions (belongs to user, interaction, lesson) and coachFeedback (belongs to submission, coach).

Create `src/db/schema/notes.ts`:
- `coachNotes` table:
  - id: uuid primary key
  - coachId: uuid FK to users.id (cascade delete)
  - studentId: uuid FK to users.id (cascade delete)
  - submissionId: uuid FK to submissions.id (nullable, cascade delete) - optional link to specific submission
  - visibility: pgEnum "note_visibility" ["internal", "shared"] - internal=coach only, shared=student can see
  - content: text
  - createdAt, updatedAt timestamps

Add relations for coachNotes (belongs to coach user, student user, optionally submission).
