  <action>
Create `src/app/api/students/[studentId]/access/route.ts`:

GET /api/students/[studentId]/access - List student's course access
- Require coach or admin role
- Get studentId from params
- Validate student exists and has role "student"
- Return all courseAccess records for student
- Join courses for course titles and descriptions
- Return shape:
  ```ts
  {
    access: Array<{
      courseId: string;
      courseTitle: string;
      accessTier: "preview" | "full";
      grantedBy: "enrollment" | "admin" | "coach";
      expiresAt: string | null;
      createdAt: string;
    }>
  }
  ```

POST /api/students/[studentId]/access - Grant course access
- Require coach or admin role
- Validate body: { courseId: string, accessTier?: "preview" | "full", expiresAt?: string }
- Default accessTier to "full" if not specified
- Check student exists and has role "student"
- Check course exists
- Check access doesn't already exist (return 409 Conflict if it does)
- Create courseAccess record with grantedBy: "coach"
- Return created access record with 201 status

DELETE /api/students/[studentId]/access - Revoke course access
- Require coach or admin role
- Query param: courseId (required)
- Validate courseId is provided
- Delete courseAccess record where userId and courseId match
- Return 204 on success
- Return 404 if access not found

Error handling:
- 400 for invalid request body or missing params
- 401 for unauthenticated
- 403 for non-coach users
- 404 for student/course not found
- 409 for access already exists (POST)
- 500 for server errors
