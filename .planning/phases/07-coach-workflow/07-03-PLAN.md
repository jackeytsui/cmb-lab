  <action>
Create `src/app/api/submissions/[submissionId]/feedback/route.ts`:

POST /api/submissions/[submissionId]/feedback - Save coach feedback
- Require coach role minimum
- Validate request body: { loomUrl?: string, feedbackText?: string }
- At least one of loomUrl or feedbackText required
- Validate Loom URL format if provided (must contain loom.com or share.loom.com)
- Upsert coachFeedback record (one feedback per submission)
- Update submission status to "reviewed" and set reviewedAt, reviewedBy
- Trigger email notification by calling /api/notify/coach-feedback internally
- Return created/updated feedback

GET /api/submissions/[submissionId]/feedback - Get existing feedback
- Require coach role minimum
- Return feedback if exists, 404 if not

Create `src/app/api/submissions/[submissionId]/notes/route.ts`:

POST /api/submissions/[submissionId]/notes - Add note to submission
- Require coach role minimum
- Validate: { content: string, visibility: "internal" | "shared" }
- Create coachNotes record linked to submission
- Return created note

GET /api/submissions/[submissionId]/notes - List notes for submission
- Require coach role minimum
- Return all notes (internal + shared) for this submission
- Include coach name for each note

DELETE /api/submissions/[submissionId]/notes?noteId=xxx - Delete a note
- Require coach role minimum
- Only allow deleting own notes (coachId matches current user)
- Return 204 on success

Create `src/app/api/notify/coach-feedback/route.ts`:

POST /api/notify/coach-feedback - Trigger email notification
- Internal API (called by feedback route, not exposed to client)
- Accept: { studentEmail, studentName, lessonTitle, coachName, loomUrl?, feedbackText? }
- Call N8N_COACH_FEEDBACK_WEBHOOK_URL if configured
- If webhook not configured, log warning and return success (dev mode)
- n8n workflow should send email to student with coach feedback notification

Follow existing webhook pattern from /api/grade/route.ts:
- 15 second timeout with AbortController
- Auth header if N8N_WEBHOOK_AUTH_HEADER configured
- Proper error handling for timeout and failures
