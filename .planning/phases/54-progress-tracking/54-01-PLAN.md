**1. Create `src/hooks/useWatchProgress.ts`:**

A custom hook that periodically saves watch progress. Accepts:
```typescript
interface UseWatchProgressOpts {
  sessionId: string | null;
  currentTimeMs: number;
  videoDurationMs: number | null;
  isPlaying: boolean;
  videoTitle?: string | null;
}
```

Implementation:
- Use `useDebouncedCallback` from `use-debounce` (already installed) with 10_000ms interval, `{ leading: false, trailing: true }`
- The debounced callback calls `fetch("/api/video/progress", { method: "POST", ... })` with sessionId, lastPositionMs, videoDurationMs, and title (only on first call)
- `useEffect` triggers the debounced save when `currentTimeMs` changes AND `isPlaying` is true AND `sessionId` is non-null AND `videoDurationMs` is non-null and > 0
- Use a `currentTimeRef` (useRef) that stays in sync with `currentTimeMs` prop for use in the visibilitychange handler (avoids stale closure)
- Store a `titleSentRef` (useRef<boolean>) to only send title once
- Page-leave handler: `useEffect` that adds `visibilitychange` listener. On `document.visibilityState === "hidden"`, call `saveProgress.flush()` first, then call `navigator.sendBeacon("/api/video/progress", new Blob([JSON.stringify({ sessionId, lastPositionMs: currentTimeRef.current, videoDurationMs })], { type: "application/json" }))` as a backup
- Cleanup: remove event listener on unmount, cancel debounced callback

**2. Wire into ListeningClient:**

Also get title via `player.getVideoData()?.title` and call `setVideoTitle(title)`. Use a ref (`durationCapturedRef`) to only do this once.

- Call `useWatchProgress({ sessionId, currentTimeMs, videoDurationMs, isPlaying: true when playing, videoTitle })` -- for `isPlaying`, derive from whether the player is playing. The simplest approach: add an `isPlaying` state boolean that is set to `true` in handlePlay and `false` in handlePause/handleEnd.
