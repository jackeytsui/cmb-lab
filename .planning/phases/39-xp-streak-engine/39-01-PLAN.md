Create the XP event ledger and daily activity summary database tables with proper indexes, enums, constraints, and relations.

Purpose: These two tables form the backbone of the entire XP/streak engine. The xp_events table is an append-only ledger recording every XP-earning action. The daily_activity table is a denormalized summary providing O(1) reads for dashboard display. A longestStreak column on the users table persists the all-time record.

Output: Drizzle schema file with two tables, a pgEnum, relations, types, and a generated migration.

  <action>
Create `src/db/schema/xp.ts` with:

1. **xpSourceEnum** — pgEnum "xp_source" with values: "lesson_complete", "practice_exercise", "practice_perfect", "voice_conversation", "daily_goal_met"

2. **xpEvents table** — append-only XP ledger:
   - id: uuid, defaultRandom, primaryKey
   - userId: uuid, notNull, references users.id with onDelete cascade
   - source: xpSourceEnum, notNull
   - amount: integer, notNull
   - entityId: uuid (nullable — null for daily_goal_met)
   - entityType: text (nullable — "lesson", "practice_set", "conversation", null)
   - createdAt: timestamp, notNull, defaultNow
   - Indexes: (userId), (userId, createdAt)

3. **dailyActivity table** — one row per user per day:
   - id: uuid, defaultRandom, primaryKey
   - userId: uuid, notNull, references users.id with onDelete cascade
   - activityDate: date, notNull (YYYY-MM-DD in user's timezone)
   - totalXp: integer, notNull, default 0
   - lessonCount: integer, notNull, default 0
   - practiceCount: integer, notNull, default 0
   - conversationCount: integer, notNull, default 0
   - goalXp: integer, notNull (snapshot of user's dailyGoalXp at time of first activity that day)
   - goalMet: boolean, notNull, default false
   - streakFreezeUsed: boolean, notNull, default false
   - createdAt: timestamp, notNull, defaultNow
   - updatedAt: timestamp, notNull, defaultNow, $onUpdate(() => new Date())
   - Constraints: unique on (userId, activityDate)
   - Indexes: (userId), (userId, activityDate)

4. **Relations** — xpEventsRelations (one user), dailyActivityRelations (one user)

5. **Type exports** — XPEvent, NewXPEvent, DailyActivity, NewDailyActivity, XPSource
