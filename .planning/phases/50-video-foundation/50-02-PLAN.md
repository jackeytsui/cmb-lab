  <action>
Create `src/lib/captions.ts` with two exported functions and supporting types:

**Types:**
```typescript
export interface NormalizedCaption {
  text: string;
  startMs: number;
  endMs: number;
  sequence: number;
}
```

**Function 1: `extractChineseCaptions(videoId: string)`**
Returns `Promise<{ captions: NormalizedCaption[]; lang: string } | null>`.

Implementation:
- Import `YoutubeTranscript` from `"youtube-transcript"`
- Define `CHINESE_LANG_CODES = ["zh", "zh-Hans", "zh-Hant", "zh-CN", "zh-TW"]`
- Loop through each lang code, call `YoutubeTranscript.fetchTranscript(videoId, { lang })`
- On success: map transcript items to `NormalizedCaption` format:
  - `text`: item.text (trim whitespace)
  - `startMs`: Math.round(item.offset) — the offset is in milliseconds
  - `endMs`: Math.round(item.offset + item.duration)
  - `sequence`: idx + 1
- Return `{ captions, lang }`
- On error: check if error message contains "language" or "available" (indicates wrong lang code) — continue to next code. For any other error, throw it.
- If all codes exhausted, return `null`

**Function 2: `parseCaptionFile(buffer: Buffer, fileName: string)`**
Returns `NormalizedCaption[]`.

Implementation:
- Import `parse` from `"@plussub/srt-vtt-parser"`
- Import `detect` from `"jschardet"`
- Define `ENCODING_MAP` matching the pattern in `src/app/api/reader/import/route.ts`:
  ```
  GB2312 -> gbk, GBK -> gbk, GB18030 -> gb18030, Big5 -> big5,
  UTF-8 -> utf-8, ASCII -> utf-8, windows-1252 -> utf-8,
  EUC-TW -> utf-8, HZ-GB-2312 -> utf-8, ISO-2022-CN -> utf-8
  ```
- Define `CJK_REGEX = /[\u4e00-\u9fff]/`
- First try: decode as UTF-8 with `new TextDecoder("utf-8").decode(buffer)`
- If CJK characters are NOT found in the decoded text, use `detect(buffer)` for encoding detection. If `detection.encoding` exists and `detection.confidence >= 0.5`, map via ENCODING_MAP and try that encoding. Fall back to UTF-8 on failure.
- Call `parse(text)` to get `{ entries }`
- Map entries to `NormalizedCaption[]`:
  - `text`: strip HTML tags with `.replace(/<[^>]*>/g, "").trim()`
  - `startMs`: entry.from
  - `endMs`: entry.to
  - `sequence`: idx + 1
- Filter out entries with empty text after stripping

Export the `ENCODING_MAP` as well (for potential reuse).

**1. Create `src/app/api/video/extract-captions/route.ts`:**

POST handler that:
1. Authenticates via `auth()` from `@clerk/nextjs/server` — return 401 if not authenticated
2. Parses JSON body: `{ videoId: string, url: string }` — validate `videoId` is an 11-char alphanumeric string
3. Gets internal user from DB: `db.query.users.findFirst({ where: eq(users.clerkId, clerkUserId) })` — return 401 if not found
4. Checks for existing session: `db.query.videoSessions.findFirst({ where: and(eq(videoSessions.userId, user.id), eq(videoSessions.youtubeVideoId, videoId)) })`
   - If exists AND has captions (captionCount > 0): fetch existing captions from videoCaptions table ordered by sequence asc, return `{ session, captions, cached: true }`
   - If exists but no captions: proceed to extraction
5. Calls `extractChineseCaptions(videoId)` from `@/lib/captions`
6. If result is null: return `{ session: null, captions: null, error: "no_chinese_captions" }` with status 200 (not an error — just no captions available)
7. If result has captions:
   - Create or update videoSession via upsert (use Drizzle's `db.insert().values().onConflictDoUpdate()` on the unique constraint):
     - Set `captionSource: "youtube_auto"`, `captionLang: result.lang`, `captionCount: result.captions.length`
   - Bulk insert captions into videoCaptions table: map `result.captions` to rows with `videoSessionId`, `sequence`, `startMs`, `endMs`, `text`
   - Return `{ session, captions: result.captions, cached: false }`
8. Wrap in try/catch. On error, log and return 500 with `{ error: "Failed to extract captions" }`

**2. Create `src/app/api/video/upload-captions/route.ts`:**

POST handler that:
1. Authenticates via `auth()` — return 401 if not authenticated
2. Parses multipart form data: expects `file` (File) and `videoSessionId` (string)
3. Gets internal user from DB — return 401 if not found
4. Validates videoSessionId: fetch the videoSession, verify it belongs to the authenticated user — return 403 if not owned
5. Validates file: must be present, file name must end in `.srt` or `.vtt` (case-insensitive), max size 2MB
6. Reads file into Buffer: `Buffer.from(await file.arrayBuffer())`
7. Calls `parseCaptionFile(buffer, file.name)` from `@/lib/captions`
8. If no captions parsed (empty array): return 400 with `{ error: "No captions found in file" }`
9. Determines source type from file extension: `.srt` -> `"upload_srt"`, `.vtt` -> `"upload_vtt"`
10. Deletes any existing captions for this videoSession: `db.delete(videoCaptions).where(eq(videoCaptions.videoSessionId, videoSessionId))`
11. Bulk inserts new captions
12. Updates videoSession: set `captionSource`, `captionCount`, `captionLang: null` (uploaded files don't have a lang code)
13. Returns `{ session: updatedSession, captions: parsedCaptions }`
14. Wrap in try/catch with 500 error response

Both routes should import from:
- `@clerk/nextjs/server` for auth
- `@/db` for db
- `@/db/schema` for table references
- `drizzle-orm` for query helpers (eq, and, asc)
- `@/lib/captions` for extraction/parsing utilities
- `next/server` for NextRequest, NextResponse
