<task type="auto">
  <name>Task 2: Refactor VideoThreadPlayer with text input, API integration, and session management</name>
  <files>src/components/video-thread/VideoThreadPlayer.tsx</files>
  <action>
Refactor the existing VideoThreadPlayer.tsx to add text response support, integrate with the respond API, and manage sessions.

**State changes:**
- Add `sessionId: string | null` to the reducer state (initialize as null)
- Add `isSubmitting: boolean` to state (prevent double-submissions)
- Add a new action type: `SET_SESSION_ID` with payload string
- Add a new action type: `SET_SUBMITTING` with payload boolean
- Keep existing INIT_THREAD, SET_CURRENT_STEP, RECORD_RESPONSE, SET_STATUS, SET_ERROR actions

**Text input UI:**
- When `currentStep.responseType === "text"`, render:
  - A `<textarea>` (or `<input type="text">`) with placeholder "Type your response..."
  - A Submit button that calls handleResponse with the typed text
  - Use local `useState` for the text input value; clear it after submission
  - Style: dark semi-transparent background matching the overlay aesthetic, white text, rounded corners

**Button/MC response UI (existing, polish):**
- Keep existing button rendering for `responseType === "button"` or `"multiple_choice"`
- Style buttons as pill-shaped with hover effects (white/semi-transparent on dark overlay)

**API integration -- replace handleResponse:**
- Replace the existing synchronous `handleResponse` function with a new async version. The old function does local-only state updates (dispatch calls); the new one calls the API first, then updates local state based on the API response. No wrapper function -- just rewrite the function body.
- Make `handleResponse` async
- Set isSubmitting to true at start, false when done (via SET_SUBMITTING dispatch)
- POST to `/api/video-threads/${thread.id}/respond` with body:
  ```json
  { "stepId": currentStep.id, "sessionId": state.sessionId, "response": { "type": responseType, "content": value } }
  ```
- On success response `{ nextStepId, sessionId, completed }`:
  - Store sessionId in state (SET_SESSION_ID) if not already set
  - Record the response locally (RECORD_RESPONSE)
  - If `completed` is true, SET_STATUS to "completed"
  - If `nextStepId` is not null, SET_CURRENT_STEP to nextStepId
  - If `nextStepId` is null and not completed, SET_STATUS to "completed" (fallback)
- On error: SET_ERROR with error message, show toast or inline error

**Video autoplay:**
- When currentStep changes (via SET_CURRENT_STEP), the Mux player should autoplay the new step's video
- The existing VideoPlayer component accepts `autoPlay` prop -- ensure it's set to true
- Add `key={currentStep.id}` on the VideoPlayer to force remount on step change (ensures autoplay triggers)

**Prompt text overlay:**
- Show `currentStep.promptText` as a semi-transparent overlay at the bottom of the video
- Only show the overlay text AFTER the video starts playing (or immediately if no video)
- Style: white text on dark gradient, large readable font

**Completion screen:**
- When status is "completed", show a centered card with:
  - Checkmark icon
  - "Thread Complete!" heading
  - "You've finished all steps in this thread." subtitle
  - A "Back to Dashboard" link to `/dashboard`

**Disable controls during submission:**
- All buttons and text input disabled when `isSubmitting` is true
- Show a small loading indicator on the submit button

**Error handling:**
- If API call fails, show error inline (not just console.log)
- Allow retry (don't lock the UI on error)
