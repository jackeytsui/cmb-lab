**Create PermissionTree component at `src/components/admin/PermissionTree.tsx`:**

2. "use client" component. Props interface:
   ```typescript
   interface CourseNode {
     id: string;
     title: string;
     modules: ModuleNode[];
   }
   interface ModuleNode {
     id: string;
     title: string;
     lessons: LessonNode[];
   }
   interface LessonNode {
     id: string;
     title: string;
   }
   type CheckState = "checked" | "indeterminate" | "unchecked";
   interface PermissionTreeProps {
     roleId: string;
     courseTree: CourseNode[];
     initialGrants: Array<{ courseId: string; moduleId: string | null; lessonId: string | null }>;
     allCourses: boolean;
   }
   ```

3. Internal state: `grants` as a `Set`-like structure tracking active permission rows. Use a state array matching the initialGrants shape. Derive a `checkStates: Map<string, CheckState>` from grants using the `deriveTreeState` algorithm from the research (Pattern 2):
   - Course with a course-level grant (moduleId=null) -> "checked" for course and all children
   - Module with a module-level grant -> "checked" for module and all its lessons
   - Individual lesson grants -> "checked" for those lessons, "indeterminate" for parent module/course if partial

4. Implement `deriveTreeState()` as a pure function that takes grants array + courseTree and returns a Map<string, CheckState>. This runs on every render (memoize with useMemo keyed on grants).

5. Auto-save pattern: Use `useState` for a `savingKey: string | null` to track which node is currently saving (show spinner on that node). When a checkbox is toggled:
   - Optimistic update: immediately update the local grants state
   - Fire a `fetch PUT /api/admin/roles/${roleId}/courses` with the appropriate payload
   - On success: `toast.success("Permissions updated")` and clear savingKey
   - On error: revert the optimistic update, `toast.error("Failed to save permission")`, clear savingKey
   - Do NOT use useDebouncedCallback here -- each toggle is an independent mutation that should fire immediately (the research recommends debounce but for checkboxes, immediate fire with optimistic UI is better UX since each click is intentional)

6. Toggle handlers (the core logic):
   - `onToggleCourse(courseId, newChecked)`:
     - If newChecked=true (Select All): Update local grants by removing all rows for this courseId, adding one course-level row (courseId, null, null). API call with `{ courseId, moduleId: null, lessonId: null, granted: true }`.
     - If newChecked=false (Deselect All): Remove all rows for this courseId. API call with `{ courseId, moduleId: null, lessonId: null, granted: false }`.
   - `onToggleModule(courseId, moduleId, newChecked)`:
     - If newChecked=true: Remove any lesson-level rows for this module, add one module-level row. API call with `{ courseId, moduleId, lessonId: null, granted: true }`.
     - If newChecked=false: Remove the module-level row and any lesson-level rows for this module. API call with `{ courseId, moduleId, lessonId: null, granted: false }`.
     - After updating module, check if ALL modules in the course are now checked -- if so, collapse to a course-level grant via additional API call.
     - After unchecking module, check if NO modules or lessons remain for the course -- if so, also remove the course-level entry.
     SIMPLIFICATION: Do NOT auto-collapse to course-level or auto-remove course entry from the client. Let the tree state derivation handle display correctly. The server stores the exact grants as sent. The UI derives checked/indeterminate from whatever grants exist.
   - `onToggleLesson(courseId, moduleId, lessonId, newChecked)`:
     - If newChecked=true: Add lesson-level row. API call with `{ courseId, moduleId, lessonId, granted: true }`.
     - If newChecked=false: Remove lesson-level row. API call with `{ courseId, moduleId, lessonId, granted: false }`.
     SIMPLIFICATION: Same as module -- no auto-collapse logic on client. Tree state derivation handles display.

7. UI structure using Collapsible from `@/components/ui/collapsible`:
   ```
   <div> {/* Tree container */}
     {allCourses && (
       <div className="...banner...">
         This role grants access to all courses. Individual course permissions below are overridden.
       </div>
     )}
     {courseTree.map(course => (
       <div key={course.id}>
         <div className="flex items-center gap-2 py-2">
           {course.modules.length > 0 && (
             <CollapsibleTrigger> <ChevronRight/ChevronDown /> </CollapsibleTrigger>
           )}
           <Checkbox
             checked={checkState === "checked" ? true : checkState === "indeterminate" ? "indeterminate" : false}
             onCheckedChange={(checked) => onToggleCourse(course.id, !!checked)}
             disabled={allCourses}
           />
           <span>{course.title}</span>
           {savingKey === course.id && <Loader2 className="animate-spin" />}
         </div>
         <CollapsibleContent>
           {course.modules.map(mod => (
             /* Similar pattern: collapsible trigger + checkbox + module title */
             /* Nested: mod.lessons.map(lesson => checkbox + lesson title) */
           ))}
         </CollapsibleContent>
       </div>
     ))}
   </div>
   ```
   Each course is wrapped in a `<Collapsible>`. Each module within a course is also wrapped in a `<Collapsible>` for lesson expansion. Use indentation (pl-6 for modules, pl-12 for lessons) to show hierarchy.

8. Style with dark zinc theme matching the app: `border-zinc-700`, `text-zinc-300`, `hover:bg-zinc-800/50`. Checkbox colors: cyan accent when checked (matching the app's cyan theme).
