**1. `getStudentVideoAssignments(userId: string): Promise<ResolvedVideoAssignment[]>`**

Follow the EXACT 5-path resolution pattern from `getStudentAssignments()` in `src/lib/assignments.ts`:

Step 1: Collect all valid target entries for this student:
- Direct: `{ type: "student", id: userId }`
- Tags: query `studentTags` where userId matches, push `{ type: "tag", id: tagId }` for each
- Course enrollment: query `courseAccess` where userId matches and not expired, push `{ type: "course", id: courseId }`
- Modules: for enrolled courseIds, query `modules` (not deleted), push `{ type: "module", id: moduleId }`
- Lessons: for enrolled moduleIds, query `lessons` (not deleted), push `{ type: "lesson", id: lessonId }`

Fetch tags and enrollments in parallel with `Promise.all` (same as assignments.ts).

Step 2: Build OR conditions grouping by type, then query `videoAssignments` matching any condition.

Step 3: LEFT JOIN `videoSessions` on `(videoSessions.youtubeVideoId = videoAssignments.youtubeVideoId AND videoSessions.userId = userId)` to get progress data. This is the KEY difference from practice assignments -- progress comes from videoSessions, not attempts. Select: assignmentId, youtubeUrl, youtubeVideoId, title (from assignment), notes, dueDate, assignedAt (createdAt), completionPercent (from session, nullable), lastPositionMs (nullable), sessionTitle (videoSessions.title, nullable), lastWatched (videoSessions.updatedAt, nullable).

Step 4: Deduplicate by youtubeVideoId (not assignmentId) -- keep most specific target using same TARGET_TYPE_PRIORITY map.

Define the `ResolvedVideoAssignment` interface:
```typescript
export interface ResolvedVideoAssignment {
  assignmentId: string;
  youtubeUrl: string;
  youtubeVideoId: string;
  title: string | null;         // Coach-provided title
  notes: string | null;
  dueDate: Date | null;
  assignedAt: Date;
  completionPercent: number | null;  // null = not started
  lastPositionMs: number | null;
  sessionTitle: string | null;       // Video title from YouTube
  lastWatched: Date | null;
}
```

**2. `getVideoAssignmentProgress(assignmentId: string): Promise<VideoAssignmentProgressResult>`**

For coach progress monitoring. Steps:

Step 1: Get the assignment details (youtubeVideoId, targetType, targetId, title, youtubeUrl).

Step 2: Resolve all target students using a `resolveTargetStudents` helper function:
- `student`: just query that user
- `tag`: query users via studentTags join
- `course`: query users via courseAccess join
- `module`: find parent course, then query users via courseAccess
- `lesson`: find parent module, then parent course, then query users via courseAccess

Return array of `{ id, name, email }`.

Step 3: For each resolved student, LEFT JOIN `videoSessions` on (student.id = videoSessions.userId AND assignment.youtubeVideoId = videoSessions.youtubeVideoId).

Step 4: Return result:
```typescript
export interface StudentVideoProgress {
  studentId: string;
  studentName: string | null;
  studentEmail: string;
  completionPercent: number;    // 0 if no session
  lastWatched: Date | null;
  totalWatchedMs: number;
}

export interface VideoAssignmentProgressResult {
  assignment: {
    id: string;
    youtubeUrl: string;
    youtubeVideoId: string;
    title: string | null;
    dueDate: Date | null;
  };
  students: StudentVideoProgress[];
}
```

Use 80 as the COMPLETION_THRESHOLD constant (>= 80% = completed). Export it for use in UI.

Import all necessary schema tables: `videoAssignments`, `videoSessions`, `users`, `studentTags`, `courseAccess`, `courses`, `modules`, `lessons`. Import drizzle operators: `eq`, `and`, `or`, `inArray`, `isNull`, `desc`, `sql`.
