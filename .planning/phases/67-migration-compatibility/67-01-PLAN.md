  <action>
Create `src/app/api/admin/migration/route.ts` with two handlers:

**GET handler (preview mode):**
1. Auth guard: call `hasMinimumRole("admin")`, return 403 if false
2. Check idempotency first: query `roles` table for any non-deleted role with name starting with "Legacy:" (`sql\`${roles.name} LIKE 'Legacy:%'\``). If found, return `{ alreadyMigrated: true, existingRoleCount: N }`
3. Query all active courseAccess records for students only:
   - Join courseAccess -> users (filter `eq(users.role, "student")`)
   - Join courseAccess -> courses (to get course title)
   - Filter: `or(isNull(courseAccess.expiresAt), gt(courseAccess.expiresAt, now))`
4. Group by user to build fingerprints:
   - For each user, sort their grants by courseId, create fingerprint string: `courseId:accessTier|courseId:accessTier|...`
   - Group users with identical fingerprints into patterns
5. For each pattern, build preview object:
   - `fingerprint`: the pattern string
   - `courseNames`: array of course titles
   - `courseTiers`: array of `{ courseId, courseTitle, accessTier }`
   - `studentCount`: number of students with this pattern
   - `studentEmails`: array of student emails
   - `suggestedRoleName`: `"Legacy: ${courseNames.join(' + ')}"` (truncate at 3 courses with "... (+N more)" if more than 3)
6. Return JSON: `{ alreadyMigrated: false, patterns, totalStudents, totalRolesToCreate }`

**POST handler (execute or verify mode):**
1. Auth guard: same as GET
2. Parse `action` from URL searchParams: `"execute"` (default) or `"verify"`

**If action = "execute":**
1. Check idempotency: same "Legacy:" prefix check as GET. If found, return `{ alreadyMigrated: true }` with 200
2. Run the same analysis as GET to get patterns
3. If zero patterns (no students to migrate), return `{ migrated: true, rolesCreated: 0, studentsAssigned: 0 }`
4. For each pattern, execute in a `db.transaction()`:
   a. Get next sortOrder: `max(roles.sortOrder)` from non-deleted roles, increment by 1
   b. Insert into `roles`: name = suggestedRoleName, description = `"Auto-migrated from courseAccess records during v9.0 deploy. N student(s)."`, color = `"#9ca3af"`, sortOrder = nextOrder. Use `.returning()` to get role.id
   c. Insert into `roleCourses`: one row per courseTier in the pattern (roleId, courseId, accessTier). Set moduleId and lessonId to null (course-level grants only for legacy migration)
   d. Insert into `roleFeatures`: one row per feature key from `FEATURE_KEYS` (all 7). This is CRITICAL -- pre-RBAC all features were available to all students. Import `FEATURE_KEYS` from `@/lib/permissions`. Exact pattern:
      ```typescript
      await tx.insert(roleFeatures).values(
        FEATURE_KEYS.map(featureKey => ({ roleId: role.id, featureKey }))
      );
      ```
   e. Insert into `userRoles`: one row per student in the pattern (userId, roleId, assignedBy = null for system migration, expiresAt = null)
5. Track totals: rolesCreated, studentsAssigned
6. Return JSON: `{ migrated: true, rolesCreated, studentsAssigned }`
7. Wrap the entire loop in try/catch -- if any transaction fails, return 500 with error message

**If action = "verify":**
1. Query all students with active courseAccess (same filter as preview: student role, non-expired)
2. Group courseAccess by student (Map<userId, { email, courseIds[] }>)
3. For each student, call `resolvePermissions(userId)` (import from `@/lib/permissions`)
4. Compare: every courseId in the student's courseAccess must return true from `permissions.canAccessCourse(courseId)`. Any missing = regression
5. Also verify: every FEATURE_KEY returns true from `permissions.canUseFeature(featureKey)` -- legacy students should have all features
6. Build results array: `{ studentId, email, directCourseIds, resolvedCourseIds, missingCourses, featuresMissing, status: "pass"|"fail" }`
7. Return JSON: `{ results, totalChecked, totalPassed, totalFailed }`

**Import pattern:**
```typescript
import "server-only";
import { NextRequest, NextResponse } from "next/server";
import { hasMinimumRole } from "@/lib/auth";
import { db } from "@/db";
import { courseAccess, users, courses, roles, roleCourses, roleFeatures, userRoles } from "@/db/schema";
import { eq, and, or, isNull, gt, sql, max, count } from "drizzle-orm";
import { resolvePermissions, FEATURE_KEYS } from "@/lib/permissions";
```

**Key decisions:**
- Skip students with zero active courseAccess records after filtering (avoid empty roles, per Pitfall 4)
- Role name truncation: if >3 courses, show first 3 and append "... (+N more)" (per Open Question 1)
- Admin/coach users skipped per research recommendation
- `assignedBy: null` for system migration (signature supports null per Decision 26)
- No need to import `createRole()` from roles.ts -- the migration needs custom logic (auto-incrementing sortOrder within transaction, bulk insert pattern)
