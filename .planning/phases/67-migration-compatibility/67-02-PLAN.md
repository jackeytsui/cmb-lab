**Page layout:**
- Page title: "Migration Tool" with description "Migrate existing courseAccess records to RBAC Legacy roles"
- Three sections/steps displayed vertically:
  1. **Preview** section
  2. **Execute** section
  3. **Verify** section

**State management:**
- `previewData`: null | preview response from GET
- `migrationResult`: null | execute response from POST
- `verifyResult`: null | verify response from POST?action=verify
- `loading`: { preview: boolean, execute: boolean, verify: boolean }
- `error`: string | null

**Preview section:**
- Button: "Load Preview" -- calls `GET /api/admin/migration`
- If `alreadyMigrated: true`: show green banner "Migration already completed. {existingRoleCount} Legacy role(s) exist."
- If patterns returned: show a card for each pattern:
  - Role name (suggested) in bold
  - Course list with access tiers as badges (e.g., "Beginner Cantonese (full)")
  - Student count and expandable list of student emails
  - All 7 features will be granted (note this in the card)

**Execute section:**
- Button: "Execute Migration" -- disabled until preview is loaded and shows patterns (not alreadyMigrated)
- Confirmation: clicking shows inline "Are you sure? This will create {N} roles and assign {M} students." with Confirm/Cancel
- On success: show green result card with rolesCreated and studentsAssigned counts
- On failure: show red error card with error message

**Verify section:**
- Button: "Run Verification" -- always enabled (can verify even without running migration, to check existing state)
- Shows results table: student email, direct course count, resolved course count, missing courses (if any), status (pass/fail badge)
- Summary bar: "X/Y students passed verification" with green/red coloring

**Styling:** Follow existing admin page patterns:
- Dark theme: bg-zinc-900, border-zinc-700, text-zinc-300/400
- Cards: rounded-lg border border-zinc-700 bg-zinc-800 p-6
- Buttons: rounded-lg bg-blue-600 hover:bg-blue-700 px-4 py-2 text-white (primary), bg-zinc-700 hover:bg-zinc-600 (secondary)
- Success state: green-500/10 bg with green-400 text
- Error state: red-500/10 bg with red-400 text
- Loading: "Loading..." text or spinner

**Fetch calls:**
```typescript

```

Keep it functional -- no over-engineering. This is an admin tool used occasionally, not a high-traffic page. Simple useState + fetch pattern is appropriate.
