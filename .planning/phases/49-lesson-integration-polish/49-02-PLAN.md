**Create page.tsx (server component):**

1. Auth guard using `getCurrentUser()` from `@/lib/auth`. Redirect to `/sign-in` if null.

2. Query all saved vocabulary for the user:
   ```tsx
   const items = await db
     .select()
     .from(savedVocabulary)
     .where(eq(savedVocabulary.userId, user.id))
     .orderBy(desc(savedVocabulary.createdAt));
   ```

3. Pass items to `<VocabularyClient items={items} />`

4. Page title: "My Vocabulary" with a brief subtitle "Words you've saved from the Reader"

**Create VocabularyClient.tsx (client component):**

"use client" component that receives `items: SavedVocabulary[]` as prop.

1. **Search filter**: A text input at the top that filters items client-side by matching traditional, simplified, pinyin, jyutping, or definitions text. Use `useState` for the search query and `useMemo` for filtered items.

2. **Empty state** (when items.length === 0): Centered layout with `Bookmark` icon (lucide), "No saved vocabulary yet" heading, and text: "Open the Reader and tap the bookmark icon on any word to save it." with a Link to `/dashboard/reader`.

3. **Vocabulary cards**: Render each item as a card-like row in a `div.space-y-2`:
   - Left side: Traditional characters (large, bold, text-zinc-100), simplified in parentheses if different from traditional
   - Middle: Pinyin (text-cyan-400) and jyutping (text-amber-400) on separate lines, definitions as a comma-separated list (text-zinc-300, text-sm)
   - Right side: Date saved (formatted with `toLocaleDateString`), a TTS play button (small, using useTTS hook for Mandarin), and a delete/unsave button (Trash2 icon, text-zinc-500 hover:text-red-400)

4. **Delete handler**: Call `DELETE /api/vocabulary?id={item.id}` and remove from local state on success. Use optimistic removal with rollback on failure.

5. **TTS integration**: Import `useTTS` hook. Each card gets a small play button that calls `speak(item.traditional, { language: "zh-CN" })`. Only one useTTS instance for the whole list (not per card).

6. **Search results count**: Show "X words" or "X of Y words" when filtered.

**Create loading.tsx (loading skeleton):**

Follow the loading skeleton convention (bg-zinc-800 Skeleton components, no sidebar/header):
- Skeleton bar for title (h-8 w-48)
- Skeleton bar for search input (h-10 w-full)
- 5 skeleton card rows (h-20 w-full each, with inner skeleton bars for character, pinyin, definition)
