<objective>
Add loading skeletons for knowledge base and AI logs pages, wrap remaining server component DB queries in try/catch with ErrorAlert, fix silent fetch failure in BatchAssignModalWrapper, add try/catch to uploads page, and replace KbEntryForm ad-hoc error styling with shared ErrorAlert.

Purpose: Knowledge base, AI logs, uploads, and content management pages are the remaining admin pages without proper error handling. KB and AI logs pages are server components with complex parallel DB queries and no try/catch. The uploads page queries directly without error handling. The BatchAssignModalWrapper silently swallows video fetch errors, making admins think there are no videos to assign when the API is down. The KbEntryForm uses a plain `<p>` tag for errors instead of ErrorAlert. This plan completes the admin UX polish by applying the same patterns from Plan 01 to these remaining pages.

Output: 2 new loading.tsx files, 5 modified server component page.tsx files with try/catch, 1 modified client component with error handling, and 1 modified form component with ErrorAlert.
</objective>
