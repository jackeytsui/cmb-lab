  <action>
Create React hook that manages WebRTC connection to OpenAI Realtime API.

**src/lib/realtime-utils.ts** - Low-level WebRTC utilities:

1. createRealtimeSession(token: string): Promise<{ pc: RTCPeerConnection, dc: RTCDataChannel }>
   - Create RTCPeerConnection
   - Add local audio track from getUserMedia({ audio: true })
   - Create data channel named "oai-events" for JSON events
   - Set remote description from OpenAI's SDP answer
   - Return peer connection and data channel

2. sendSessionUpdate(dc: RTCDataChannel, instructions: string): void
   - Send session.update event with instructions
   - Event format: { type: "session.update", session: { instructions } }

3. cleanupRealtimeSession(pc: RTCPeerConnection): void
   - Close all tracks
   - Close peer connection

**WebRTC Connection Flow:**
1. Fetch ephemeral token from /api/realtime/token
2. Create offer SDP
3. POST offer to https://api.openai.com/v1/realtime with Authorization: Bearer {ephemeral_token}
4. Set remote description from response SDP
5. Audio tracks automatically connect

**src/hooks/useRealtimeConversation.ts**:

State:
- status: 'idle' | 'connecting' | 'connected' | 'error'
- error: string | null
- isMuted: boolean (local mic mute)

Methods:
- connect(instructions?: string): Promise<void> - Start conversation
- disconnect(): void - End conversation
- toggleMute(): void - Mute/unmute local mic
- sendMessage(text: string): void - Send text input (optional, for debugging)

Implementation notes:
- Use useRef for RTCPeerConnection and RTCDataChannel (not state)
- Audio output handled automatically by RTCPeerConnection (remote track plays through default speakers)
- Clean up on unmount
- Handle ontrack event to get remote audio
- Create audio element dynamically, set srcObject to remote stream, play()

Data channel events to handle:
- session.created - Connection established
- response.audio_transcript.delta - AI speaking (for transcript capture later)
- error - Log and set error state
