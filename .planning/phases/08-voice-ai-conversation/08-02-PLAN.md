**src/db/schema/conversations.ts** - Database schema for voice conversations:

conversations table:
- id: uuid primaryKey defaultRandom
- userId: uuid FK to users.id (cascade delete)
- lessonId: uuid FK to lessons.id (cascade delete)
- startedAt: timestamp notNull defaultNow
- endedAt: timestamp (nullable - set when conversation ends)
- durationSeconds: integer (nullable - calculated on end)
- createdAt: timestamp notNull defaultNow

conversationTurns table:
- id: uuid primaryKey defaultRandom
- conversationId: uuid FK to conversations.id (cascade delete)
- role: pgEnum "turn_role" ('user', 'assistant')
- content: text notNull (transcribed text)
- audioUrl: text (nullable - for future audio storage)
- timestamp: integer notNull (seconds from conversation start)
- createdAt: timestamp notNull defaultNow

Relations:
- conversation belongs to user and lesson
- conversation has many turns

Export types: Conversation, NewConversation, ConversationTurn, NewConversationTurn

**src/lib/lesson-context.ts** - Build AI instructions from lesson data with pronunciation feedback:

buildLessonInstructions(lessonId: string): Promise<string>

1. Query lesson with module and course info
2. Query interactions for this lesson (get prompts and expected answers)
3. Build instruction string:

```
You are a Cantonese and Mandarin language tutor helping a student practice.

Current lesson: {lesson.title}
Course: {course.title}
Module: {module.title}

Vocabulary and phrases from this lesson:
{For each interaction: prompt and expectedAnswer}

Guidelines:
- Speak in the language the student uses (Cantonese or Mandarin)
- If student speaks English, gently encourage them to try in Chinese
- Reference the lesson vocabulary naturally in conversation
- Keep responses conversational and encouraging
- If student struggles, provide hints from the lesson content
- Praise correct usage of lesson vocabulary

PRONUNCIATION FEEDBACK (CRITICAL):
- Listen carefully for pronunciation errors in the student's speech
- When you detect mispronunciation, correct it immediately by:
  1. Saying the word slowly and clearly
  2. Breaking it into syllables or tones if helpful
  3. Asking the student to repeat after you
- Pay special attention to:
  - Tone accuracy (Cantonese has 6-9 tones, Mandarin has 4+1)
  - Initial consonants that differ between Cantonese/Mandarin
  - Final sounds and vowel distinctions
- After correction, continue the conversation naturally
- If the student repeats correctly, acknowledge with brief praise ("Good!", "That's right!")
- If still incorrect after 2 attempts, move on gently and revisit later

Teaching approach: This course teaches students to leverage Cantonese knowledge to learn Mandarin. Help students see connections between the two languages. Point out when a word sounds similar or different in both languages.
```

Handle case where lesson not found: return generic language tutor instructions with pronunciation feedback section.

**src/app/api/conversations/route.ts**:

GET - List conversations:
- Requires auth (any role can view their own)
- Coach/admin can view any student's conversations via ?studentId=
- Query params: studentId?, lessonId?, limit (default 20), offset (default 0)
- Returns conversations with lesson info (join)
- Order by startedAt desc

POST - Create conversation:
- Requires auth
- Body: { lessonId: string }
- Creates conversation record with userId from auth, lessonId from body
- Returns created conversation

**src/app/api/conversations/[conversationId]/route.ts**:

GET - Get conversation with turns:
- Requires auth (owner or coach/admin)
- Returns conversation with all turns ordered by timestamp
- Include lesson info (title, course title, module title)

PATCH - Update conversation (end it or add turns):
- Requires auth (owner only)
- Body options:
  - { endedAt: true } - Set endedAt to now, calculate durationSeconds
  - { turns: [{ role, content, timestamp }] } - Batch insert turns
- Returns updated conversation
