Create `src/db/schema/chat.ts` with two tables following the pattern in `src/db/schema/conversations.ts`:

**chat_conversations table:**
- `id`: uuid, defaultRandom, primaryKey
- `userId`: uuid, notNull, references users.id with onDelete cascade
- `lessonId`: uuid, nullable, references lessons.id with onDelete "set null" (general chat has no lesson)
- `title`: text, nullable (auto-generated from first message or lesson title)
- `createdAt`: timestamp, notNull, defaultNow
- `updatedAt`: timestamp, notNull, defaultNow, with $onUpdate(() => new Date())

**chat_messages table:**
- `id`: uuid, defaultRandom, primaryKey
- `conversationId`: uuid, notNull, references chatConversations.id with onDelete cascade
- `role`: text, notNull (stores "user" or "assistant" — use plain text, NOT an enum, because AI SDK uses string roles)
- `parts`: jsonb, notNull (stores the UIMessage parts array directly — this is the key to restoring messages correctly)
- `createdAt`: timestamp, notNull, defaultNow

**Relations:**
- chatConversations: one(users), one(lessons, nullable), many(chatMessages)
- chatMessages: one(chatConversations)

**Type exports:**
- ChatConversation, NewChatConversation (from $inferSelect/$inferInsert)
- ChatMessage, NewChatMessage

Create `src/lib/chat-persistence.ts` with three functions. Import db from `@/db`, schema tables from `@/db/schema`, and eq/desc/and from `drizzle-orm`.

**Function 1: saveChat**
```typescript
export async function saveChat({
  chatId,
  messages,
  userId,
  lessonId,
}: {
  chatId: string;
  messages: Array<{ id: string; role: string; parts: unknown[] }>;
  userId: string;
  lessonId?: string | null;
}): Promise<void>
```
Implementation:
1. Upsert the conversation: use `db.insert(chatConversations).values({ id: chatId, userId, lessonId: lessonId ?? null, title: null }).onConflictDoUpdate({ target: chatConversations.id, set: { updatedAt: new Date() } })`.
2. Auto-generate title if not set: after upsert, check if conversation has no title. If not, derive title from the first user message (truncate to 50 chars) or from the lesson title if lessonId is provided. Update with `db.update(chatConversations).set({ title }).where(eq(chatConversations.id, chatId))`.
3. Delete existing messages for this conversation: `db.delete(chatMessages).where(eq(chatMessages.conversationId, chatId))`.
4. Insert all messages: `db.insert(chatMessages).values(messages.map(m => ({ conversationId: chatId, role: m.role, parts: m.parts })))` — only if messages.length > 0.

Note: The delete-then-insert pattern is used because onFinish provides the FULL message array. This avoids complex diff logic and ensures consistency with what the AI SDK has.

**Function 2: loadChat**
```typescript
export async function loadChat(chatId: string): Promise<{
  conversation: ChatConversation | null;
  messages: Array<{ role: string; parts: unknown[] }>;
}>
```
Implementation:
1. Query conversation: `db.select().from(chatConversations).where(eq(chatConversations.id, chatId)).limit(1)`
2. If not found, return `{ conversation: null, messages: [] }`
3. Query messages: `db.select().from(chatMessages).where(eq(chatMessages.conversationId, chatId)).orderBy(chatMessages.createdAt)`
4. Return conversation + messages array (map to { role, parts } shape)

**Function 3: listUserConversations**
```typescript
export async function listUserConversations(
  userId: string,
  lessonId?: string | null
): Promise<Array<{ id: string; title: string | null; lessonId: string | null; updatedAt: Date; messageCount: number }>>
```
Implementation:
1. Build where clause: `eq(chatConversations.userId, userId)` and optionally `eq(chatConversations.lessonId, lessonId)` if lessonId provided. Use `and()` to combine.
2. Query conversations ordered by updatedAt desc, limited to 50 most recent.
3. For each conversation, get message count via a subquery or separate count query. A simple approach: use `db.select({ count: count() }).from(chatMessages).where(eq(chatMessages.conversationId, conv.id))`.
4. Alternatively, use a single query with a left join and groupBy for efficiency. Either approach is fine for <= 50 conversations.
5. Return array of `{ id, title, lessonId, updatedAt, messageCount }`.

Import `count` from `drizzle-orm` for the count aggregation. Import `desc` for ordering.
