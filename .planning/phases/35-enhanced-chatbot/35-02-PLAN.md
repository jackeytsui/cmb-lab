  <action>
Create `src/components/chat/ChatBubble.tsx` — a presentational iMessage-style bubble component.

Props interface:
```typescript
interface ChatBubbleProps {
  role: "user" | "assistant";
  timestamp?: Date;
  children: React.ReactNode;
  className?: string;
}
```

Implementation:
- "use client" directive at top
- User messages: right-aligned (`justify-end`), blue-600 background, white text, `rounded-2xl rounded-br-sm` (iMessage tail on bottom-right)
- Assistant messages: left-aligned (`justify-start`), zinc-800 background, zinc-100 text, `rounded-2xl rounded-bl-sm` (tail on bottom-left)
- Max width: `max-w-[80%]` to prevent full-width stretching
- Padding: `px-4 py-2.5`
- Word break: `break-words`
- Timestamp: Rendered below the bubble in `text-[10px] text-zinc-500`, right-aligned for user, left-aligned for assistant
- Timestamp format: Use a `formatRelativeTime` helper function defined in the same file that shows:
  - "just now" if < 1 minute ago
  - "Xm ago" if < 60 minutes ago
  - "Xh ago" if < 24 hours ago
  - "HH:MM" (24h format) if today
  - "Mon HH:MM" if this week
  - Full date otherwise (via toLocaleDateString)
- Wrap the bubble in `<motion.div>` from framer-motion with `initial={{ opacity: 0, y: 8 }}`, `animate={{ opacity: 1, y: 0 }}`, `transition={{ duration: 0.2 }}`.
- Render timestamp as a `<p>` OUTSIDE the bubble div but inside the outer wrapper, with `mb-2` spacing after the timestamp line.
- If no timestamp prop, skip the timestamp line.

Export both `ChatBubble` and `formatRelativeTime` (the latter for potential reuse).
