  <files>src/app/(dashboard)/layout.tsx, next.config.ts</files>
  <action>
  **Part A — Build Fix:**

  Add `export const dynamic = "force-dynamic";` to `src/app/(dashboard)/layout.tsx` BEFORE the default export function. This single line prevents Next.js from attempting static generation on any dashboard page, which fixes the "Missing publishableKey" build error.

  Do NOT add force-dynamic to individual page files or to the root layout — only the `(dashboard)/layout.tsx`.

  **Part B — Security Headers:**

  In `next.config.ts`, update the `/(.*) ` source headers array to add three new headers after the existing three (X-Content-Type-Options, X-Frame-Options, Referrer-Policy):

  1. **Content-Security-Policy** — Build the CSP string as a const above the config:
     ```
     default-src 'self';
     script-src 'self' 'unsafe-inline' ${isDev ? "'unsafe-eval'" : ""} https://*.clerk.accounts.dev https://challenges.cloudflare.com;
     style-src 'self' 'unsafe-inline';
     img-src 'self' blob: data: https://img.clerk.com https://image.mux.com;
     font-src 'self';
     connect-src 'self' https://*.clerk.accounts.dev https://api.openai.com https://stream.mux.com https://*.upstash.io;
     frame-src https://challenges.cloudflare.com;
     worker-src 'self' blob:;
     object-src 'none';
     base-uri 'self';
     form-action 'self';
     frame-ancestors 'none';
     upgrade-insecure-requests;
     ```
     Use `const isDev = process.env.NODE_ENV === "development";` to conditionally include `'unsafe-eval'` for Next.js hot reload in dev mode. Collapse the multiline string by replacing newlines to produce a single-line header value.

     Add a code comment: `// Production: replace *.clerk.accounts.dev with your custom Clerk domain if using one`

  2. **Strict-Transport-Security**: `max-age=63072000; includeSubDomains; preload`

  3. **Permissions-Policy**: `camera=(self), microphone=(self), geolocation=()`
     (camera and microphone needed for voice AI and audio recording)

  **Verify the build passes after these changes** by running `npm run build`.
  </action>
