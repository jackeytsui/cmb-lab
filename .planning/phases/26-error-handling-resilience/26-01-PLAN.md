Create `src/components/ui/error-alert.tsx` as a 'use client' component with two variants:

**Inline variant** (default): For embedding inside existing component layouts. Renders as a compact red-tinted banner with:

**Block variant**: For standalone centered error displays (e.g., replacing a full section). Renders as a centered column with:
- Circular icon container (w-12 h-12 bg-red-600/20 rounded-full)
- `AlertCircle` icon (w-6 h-6 text-red-400)

**Props interface:**
```typescript
interface ErrorAlertProps {
  message: string
  onRetry?: () => void
  className?: string
  variant?: 'inline' | 'block'
}
```

Export the `ErrorAlert` component as a named export. Default variant is 'inline'. The `className` prop should be appended to the outermost container div.

<task type="auto">
  <name>Task 2: Create Next.js error.tsx route boundaries</name>
  <files>src/app/error.tsx, src/app/(dashboard)/error.tsx, src/app/global-error.tsx</files>
  <action>
Create three error boundary files following the Next.js 16 App Router `error.tsx` convention:

**1. `src/app/(dashboard)/error.tsx`** — Dashboard-level error boundary (most important, covers all authenticated routes):
- 'use client' directive
- Default export function `DashboardError({ error, reset })` with typed props: `{ error: Error & { digest?: string }; reset: () => void }`
- useEffect that console.error's the error for debugging
- Centered layout (min-h-[50vh] to avoid full-page takeover since dashboard has its own layout, flex items-center justify-center p-8)
- Max-w-md container with:
  - Red circular icon (w-16 h-16 bg-red-600/20 rounded-full) with AlertCircle (w-8 h-8 text-red-400)
  - "Something went wrong" heading (text-xl font-semibold text-white mb-2)
  - Error message or fallback text (text-zinc-400 text-sm mb-6): use `error.message` if present, else "An unexpected error occurred. Please try again."
  - Two buttons in a flex row (gap-3):
    - "Try again" button calling `reset()` (bg-cyan-600 hover:bg-cyan-500 text-white rounded-lg, with RefreshCw icon)
    - "Dashboard" link to `/dashboard` (bg-zinc-800 hover:bg-zinc-700 text-zinc-300 rounded-lg, with Home icon from lucide-react)
- Import AlertCircle, RefreshCw, Home from lucide-react
- Import Link from next/link

**2. `src/app/error.tsx`** — Root-level error boundary (catches errors in non-dashboard routes like /verify):
- Same structure as dashboard error.tsx but with min-h-screen (full page since no layout wrapper)
- "Home" link goes to `/` instead of `/dashboard`
- Heading: "Something went wrong"

**3. `src/app/global-error.tsx`** — Global error boundary (catches root layout errors):
- 'use client' directive
- MUST include its own `<html>` and `<body>` tags (required by Next.js for global-error since it replaces the root layout)
- Body has bg-zinc-900 class
- Same visual structure as root error.tsx
- Simpler: only a "Try again" button (no links, since the layout itself is broken)

All three files should use the same dark theme styling (bg-zinc-900, text-white, text-zinc-400, red/cyan accents) consistent with the rest of the app.
  </action>
