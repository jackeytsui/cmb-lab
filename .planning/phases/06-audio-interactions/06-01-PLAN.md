  <action>
Create `src/lib/audio-utils.ts` with:
- `getSupportedMimeType()`: Returns best MIME type for current browser
  - Try in order: audio/webm;codecs=opus, audio/webm, audio/ogg;codecs=opus, audio/mp4, audio/mp4;codecs=mp4a.40.2
  - Use `MediaRecorder.isTypeSupported()` to detect
  - Fallback to 'audio/webm' if none supported
- `validateAudioBlob(blob: Blob)`: Returns boolean
  - Check blob exists and size > 0
  - Check blob.type starts with valid audio type (audio/webm, audio/mp4, audio/ogg)
- `getFileExtensionForMimeType(mimeType: string)`: Returns file extension string
  - audio/webm* -> 'webm'
  - audio/mp4* -> 'mp4'
  - audio/ogg* -> 'ogg'
  - default -> 'webm'

Create `src/hooks/useAudioRecorder.ts` with:
- State: `status` ('idle' | 'recording' | 'stopped' | 'error')
- State: `audioBlob` (Blob | null)
- State: `audioUrl` (string | null) - for playback
- State: `error` (string | null)
- State: `recordingTime` (number) - seconds elapsed
- State: `mimeType` (string | null) - detected MIME type for the recording
- Ref: `mediaRecorderRef` (MediaRecorder | null)
- Ref: `chunksRef` (Blob[])
- Ref: `streamRef` (MediaStream | null)

Methods:
- `startRecording()`: async
  1. Call `navigator.mediaDevices.getUserMedia({ audio: true })`
  2. Detect MIME type via `getSupportedMimeType()`
  3. Store mimeType in state for later use
  4. Create MediaRecorder with detected MIME
  5. Set up ondataavailable to push chunks
  6. Set up onstop to create blob and URL
  7. Start recording, set status to 'recording'
  8. Handle errors with user-friendly messages:
     - Permission denied: "Microphone access denied. Please allow microphone access in your browser settings."
     - NotFoundError: "No microphone found. Please connect a microphone and try again."
     - NotSupportedError: "Audio recording is not supported in this browser."
     - Generic error: "Failed to start recording. Please try again."

- `stopRecording()`:
  1. Check mediaRecorder exists and state is 'recording'
  2. Call mediaRecorder.stop()
  3. Stop all stream tracks (release microphone)
  4. Status changes to 'stopped' in onstop handler

- `reset()`:
  1. Revoke audioUrl if exists (prevent memory leak)
  2. Clear chunks, blob, url, error, mimeType
  3. Set status back to 'idle'

Timer logic in useEffect:
- When status is 'recording', increment recordingTime every second
- Clear interval on status change or unmount
- Maximum 60 seconds default (can be prop later)

Return: { status, audioBlob, audioUrl, mimeType, error, recordingTime, startRecording, stopRecording, reset }

**iOS Safari considerations:**
- iOS Safari supports MediaRecorder since iOS 14.3 but prefers audio/mp4
- The MIME type detection order handles this (mp4 is tried after webm/ogg)
- getUserMedia requires user gesture on iOS - the Start Recording button provides this
