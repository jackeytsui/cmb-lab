  <action>
    Create `src/lib/rate-limit.ts` with the following exports:

    1. **Redis instance**: `Redis.fromEnv()` -- reads UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN automatically.

    2. **Limiter instances** (all use `Ratelimit.slidingWindow`):
       - `aiChatLimiter`: 20 requests per "1 m", prefix "ratelimit:chat"
       - `aiChatLimiterElevated`: 60 requests per "1 m", prefix "ratelimit:chat:elevated"
       - `gradingLimiter`: 10 requests per "1 m", prefix "ratelimit:grade"
       - `gradingLimiterElevated`: 30 requests per "1 m", prefix "ratelimit:grade:elevated"
       - `webhookLimiter`: 10 requests per "1 m", prefix "ratelimit:webhook"
       - All with `analytics: true`

    3. **`rateLimitResponse(result)`**: Takes an object with `{ limit, remaining, reset }`, returns `NextResponse.json(...)` with:
       - Status 429
       - Body: `{ error: "Too many requests. Please slow down and try again shortly.", retryAfter }`
       - Headers: `Retry-After` (seconds, minimum 1), `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`
       - retryAfter calculation: `Math.max(1, Math.ceil((result.reset - Date.now()) / 1000))`

    4. **`getClientIp(req: Request)`**: Returns string. Check `x-forwarded-for` header first (take first IP from comma-separated list, trimmed), fall back to `x-real-ip`, fall back to `"unknown"`.

    5. **`selectLimiter(role: string, standard: Ratelimit, elevated: Ratelimit)`**: Returns elevated if role is "admin" or "coach", otherwise standard.

    Reference the complete code example in 19-RESEARCH.md. Use `import { NextResponse } from "next/server"` for the response helper.
  </action>

  </verify>
