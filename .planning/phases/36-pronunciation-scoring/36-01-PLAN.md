  <files>

  <action>
1. Create `src/types/pronunciation.ts` with these types:
   - `PronunciationWordResult`: `{ word: string; accuracyScore: number; errorType: "None" | "Mispronunciation" | "Omission" | "Insertion" | "UnexpectedBreak" | "MissingBreak" | "Monotone" }`
   - `PronunciationAssessmentResult`: `{ overallScore: number; accuracyScore: number; fluencyScore: number; completenessScore: number; prosodyScore?: number; words: PronunciationWordResult[]; recognizedText: string }`

2. Create `src/lib/pronunciation.ts` with these exports:
   - `mapToAzureContentType(mimeType: string): string` — Maps browser MIME types to Azure-compatible Content-Type values. Mapping: `audio/ogg*` -> `audio/ogg; codecs=opus`, `audio/webm*` -> `audio/webm; codecs=opus`, `audio/wav*` -> `audio/wav; codecs=audio/pcm; samplerate=16000`, default -> `audio/ogg; codecs=opus`
   - `assessPronunciation(audioBuffer: Buffer, referenceText: string, language: "zh-CN" | "zh-HK", mimeType: string): Promise<PronunciationAssessmentResult>` — Calls Azure Speech REST API pronunciation assessment endpoint. Implementation:
     a. Read `AZURE_SPEECH_KEY` and `AZURE_SPEECH_REGION` from env. Throw if missing.
     b. Build pronunciation assessment config as base64-encoded JSON: `{ ReferenceText: referenceText, GradingSystem: "HundredMark", Granularity: "Phoneme", Dimension: "Comprehensive", EnableMiscue: "False", EnableProsodyAssessment: "True" }`
     c. POST to `https://{region}.stt.speech.microsoft.com/speech/recognition/conversation/cognitiveservices/v1?language={language}&format=detailed`
     d. Headers: `Ocp-Apim-Subscription-Key`, `Content-Type` (from mapToAzureContentType), `Pronunciation-Assessment` (base64 config), `Accept: application/json`
     e. Body: raw audioBuffer
     f. Parse response: extract `NBest[0].PronunciationAssessment` for overall scores, `NBest[0].Words[]` for per-word results. Map `Words[].PronunciationAssessment.AccuracyScore` and `ErrorType`. For Chinese, each character IS a word.
     g. If `NBest` is empty, throw "No recognition results from Azure"
   - `generatePronunciationFeedback(result: PronunciationAssessmentResult): string` — Generate human-readable feedback based on overall score: >=90 "Excellent pronunciation!", >=70 "Good pronunciation. Keep practicing the highlighted characters.", >=50 "Fair pronunciation. Focus on the characters marked in yellow and red.", <50 "Keep practicing! Pay attention to the tone and clarity of each character."

   Use a 20-second timeout on the Azure fetch call via AbortController. Log errors to console.error with prefix "Azure Speech:".
  </action>

  </verify>

      if (process.env.AZURE_SPEECH_KEY && process.env.AZURE_SPEECH_REGION && targetPhrase) {
        try {

          const pronunciationResult = await assessPronunciation(

            azureLocale,
            audioFile.type || "audio/webm"

            score: pronunciationResult.overallScore,
            feedback: generatePronunciationFeedback(pronunciationResult),
            pronunciationDetails: pronunciationResult,
          };

          return NextResponse.json(gradeResult);

          console.error("Azure pronunciation assessment failed, falling back to n8n:", error);
          // Fall through to n8n
        }
      }
