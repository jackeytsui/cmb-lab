  <action>
    **1. Create `src/lib/user-roles.ts`** -- CRUD service for user role assignments. Follow the pattern from `src/lib/tags.ts` and `src/lib/roles.ts`.

    Three exported functions:
    - `getUserRoles(userId: string)` -- Returns assigned roles with role name, color, description, expiresAt, assignedBy, createdAt. Uses `db.select()` from userRoles JOIN roles WHERE userId matches AND `roles.deletedAt IS NULL`. Returns array of objects with `id, roleId, roleName, roleColor, roleDescription, assignedBy, expiresAt, createdAt`.
    - `assignRole(userId, roleId, assignedBy, expiresAt?)` -- Upsert: check if userRoles record exists for userId+roleId. If exists, update expiresAt and assignedBy. If not, insert new record. Returns `{ action: "created" | "updated", id: string }`.
    - `removeRole(userId, roleId)` -- Delete from userRoles WHERE userId AND roleId match. Returns boolean (true if deleted, false if not found).

    Import from `@/db` and `@/db/schema` (userRoles, roles tables). Use `eq, and, isNull` from drizzle-orm.

    **2. Create `src/app/api/admin/students/[studentId]/roles/route.ts`** -- REST endpoints.

    GET handler:
    - Auth check: `auth()` for clerkId, `hasMinimumRole("coach")` for access.
    - Call `getUserRoles(studentId)` for assigned roles.
    - Query all non-deleted roles NOT already assigned (for the picker dropdown): query `roles` table WHERE `deletedAt IS NULL`, then filter out roleIds already in assigned list.
    - Return `{ roles: [...assigned], availableRoles: [...available] }`.

    POST handler:
    - Auth check (same pattern).
    - Validate body with Zod: `{ roleId: z.string().uuid(), expiresAt: z.string().datetime().optional() }`.
    - Get current user via `getCurrentUser()` for assignedBy field.
    - When expiresAt is provided, convert to Date and set time to end-of-day UTC (23:59:59.999Z) to avoid timezone confusion (Pitfall 4 from research).
    - Call `assignRole(studentId, roleId, currentUser.id, expiresAt)`.
    - Re-fetch the assignment with role details for the response.
    - Return `{ assignment: {...} }` with 201 status.

    DELETE handler:
    - Auth check.
    - Validate body: `{ roleId: z.string().uuid() }`.
    - Call `removeRole(studentId, roleId)`.
    - Return 200 on success, 404 if not found.

    **3. Extend `src/app/api/admin/students/bulk/route.ts`**:

    - Add `"assign_role"` and `"remove_role"` to the `bulkSchema.operation` enum.
    - Add optional `expiresAt: z.string().datetime().optional()` to bulkSchema.
    - Import `assignRole, removeRole` from `@/lib/user-roles`.
    - Add two cases to the switch statement:
      - `assign_role`: Call `assignRole(studentId, targetId, currentUser.id, expiresAt ? new Date(expiresAt) : undefined)`. Push success result.
      - `remove_role`: Call `removeRole(studentId, targetId)`. Push success/failure result based on return value.

    IMPORTANT: Do NOT break existing operations. The expiresAt field must be optional and only used by assign_role.
