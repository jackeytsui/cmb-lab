  <action>
    **1. Create `src/components/auth/FeatureGate.tsx`** -- Server component for feature permission checking.

    This is an async Server Component (NOT "use client"). It:
    - Takes props: `{ feature: FeatureKey; children: React.ReactNode; fallback?: React.ReactNode }`
    - Calls `auth()` from `@clerk/nextjs/server` to get clerkId.
    - If no clerkId, renders fallback or null (unauthenticated user).
    - Queries `db.query.users.findFirst()` WHERE `users.clerkId = clerkId` to get internal userId.
    - Calls `resolvePermissions(userId)` from `@/lib/permissions`.
    - If `permissions.canUseFeature(feature)` returns true, renders `<>{children}</>`.
    - Otherwise, renders fallback or a default `<FeatureLockedFallback>`.

    The `FeatureLockedFallback` is a named function component in the same file (not exported). It renders:
    - Centered layout with `py-16 text-center`.
    - Lock icon (from lucide-react) at w-12 h-12 text-zinc-600.
    - Heading: "{Feature Label} is Locked" (use a labels map for human-readable names: ai_conversation -> "AI Conversation Bot", practice_sets -> "Practice Sets", dictionary_reader -> "Dictionary & Reader", listening_lab -> "YouTube Listening Lab", video_threads -> "Video Threads", certificates -> "Certificates", ai_chat -> "AI Chat").
    - Subtext: "This feature is not included in your current plan. Contact your coach to upgrade your access." in text-zinc-500.

    Import `FeatureKey` type and `resolvePermissions` from `@/lib/permissions`. Import `auth` from `@clerk/nextjs/server`. Import `db` from `@/db` and `users` from `@/db/schema`. Import `eq` from `drizzle-orm`. Import `Lock` from `lucide-react`.
