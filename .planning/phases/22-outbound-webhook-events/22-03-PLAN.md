Create `src/app/api/cron/ghl-webhooks/route.ts` -- a Vercel cron-triggered route that retries failed outbound webhook deliveries.

**Route configuration:**
- Export `maxDuration = 60` (Vercel Pro plan max).
- Export `dynamic = "force-dynamic"` (no caching).

**GET handler:**
1. Verify `CRON_SECRET`: Read `authorization` header, compare to `Bearer ${process.env.CRON_SECRET}`. Return 401 if mismatch. If `CRON_SECRET` env var is not set, log a warning and return 200 (graceful skip in development).

2. Query failed outbound events:
```sql
SELECT * FROM sync_events
WHERE direction = 'outbound'
  AND status = 'failed'
  AND retry_count < 5
ORDER BY created_at ASC
LIMIT 10
```
Use Drizzle ORM: `db.select().from(syncEvents).where(and(...)).orderBy(syncEvents.createdAt).limit(10)`

3. For each event, calculate exponential backoff:
- `backoffMs = Math.pow(4, event.retryCount) * 60_000`
- `retryAfter = new Date(event.updatedAt.getTime() + backoffMs)` -- use `updatedAt` (or `createdAt` if updatedAt not available on sync_events; check schema). Actually sync_events has no updatedAt -- use `createdAt` for first retry, but for subsequent retries we need the last failure time. Since `markEventFailed` doesn't update a timestamp, use `createdAt` + cumulative backoff: `totalBackoff = sum of 4^i * 60000 for i in 0..retryCount-1`. Simpler approach: just use `createdAt` and compute total elapsed time needed: `minElapsed = (4^0 + 4^1 + ... + 4^(retryCount-1)) * 60000`. Even simpler: use `4^retryCount * 60000` from createdAt as a rough approximation. Skip if `Date.now() - event.createdAt.getTime() < backoffMs`.

4. For each event that is ready for retry:
- Call `deliverWebhookFromEvent(event)` (imported from `@/lib/ghl/webhooks`).
- On success: `markEventCompleted(event.id)`.
- On failure: `markEventFailed(event.id, errorMessage)`.
- Wrap in try/catch so one failure doesn't abort the batch.

5. Return JSON response: `{ retried, succeeded, failed, skipped }` with counts.

**Imports needed:**

- `syncEvents` from `@/db/schema`
- `eq, and, lt, sql` from `drizzle-orm`
- `deliverWebhookFromEvent` from `@/lib/ghl/webhooks`
- `markEventCompleted, markEventFailed` from `@/lib/ghl/sync-logger`

**Part A: Create `src/app/api/cron/ghl-inactive/route.ts`**

A daily cron route that detects students inactive for 7+ days and dispatches webhooks.

**Route configuration:**
- Export `maxDuration = 60`.
- Export `dynamic = "force-dynamic"`.

**GET handler:**
1. Verify `CRON_SECRET` (same pattern as retry cron).

2. Query inactive students. Model after the analytics route's inactivity calculation. The query should:
- Select users with role='student' who have at least one lesson_progress record.
- Compute `lastActivityDate` as MAX(lesson_progress.lastAccessedAt) per user (note: lesson_progress schema has NO `updatedAt` column; `lastAccessedAt` is the correct activity timestamp).
- Filter where `lastActivityDate < NOW() - INTERVAL '7 days'`.
- Also include students who enrolled (have a user record) but have ZERO lesson_progress records and whose `createdAt` is older than 7 days.
- Join with users table to get email and name.
- Limit to 20 students per invocation (to stay within Vercel timeout).

```typescript
const inactiveStudents = await db
  .select({
    userId: users.id,
    email: users.email,
    name: users.name,
    lastActivityDate: sql<Date | null>`MAX(${lessonProgress.lastAccessedAt})`,
    totalCompleted: sql<number>`COUNT(CASE WHEN ${lessonProgress.completedAt} IS NOT NULL THEN 1 END)::int`,
  })
  .from(users)
  .leftJoin(lessonProgress, eq(users.id, lessonProgress.userId))
  .where(eq(users.role, "student"))
  .groupBy(users.id, users.email, users.name)
  .having(
    sql`MAX(${lessonProgress.lastAccessedAt}) < NOW() - INTERVAL '7 days'
        OR MAX(${lessonProgress.lastAccessedAt}) IS NULL`
  )
  .limit(20);
```

3. For each inactive student, check deduplication:
- Query `sync_events` for a recent (within 7 days) `student.inactive` event where `entityId = userId` and `direction = 'outbound'` and `status IN ('completed', 'pending')`.
- If found, skip (already notified within this inactivity window).

4. For each non-duplicate inactive student:
- Call `dispatchWebhook({ userId, userEmail: email, eventType: "student.inactive", entityType: "user", entityId: userId, context: { daysSinceActivity, lastActivityDate, totalLessonsCompleted } })`.
- Compute `daysSinceActivity` from `lastActivityDate` (or from `user.createdAt` if no activity).

5. Return JSON: `{ checked, dispatched, skipped }`.
