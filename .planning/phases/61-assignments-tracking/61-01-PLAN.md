```typescript
export const threadAssignments = pgTable("thread_assignments", {
  id: uuid("id").defaultRandom().primaryKey(),
  threadId: uuid("thread_id")
    .notNull()
    .references(() => videoThreads.id, { onDelete: "cascade" }),
  targetType: assignmentTargetTypeEnum("target_type").notNull(),
  targetId: uuid("target_id").notNull(),
  assignedBy: uuid("assigned_by")
    .notNull()
    .references(() => users.id),
  notes: text("notes"),
  dueDate: timestamp("due_date"),
  createdAt: timestamp("created_at").notNull().defaultNow(),
}, (table) => [
  index("thread_assignments_assigned_by_idx").on(table.assignedBy),
  index("thread_assignments_thread_id_idx").on(table.threadId),
  unique("thread_assignments_thread_target_unique").on(
    table.threadId,
    table.targetType,
    table.targetId
  ),
]);
```

Add relations for threadAssignments (belongsTo videoThreads and users). Add to videoThreadsRelations: `assignments: many(threadAssignments)`. Export types: `ThreadAssignment`, `NewThreadAssignment`.

**2. Create src/lib/thread-assignments.ts** following the exact pattern of src/lib/video-assignments.ts:

- `createThreadAssignment(data)` — validates targetType, validates target exists (reuse same validateTargetExists pattern), inserts with unique constraint catch (23505). Accepts: `{ threadId, targetType, targetId, assignedBy, notes?, dueDate? }`.

- `deleteThreadAssignment(id)` — hard delete, returns deleted row or null.

- `listCoachThreadAssignments(coachId)` — select from threadAssignments where assignedBy = coachId, JOIN videoThreads for title, ordered by createdAt desc. Return array with threadTitle included.

- `getStudentThreadAssignments(userId)` — Resolves all assigned threads for a student through 5 target paths (student, tag, course, module, lesson). Same resolution pattern as video-assignments.ts `getStudentVideoAssignments`. LEFT JOIN videoThreadSessions to derive completion status per thread:
  - "not_started": no session exists for this student+thread
  - "in_progress": session exists with status = "in_progress"
  - "completed": session exists with status = "completed"
  Deduplicates by threadId (most specific target wins). Returns array of `ResolvedThreadAssignment` objects with `{ assignmentId, threadId, threadTitle, threadDescription, notes, dueDate, assignedAt, completionStatus }`.

- `getThreadAssignmentProgress(assignmentId)` — Get per-student progress for a specific thread assignment. Resolves target students (reuse `resolveTargetStudents` pattern from video-assignments.ts), LEFT JOINs videoThreadSessions + counts videoThreadResponses per session. Returns `{ assignment: {...}, students: [{ studentId, studentName, studentEmail, completionStatus, responseCount, startedAt, completedAt }] }`.

Define TypeScript interfaces inline: `ResolvedThreadAssignment`, `StudentThreadProgress`, `ThreadAssignmentProgressResult`.

Note: Import the same target resolution dependencies (courses, modules, lessons, courseAccess, studentTags, tags, users) as video-assignments.ts does.
