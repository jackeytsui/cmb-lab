<task type="auto">
  <name>Task 1: Create prompts schema with version history tables</name>
  <files>src/db/schema/prompts.ts, src/db/schema/index.ts</files>
  <action>
Create `src/db/schema/prompts.ts` with:

1. **promptTypeEnum** pgEnum with values: "grading_text", "grading_audio", "voice_ai", "chatbot"

2. **aiPrompts** table:
   - id: uuid, defaultRandom, primaryKey
   - slug: text, notNull, unique (e.g., "voice-tutor-system")
   - name: text, notNull (human-readable name)
   - type: promptTypeEnum, notNull
   - description: text (nullable, what this prompt controls)
   - currentContent: text, notNull
   - currentVersion: integer, notNull, default(1)
   - createdAt: timestamp, notNull, defaultNow
   - updatedAt: timestamp, notNull, defaultNow, $onUpdate(() => new Date())

3. **aiPromptVersions** table:
   - id: uuid, defaultRandom, primaryKey
   - promptId: uuid, notNull, references aiPrompts.id with onDelete cascade
   - version: integer, notNull
   - content: text, notNull
   - changeNote: text (nullable)
   - createdBy: uuid, references users.id (nullable for seed data)
   - createdAt: timestamp, notNull, defaultNow

4. **Relations:**
   - aiPromptsRelations: aiPrompts has many aiPromptVersions
   - aiPromptVersionsRelations: aiPromptVersions belongs to aiPrompts, createdByUser belongs to users

5. Export types: AiPrompt, NewAiPrompt, AiPromptVersion, NewAiPromptVersion

Then update `src/db/schema/index.ts` to export everything from "./prompts".

Follow existing patterns from conversations.ts and notes.ts for reference.
  </action>

Run `npx drizzle-kit generate` - should create migration file without errors.
TypeScript compiles: `npx tsc --noEmit` passes.
  </verify>
