**`src/components/admin/exercises/OrderingForm.tsx`** ("use client"):

Form data type:
```typescript
type OrderingFormData = {
  items: { id: string; text: string }[];
  explanation?: string;
};
```

Note: `correctPosition` is NOT a form field -- it is auto-assigned based on the item's position in the form array. The order the coach enters items IS the correct order. This is the simplest UX: "Enter items in the correct order. Students will see them shuffled."

Zod form schema: z.object with items array min(2).max(10) of {id: string, text: string min(1, "Item text required")}, optional explanation.

Fields:
- Section label: "Ordering Items" with helper text "Enter items in the correct order. Students will see them shuffled and must arrange them correctly."
- Items list via useFieldArray({ control, name: "items" }):
  - Each item: numbered label ("1.", "2.", etc.) + input field + remove button (disabled if only 2 items)
- "Add Item" button (disabled if 10 items reached): `append({ id: nanoid(), text: "" })`
- Explanation textarea (optional)
- ErrorAlert + Submit/Cancel buttons

Default values: edit from exercise?.definition?.items (strip correctPosition for form), else 2 empty items.

Submit: Build definition with correctPosition auto-assigned:
```typescript
const definition = {
  type: "ordering" as const,
  items: formData.items.map((item, index) => ({
    id: item.id,
    text: item.text,
    correctPosition: index,
  })),
  explanation: formData.explanation || undefined,
};
```
Same POST/PUT API pattern.
