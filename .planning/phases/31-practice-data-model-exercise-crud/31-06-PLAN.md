**`src/app/(dashboard)/admin/exercises/page.tsx`** (server component):

This is the main exercise management page. Since exercises belong to practice sets, this page shows practice sets with their exercises.

Structure:
1. Access control: `hasMinimumRole("coach")`, redirect to /dashboard if not
2. Fetch practice sets directly from DB (server component, no self-fetch):
   ```typescript
   import { db } from "@/db";
   import { practiceSets, practiceExercises } from "@/db/schema";
   import { eq, isNull, desc, asc } from "drizzle-orm";
   ```
   - Query all practice sets where deletedAt IS NULL, ordered by createdAt DESC
   - For each set, count exercises (or fetch them if needed for display)
3. Layout:
   - AppHeader with title "Practice Exercises"
   - Action buttons row: "New Practice Set" button (creates a set via API call in a client component) + "New Exercise" link to /admin/exercises/new
   - For each practice set: collapsible section showing set title, status badge (draft/published/archived), exercise count
   - Inside each set section: render ExerciseListClient component (client wrapper for ExerciseList) with the set's exercises
   - If no practice sets: empty state "No practice sets yet. Create your first practice set to start adding exercises."

**IMPORTANT (anti-pattern avoidance):** The server component (page.tsx) MUST query the DB directly for practice sets and their exercises, then pass the resulting `practiceSets` and `exercises` arrays as props to ExerciseListClient. ExerciseListClient MUST NOT fetch its own data — server components that self-fetch their own API routes don't forward auth cookies and always 401. The data flow is: `page.tsx (server, DB query) → passes data as props → ExerciseListClient (client, mutations only)`.

Create a client component `ExerciseListClient` (as a separate file `src/app/(dashboard)/admin/exercises/ExerciseListClient.tsx` or co-located) that:
- **Receives props:** `practiceSets: PracticeSet[]` and `exercises: PracticeExercise[]` (data fetched by parent server component)
- Renders ExerciseList with the received data
- Edit action: router.push to /admin/exercises/[exerciseId]
- Delete action: API call to DELETE /api/admin/exercises/[id], then router.refresh() (server re-fetches fresh data)
- "Add Exercise" button per set: links to /admin/exercises/new?setId=[practiceSetId]
- "New Practice Set" button: shows a simple modal/dialog to create a set (title input, calls POST /api/admin/practice-sets, then router.refresh())

Use the same styling patterns as existing admin pages: dark zinc backgrounds, zinc-700 borders, consistent spacing.
