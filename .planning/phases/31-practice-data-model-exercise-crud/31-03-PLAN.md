**`src/components/admin/exercises/MultipleChoiceForm.tsx`** ("use client"):

Props: `{ exercise?: PracticeExercise; language: string; practiceSetId: string; onSave: (e: PracticeExercise) => void; onCancel: () => void; isSaving: boolean; setIsSaving: (v: boolean) => void }`

Form setup:
- Use `useForm` with explicit `type MCQFormData = { question: string; options: { id: string; text: string }[]; correctOptionId: string; explanation?: string }`
- Use `zodResolver(multipleChoiceFormSchema) as any` for Zod v4 compatibility (per research pitfall 1)
- Define `multipleChoiceFormSchema` locally (or import from exercises.ts) -- Zod object with question min(5), options array min(2).max(6), correctOptionId, optional explanation. Note: this is the FORM schema (without `type` discriminator) for react-hook-form.
- Default values from `exercise?.definition` if editing, else 2 empty options with nanoid() IDs
- Use `useFieldArray({ control, name: "options" })` for dynamic options

Fields:
- Question textarea (label: "Question", placeholder: "Enter the question text...")
- Options list (rendered via fields.map):
  - Each option: Input field + radio button for correct answer + remove button (disabled if only 2 options)
  - Radio buttons are wired to `correctOptionId` via watch/setValue
  - Each option input: `{...register(\`options.${index}.text\`)}`
- "Add Option" button (disabled if 6 options reached): `append({ id: nanoid(), text: "" })`
- Explanation textarea (optional, label: "Explanation (shown after answering)")
- Form error display using ErrorAlert component for API errors
- Submit button ("Save Exercise" / "Saving...") + Cancel button

Submit handler:
- setIsSaving(true)
- Build full exercise payload: `{ practiceSetId, type: "multiple_choice", language, definition: { type: "multiple_choice", ...formData } }`
- If editing (exercise exists): PUT to `/api/admin/exercises/${exercise.id}`
- If creating: POST to `/api/admin/exercises`
- On success: call onSave(responseData.exercise)
- On error: set error state, display ErrorAlert
- setIsSaving(false) in finally

Styling: Dark zinc-800/900 background, zinc-700 borders, consistent with existing admin forms. Use `cn()` utility for class merging.

**`src/components/admin/exercises/FillInBlankForm.tsx`** ("use client"):

Props: same as MCQForm

Form setup:
- Type `FillInBlankFormData = { sentence: string; blanks: { id: string; correctAnswer: string; acceptableAnswers: string }[]; explanation?: string }`
- Note: `acceptableAnswers` stored as comma-separated string in the form, converted to array on submit
- Default values from exercise?.definition if editing
- `useFieldArray({ control, name: "blanks" })` for dynamic blanks

Fields:
- Sentence template textarea (label: "Sentence Template", placeholder: "Use {{blank}} where blanks should appear")
- Helper text: "Use {{blank}} to mark where students type their answers. Example: I {{blank}} to the store {{blank}}."
- Auto-detect blanks: When sentence changes, count `{{blank}}` occurrences and sync the blanks array length (add/remove blank definitions to match). Use `watch("sentence")` + `useEffect`.
- Blanks list (one card per blank):
  - Label: "Blank #1", "Blank #2", etc.
  - Correct answer input (required)
  - Acceptable answers input (comma-separated, optional, label: "Also accept (comma-separated)")
- Explanation textarea (optional)
- ErrorAlert + Submit/Cancel buttons (same pattern as MCQ)

Submit handler:
- Build definition: `{ type: "fill_in_blank", sentence, blanks: formData.blanks.map(b => ({ id: b.id, correctAnswer: b.correctAnswer, acceptableAnswers: b.acceptableAnswers ? b.acceptableAnswers.split(",").map(s => s.trim()).filter(Boolean) : undefined })), explanation }`
- Same POST/PUT pattern as MCQ
