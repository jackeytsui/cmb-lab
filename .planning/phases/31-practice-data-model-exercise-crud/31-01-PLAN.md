<task type="auto">
  <name>Task 1: Create TypeScript types and Zod schemas for all 6 exercise types</name>
  <files>src/types/exercises.ts</files>
  <action>
Create `src/types/exercises.ts` with:

**TypeScript interfaces** (6 exercise definitions):
- `MultipleChoiceDefinition`: type literal "multiple_choice", question (string), options (array of {id: string, text: string}), correctOptionId (string), explanation (optional string)
- `FillInBlankDefinition`: type literal "fill_in_blank", sentence (string with {{blank}} placeholders), blanks (array of {id: string, correctAnswer: string, acceptableAnswers?: string[]}), explanation (optional string)
- `MatchingDefinition`: type literal "matching", pairs (array of {id: string, left: string, right: string}), explanation (optional string)
- `OrderingDefinition`: type literal "ordering", items (array of {id: string, text: string, correctPosition: number}), explanation (optional string)
- `AudioRecordingDefinition`: type literal "audio_recording", targetPhrase (string), referenceText (optional string), explanation (optional string)
- `FreeTextDefinition`: type literal "free_text", prompt (string), sampleAnswer (optional string), rubric (optional string), minLength (optional number), maxLength (optional number), explanation (optional string)
- `ExerciseDefinition` = union of all 6

**Zod schemas** (one per type + discriminated union):
- `multipleChoiceSchema`: z.object with type literal, question min(5), options array min(2).max(6) of {id: string, text: string min(1)}, correctOptionId string, optional explanation
- `fillInBlankSchema`: z.object with type literal, sentence min(5), blanks array min(1) of {id: string, correctAnswer min(1), optional acceptableAnswers array of strings}, optional explanation
- `matchingSchema`: z.object with type literal, pairs array min(2).max(10) of {id: string, left min(1), right min(1)}, optional explanation
- `orderingSchema`: z.object with type literal, items array min(2).max(10) of {id: string, text min(1), correctPosition int min(0)}, optional explanation
- `audioRecordingSchema`: z.object with type literal, targetPhrase min(1), optional referenceText, optional explanation
- `freeTextSchema`: z.object with type literal, prompt min(5), optional sampleAnswer, optional rubric, optional minLength int min(1), optional maxLength int min(1), optional explanation
- `exerciseDefinitionSchema` = z.discriminatedUnion("type", [all 6 schemas])

**Top-level exercise form schema** (for the full exercise including metadata):
- `exerciseFormSchema`: z.object with practiceSetId (uuid string), type (z.enum of 6 types), language (z.enum of "cantonese" | "mandarin" | "both"), definition (exerciseDefinitionSchema), sortOrder (optional number default 0)

Export ALL interfaces, types, and schemas. Use `import { z } from "zod"` (Zod v4 is installed at 4.3.6).

<task type="auto">
  <name>Task 2: Create database schema with 4 tables, 3 enums, and relations</name>
  <files>src/db/schema/practice.ts</files>
  <action>
Create `src/db/schema/practice.ts` following the exact patterns from interactions.ts and courses.ts:

**Imports:**
- `pgTable, uuid, text, timestamp, integer, jsonb, pgEnum, boolean` from "drizzle-orm/pg-core"
- `relations` from "drizzle-orm"
- `users` from "./users"
- `interactionLanguageEnum` from "./interactions" (REUSE existing enum, do NOT create new)
- `courses, modules, lessons` from "./courses"
- `type ExerciseDefinition` from "@/types/exercises"

**Enums (3):**
- `exerciseTypeEnum`: pgEnum("exercise_type", ["multiple_choice", "fill_in_blank", "matching", "ordering", "audio_recording", "free_text"])
- `practiceSetStatusEnum`: pgEnum("practice_set_status", ["draft", "published", "archived"])
- `assignmentTargetTypeEnum`: pgEnum("assignment_target_type", ["course", "module", "lesson", "student", "tag"])

**Tables (4):**

1. `practiceSets` = pgTable("practice_sets"):
   - id: uuid defaultRandom primaryKey
   - title: text notNull
   - description: text (nullable)
   - status: practiceSetStatusEnum notNull default("draft")
   - createdBy: uuid notNull references users.id
   - createdAt: timestamp notNull defaultNow
   - updatedAt: timestamp notNull defaultNow $onUpdate(() => new Date())
   - deletedAt: timestamp (nullable, soft delete)

2. `practiceExercises` = pgTable("practice_exercises"):
   - id: uuid defaultRandom primaryKey
   - practiceSetId: uuid notNull references practiceSets.id onDelete cascade
   - type: exerciseTypeEnum notNull
   - language: interactionLanguageEnum("language") notNull
   - definition: jsonb("definition").notNull().$type<ExerciseDefinition>()
   - sortOrder: integer notNull default(0)
   - createdAt: timestamp notNull defaultNow
   - updatedAt: timestamp notNull defaultNow $onUpdate(() => new Date())
   - deletedAt: timestamp (nullable, soft delete)

3. `practiceSetAssignments` = pgTable("practice_set_assignments"):
   - id: uuid defaultRandom primaryKey
   - practiceSetId: uuid notNull references practiceSets.id onDelete cascade
   - targetType: assignmentTargetTypeEnum notNull
   - targetId: uuid notNull (polymorphic: points to course/module/lesson/user/tag ID)
   - assignedBy: uuid notNull references users.id
   - dueDate: timestamp (nullable)
   - createdAt: timestamp notNull defaultNow
   - Add unique constraint on (practiceSetId, targetType, targetId) to prevent duplicate assignments

4. `practiceAttempts` = pgTable("practice_attempts"):
   - id: uuid defaultRandom primaryKey
   - practiceSetId: uuid notNull references practiceSets.id onDelete cascade
   - userId: uuid notNull references users.id onDelete cascade
   - score: integer (nullable, 0-100 overall score)
   - totalExercises: integer notNull
   - correctCount: integer notNull default(0)
   - startedAt: timestamp notNull defaultNow
   - completedAt: timestamp (nullable)
   - results: jsonb("results").$type<Record<string, { score: number; isCorrect: boolean; response: string }>>() (per-exercise results, nullable)

**Relations:**
- practiceSetsRelations: one(users, createdBy), many(practiceExercises), many(practiceSetAssignments), many(practiceAttempts)
- practiceExercisesRelations: one(practiceSets, practiceSetId)
- practiceSetAssignmentsRelations: one(practiceSets, practiceSetId), one(users, assignedBy)
- practiceAttemptsRelations: one(practiceSets, practiceSetId), one(users, userId)

**Type inference exports:**
- PracticeSet, NewPracticeSet
- PracticeExercise, NewPracticeExercise
- PracticeSetAssignment, NewPracticeSetAssignment
- PracticeAttempt, NewPracticeAttempt
