    **ListeningClient.tsx** -- Wire useTTS with video pause:
    - Import `useTTS` from `@/hooks/useTTS`.
    - Destructure: `const { speak, stop: stopTts, isLoading: ttsLoading, isPlaying: ttsPlaying } = useTTS();`
    - Add state: `const [ttsLineIndex, setTtsLineIndex] = useState(-1);`
    - Add `handleTtsPlay` callback:
      ```typescript
      const handleTtsPlay = useCallback((index: number) => {
        // Get the display text for the line (converted or original)
        const text = displayTexts?.[index] ?? captions?.[index]?.text;
        if (!text) return;

        // Pause video to avoid audio overlap (per research pitfall 5)
        pauseVideo();

        // Track which line is speaking
        setTtsLineIndex(index);

        // Use "zh-CN" as default; could use user's language preference in future
        speak(text, { language: "zh-CN" });
      }, [displayTexts, captions, pauseVideo, speak]);
      ```
    - Pass TTS props to `<TranscriptPanel>`:
      - `onTtsPlay={handleTtsPlay}`
      - `ttsLineIndex={ttsLineIndex}`
      - `isTtsLoading={ttsLoading}`
      - `isTtsPlaying={ttsPlaying}`
    - NOTE: The TTS button should gracefully handle 503 errors from the TTS API when Azure credentials aren't configured. The existing `useTTS` hook already sets `error` state on non-200 responses, so the button will show the error state (brief loading then idle) without crashing.
