  <action>
    Extend the existing `useVideoSync` hook with two new polling-loop features. All logic goes inside the existing 250ms `setInterval` callback to avoid multiple timers.

    **Loop range state and detection:**
    - Add state: `const [loopRange, setLoopRange] = useState<{ startIndex: number; endIndex: number } | null>(null);`
    - Add ref: `const seekingRef = useRef(false);` -- guards against double-trigger after seekTo (per research pitfall 2).
    - Add ref: `const seekCooldownRef = useRef(0);` -- counter decremented each poll tick; when > 0, skip loop boundary check.
    - In the polling callback, AFTER computing `currentMs` and `newIndex`:
      ```
      if (loopRange && isPlayingRef.current && seekCooldownRef.current <= 0) {
        const endMs = captions[loopRange.endIndex].endMs;
        const startMs = captions[loopRange.startIndex].startMs;
        if (currentMs >= endMs) {
          player.seekTo(startMs / 1000, true);
          seekCooldownRef.current = 3; // Skip next 3 poll ticks (750ms)
          return; // Skip further processing this tick
        }
      }
      if (seekCooldownRef.current > 0) seekCooldownRef.current--;
      ```
    - Expose `loopRange` and `setLoopRange` in return object.

    **Auto-pause state and detection:**
    - Add state: `const [autoPauseEnabled, setAutoPauseEnabled] = useState(false);`
    - Add state: `const [isAutoPaused, setIsAutoPaused] = useState(false);`
    - Add ref: `const lastAutoPausedRef = useRef(-1);` -- tracks the last caption index where auto-pause fired (prevents re-trigger per research pitfall 3).
    - Add ref: `const activeCaptionIndexRef = useRef(-1);` -- shadow of the activeCaptionIndex state, updated inside the poll callback to track transitions.
    - In the polling callback, AFTER the loop check and AFTER computing `newIndex`, but BEFORE updating `setActiveCaptionIndex`:
      ```
      if (autoPauseEnabled && isPlayingRef.current) {
        const prevIndex = activeCaptionIndexRef.current;
        // Caption transition: we were on a valid caption, now we've moved to a different index (next caption or gap)
        if (prevIndex >= 0 && prevIndex !== newIndex && prevIndex !== lastAutoPausedRef.current) {
          player.pauseVideo();
          lastAutoPausedRef.current = prevIndex;
          setIsAutoPaused(true);
          // Don't stop polling -- let student click to resume
        }
      }
      activeCaptionIndexRef.current = newIndex;
      ```
    - Add `resumeFromAutoPause` callback:
      ```
      const resumeFromAutoPause = useCallback(() => {
        const player = playerRef.current;
        if (!player) return;
        player.playVideo();
        setIsAutoPaused(false);
      }, []);
      ```
    - When `seekToCaption` is called (student clicks a line), reset `lastAutoPausedRef.current = -1` so auto-pause can fire on the new position.
    - When `setLoopRange` is called (loop cleared or changed), also reset `lastAutoPausedRef.current = -1`.
    - Expose: `autoPauseEnabled`, `setAutoPauseEnabled`, `isAutoPaused`, `resumeFromAutoPause`.

    **Polling callback order (critical):**
    1. Get currentMs from player
    2. Loop boundary check (seekTo if past end)
    3. Seek cooldown decrement
    4. Compute newIndex via binary search
    5. Auto-pause boundary check (pause if caption transition)
    6. Update activeCaptionIndexRef
    7. Update activeCaptionIndex state
    8. Update currentTimeMs state
