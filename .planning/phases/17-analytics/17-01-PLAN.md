       - `parseDateRange(searchParams: URLSearchParams): { from: Date | null; to: Date | null }` - extracts `from` and `to` query params, parses ISO date strings. Returns `{ from: null, to: null }` when params are absent/empty.
       - `formatCsvRow(values: (string|number|null)[]): string` - escapes commas/quotes, joins with comma, adds newline.
       - `formatCsvResponse(headers: string[], rows: (string|number|null)[][]): NextResponse` - builds full CSV string with header row. Returns NextResponse with `text/csv` content-type and `Content-Disposition: attachment; filename="analytics.csv"` header.

       DO NOT create a `requireAdmin()` helper. This two-step pattern is used consistently across all admin routes and must not be abstracted differently here.

    2. Create `src/app/api/admin/analytics/overview/route.ts`:
       - Export `async function getOverviewData(from: Date | null, to: Date | null)` that returns `{ activeStudents: number, pendingReviews: number, totalStudents: number, totalCourses: number }`.
         - `activeStudents`: Count of distinct users with `lessonProgress.lastAccessedAt` within the last 7 days (or within date range if provided). Join with users table to filter role='student'.
         - `pendingReviews`: Count of submissions where `status = 'pending_review'`. If date range provided, filter by `submissions.createdAt`.
         - `totalStudents`: Count of users where role='student'.
         - `totalCourses`: Count of courses where deletedAt is null.
       - Export `async function GET(request: NextRequest)` handler:
         - Auth check: `const { userId } = await auth(); if (!userId) return 401; const hasAccess = await hasMinimumRole("admin"); if (!hasAccess) return 403;`
         - Parse date range with `parseDateRange()`
         - Call `getOverviewData(from, to)` and return `NextResponse.json(data)`
         - Wrap in try/catch, return 500 on error

    3. Create `src/app/api/admin/analytics/completion/route.ts`:
       - Export `async function getCompletionData(from: Date | null, to: Date | null)` that returns array of `{ courseId, courseTitle, totalLessons, enrolledStudents, completedStudents, completionRate }`.
         - For each published course (where deletedAt is null):
           - Count total lessons in that course (via modules -> lessons, where deletedAt is null for both)
           - Count enrolled students (from courseAccess)
           - Count students who completed ALL lessons in the course (lessonProgress.completedAt is not null for every lesson)
           - If date range: only count completions where completedAt falls in range
         - `completionRate` is percentage (0-100, rounded to 1 decimal)
         - Sort by completionRate ascending (worst performing courses first)
       - Export `GET` handler following same auth + parseDateRange + try/catch pattern as overview.

    Follow existing API patterns: NextRequest/NextResponse, try/catch with 500 error, auth check at top using `auth()` + `hasMinimumRole("admin")` as two separate calls. Use drizzle-orm sql template for complex aggregations.

  <name>Task 2: Drop-off and at-risk students endpoints</name>

  <action>
    1. Create `src/app/api/admin/analytics/dropoff/route.ts`:
       - Export `async function getDropoffData(from: Date | null, to: Date | null, limit?: number)` that returns array of `{ lessonId, lessonTitle, moduleTitle, courseTitle, startedCount, completedCount, dropoffCount, dropoffRate }`.
         - "Drop-off" = lessons where students started (lessonProgress.startedAt exists) but never completed (completedAt is null)
         - For each lesson: count students who started but didn't complete, and total who started
         - `dropoffRate` = (startedCount - completedCount) / startedCount * 100
         - Sort by dropoffRate descending (highest drop-off first)
         - If date range: filter by lessonProgress.startedAt within range
         - Default limit: 20. Accept `limit` parameter.
       - Export `GET` handler:
         - Auth check: `const { userId } = await auth(); if (!userId) return 401; const hasAccess = await hasMinimumRole("admin"); if (!hasAccess) return 403;`
         - Parse date range, parse `limit` query param (default 20)
         - Call `getDropoffData(from, to, limit)` and return JSON
         - try/catch with 500

    2. Create `src/app/api/admin/analytics/students/route.ts`:
       - Export `async function getStudentsData(from: Date | null, to: Date | null, daysInactive?: number)` that returns array of `{ userId, name, email, lastActivity, daysSinceActivity, totalLessonsCompleted }`.
         - "At-risk" = students with no lessonProgress.lastAccessedAt in the last N days
         - Get all students (users.role = 'student'), left join with max(lessonProgress.lastAccessedAt) grouped by userId
         - Sort by daysSinceActivity descending (most inactive first)
         - Default daysInactive threshold: 7
         - If date range: filter lessonProgress records within range for the "totalLessonsCompleted" count
       - Export `GET` handler:
         - Same auth pattern as above
         - Parse date range, parse `daysInactive` query param (default 7)
         - Call `getStudentsData(from, to, daysInactive)` and return JSON
         - try/catch with 500
  </action>

         - For each lesson that has interactions: compute average number of attempts per interaction before students get it correct
         - Join interactionAttempts with interactions, group by interaction, calculate avg(attemptNumber) where isCorrect = true (the attempt number when they finally got it right)
         - Aggregate up to lesson level: avg of the per-interaction averages
         - Sort by avgAttemptsToPass descending (hardest first)
         - If date range: filter by interactionAttempts.createdAt within range
       - Export `GET` handler:
         - Auth check: `const { userId } = await auth(); if (!userId) return 401; const hasAccess = await hasMinimumRole("admin"); if (!hasAccess) return 403;`
         - Parse date range, call `getDifficultyData(from, to)`, return JSON
         - try/catch with 500

    2. Create `src/app/api/admin/analytics/export/route.ts`:
       - Export `GET` handler:
         - Auth check: use `hasMinimumRole("coach")` instead of "admin" (ANLYT-08 says coaches can export too)
         - Accepts `metric` query param (required): one of "overview" | "completion" | "dropoff" | "students" | "difficulty"
         - Accepts `from`/`to` date range params (passed through to data functions)
         - Uses a switch/case on the `metric` param to call the correct data function:
           ```
           switch (metric) {
             case "overview": data = await getOverviewData(from, to); break;
             case "completion": data = await getCompletionData(from, to); break;
             case "dropoff": data = await getDropoffData(from, to); break;
             case "students": data = await getStudentsData(from, to); break;
             case "difficulty": data = await getDifficultyData(from, to); break;

           }
           ```

           - `import { getDropoffData } from "../dropoff/route";`
           - `import { getStudentsData } from "../students/route";`
           - `import { getDifficultyData } from "../difficulty/route";`
         - After getting data, map it to CSV rows with appropriate headers per metric:
           - overview: "Metric,Value" (one row per stat)
           - completion: "Course,Total Lessons,Enrolled,Completed,Completion Rate"
           - dropoff: "Lesson,Module,Course,Started,Completed,Drop-off Rate"
           - students: "Name,Email,Last Activity,Days Inactive,Lessons Completed"
           - difficulty: "Lesson,Module,Course,Interactions,Avg Attempts"
         - Return using `formatCsvResponse(headers, rows)` from `src/lib/analytics.ts`
