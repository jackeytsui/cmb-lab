**Conversations page (`src/app/(dashboard)/coach/conversations/page.tsx`):**

- The auth checks (hasMinimumRole, getCurrentUser) stay OUTSIDE try/catch

- In the catch block: `console.error("Failed to load conversations:", error);` then return the same page shell (min-h-screen bg-zinc-900, AppHeader title="Student Conversations", container, back link to /coach) with `<ErrorAlert variant="block" message="Unable to load conversations. - The `searchParams` handling (studentIdFilter) stays outside try/catch since it doesn't fail

**Conversation detail page (`src/app/(dashboard)/coach/conversations/[conversationId]/page.tsx`):**
- Add import: `import { ErrorAlert } from "@/components/ui/error-alert";`
- Apply the same discriminated result pattern from the submission detail page (28-01):
  - Wrap the conversationData query in try/catch
  - On success with empty result: call `notFound();` (genuinely missing conversation)
  - On catch: return page shell (min-h-screen, container, back link to /coach/conversations) with `<ErrorAlert variant="block" message="Unable to load conversation details. Please try again." />`
- Wrap the turnsData query in a separate try/catch AFTER the conversation is confirmed to exist:
  - On catch: set turns to empty array `[]` and continue rendering (graceful degradation - show conversation metadata even if turns fail to load)
  - Display an inline ErrorAlert above the transcript: `<ErrorAlert message="Unable to load conversation transcript." className="mb-4" />` when turns failed
  - This follows the same graceful degradation pattern as the lesson player (video plays without interactions)
- Use a `transcriptError` boolean flag to track whether turns failed to load, initialized to false, set to true in the turns catch block. Use `transcriptError` consistently throughout (not `turnsError`) to match Phase 27 naming conventions.

**CoachNotesPanel (`src/components/coach/CoachNotesPanel.tsx`) -- styling consistency + CRUD feedback:**
