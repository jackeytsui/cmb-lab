  <name>Task 1: Create role-analytics query library and API route</name>
  <files>src/lib/role-analytics.ts, src/app/api/admin/roles/analytics/route.ts</files>
  <action>
Create `src/lib/role-analytics.ts` with `import "server-only"` at top. Three exported async functions:

1. `getRolesWithActiveStudentCounts()` -- Query all non-deleted roles ordered by sortOrder/name. Separately query userRoles grouped by roleId with expiration filter: `or(isNull(userRoles.expiresAt), gt(userRoles.expiresAt, now))`. Join counts via Map. Return array of `{ id, name, description, color, allCourses, activeStudentCount }`. This is separate from the existing `getRoles()` in roles.ts to avoid breaking the roles list page behavior (which counts ALL assignments). See Research Pitfall 1.

2. `getExpiringAssignments(withinDays: number)` -- Query userRoles JOIN roles JOIN users WHERE: `isNull(roles.deletedAt)`, `isNotNull(userRoles.expiresAt)`, `gt(userRoles.expiresAt, now)`, `lt(userRoles.expiresAt, cutoff)`. Select userId, userName (users.name), userEmail (users.email), roleName (roles.name), roleColor (roles.color), expiresAt (userRoles.expiresAt). Order by expiresAt ASC. Both bounds are critical to avoid showing already-expired roles (Research Pitfall 5).

3. `getMultiRoleStudents()` -- Two-step approach per Research Pattern 3:
   Step 1: SELECT userId, COUNT(*) FROM userRoles JOIN roles WHERE non-deleted AND active GROUP BY userId HAVING COUNT > 1.
   Step 2: If any found, batch-fetch all assignments for those userIds using inArray(). Join users table for name/email. Group results by userId into Map, return array of `{ userId, name, email, roleCount, roles: [{ name, color, expiresAt }] }`. This avoids N+1 (Research Pitfall 4).

Create `src/app/api/admin/roles/analytics/route.ts` with GET handler:
- Auth: `auth()` for userId check (401), `hasMinimumRole("coach")` for access (403). Use "coach" not "admin" since coaches should see role analytics per phase goal.
- Execute `Promise.all([getRolesWithActiveStudentCounts(), getExpiringAssignments(7), getExpiringAssignments(30), getMultiRoleStudents()])`.
- Return JSON: `{ roles, expiring7d, expiring30d, multiRoleStudents }`.
- Wrap in try/catch, log error, return 500 on failure. Follow exact pattern from `src/app/api/admin/analytics/overview/route.ts`.
