       ```
       const celebration = useCelebration({ score: 100 });
       const [nextLessonData, setNextLessonData] = useState<{ id: string; title: string } | null>(null);
       const celebratedRef = useRef(false);
       ```

    5. Create a helper function that fetches next lesson data FIRST, THEN shows the celebration. This prevents a race condition where the celebration overlay renders before the next lesson data arrives, causing SmartCTAs to show stale/missing CTA buttons.

       ```
       const triggerCelebrationIfComplete = useCallback(async (completed: boolean) => {
         if (!completed || celebratedRef.current) return;
         celebratedRef.current = true;

         // Fetch next lesson BEFORE showing celebration to avoid race condition
         // where SmartCTAs render without next lesson data
         if (lessonId) {
           try {
             const res = await fetch(`/api/lessons/${lessonId}/next`);
             const data = await res.json();
             setNextLessonData(data.nextLesson ?? null);
           } catch {
             setNextLessonData(null);
           }
         }

         // Show celebration AFTER next lesson data is available
         celebration.show();
       }, [lessonId, celebration]);
       ```

       IMPORTANT: This is an async function. The `await` ensures next lesson data is set in state BEFORE celebration.show() is called. No setTimeout needed -- the celebration shows immediately after data is ready. The staggered animation in the overlay provides the visual delay.

    6. Use `triggerCelebrationIfComplete` in all completion paths:
       - updateVideoProgress(percent).then(triggerCelebrationIfComplete)
       - markInteractionComplete().then(triggerCelebrationIfComplete) (in handleInteractionDone)
       - markInteractionComplete().then(triggerCelebrationIfComplete) (in useImperativeHandle's completeInteraction)

       Since triggerCelebrationIfComplete is async and .then() passes the resolved value, this works correctly. The Promise<boolean> return from updateVideoProgress/markInteractionComplete feeds directly into the callback parameter.
