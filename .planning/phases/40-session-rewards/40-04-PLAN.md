    3. Compute XP earned client-side (same formula as server-side in xp-service.ts):
       ```
       const computePracticeXP = useCallback((score: number, exerciseCount: number) => {
         const perExercise = Math.round(
           XP_AMOUNTS.practice_exercise_min +
           (score / 100) * (XP_AMOUNTS.practice_exercise_max - XP_AMOUNTS.practice_exercise_min)
         );
         const baseXP = perExercise * exerciseCount;
         const perfectBonus = score >= 100 ? XP_AMOUNTS.practice_perfect : 0;
         return baseXP + perfectBonus;
       }, []);
       ```

    4. Create a useCelebration instance:
       ```
       const celebration = useCelebration({ score: player.totalScore });
       ```

       Add a useEffect that fires the celebration when status transitions to "completed":
       ```
       useEffect(() => {
         if (player.state.status === "completed" && !celebrationFiredRef.current) {
           celebrationFiredRef.current = true;
           celebration.show();
         }
       }, [player.state.status, celebration]);
       ```

    5. Modify the handleRetryAll callback to reset ALL celebration state, including the useCelebration hook's internal hasFired ref:
       ```
       const handleRetryAll = useCallback(async () => {
         setShowCelebration(true);              // Reset so celebration shows again on next completion
         celebrationFiredRef.current = false;   // Allow local ref to fire again
         celebration.reset();                   // CRITICAL: Reset useCelebration's internal hasFired ref
                                                // Without this, the hook's show() will early-return on retry
                                                // because its internal hasFired.current is still true
         player.retryAll();
         // ... rest of existing retry logic (create new attempt)
       ```

    6. Create a dismiss handler for the celebration:
       ```
       const handleCelebrationDismiss = useCallback(() => {
         setShowCelebration(false);
         celebration.dismiss();
       }, [celebration]);
       ```

    7. Modify the "completed" status rendering block. Replace the existing:
       ```
       if (player.state.status === "completed") {
         return (
           <div className="max-w-2xl mx-auto">
             <PracticeResults ... />
           </div>
         );
       }
       ```

       With:
       ```
       if (player.state.status === "completed") {
         return (
           <div className="max-w-2xl mx-auto">
             <AnimatePresence>
               {celebration.isVisible && showCelebration && (
                 <CelebrationOverlay
                   type="practice"
                   score={player.totalScore}
                   xpEarned={computePracticeXP(player.totalScore, exercises.length)}
                   streakCount={0}
                   isFirstAttempt={!player.state.attemptId}
                   correctCount={player.totalCorrect}
                   totalExercises={exercises.length}
                   practiceSetId={practiceSet.id}
                   onDismiss={handleCelebrationDismiss}
                   onRetry={() => {
                     handleCelebrationDismiss();
                     handleRetryAll();
                   }}
                 />
               )}
             </AnimatePresence>
             {(!celebration.isVisible || !showCelebration) && (
               <PracticeResults
                 practiceSet={practiceSet}
                 exercises={exercises}
                 results={player.state.results}
                 responses={player.state.responses}
                 startedAt={player.state.startedAt}
                 completedAt={player.state.completedAt}
                 onRetryAll={handleRetryAll}
                 onRetryExercise={player.retryExercise}
               />
             )}
           </div>
         );
       }
       ```

    8. IMPORTANT: The `handleRetryAll` callback currently has dependencies on `player`, `practiceSet.id`, and `exercises.length`. After adding celebration reset logic, add `celebration` to the dependency array for handleRetryAll. The setShowCelebration and celebrationFiredRef don't need to be in deps (state setter is stable, ref is stable).

    9. Streak count is 0 for simplicity (same rationale as lesson plan -- avoids an extra API call for marginal benefit). StreakBadge renders nothing when streakCount is 0.
