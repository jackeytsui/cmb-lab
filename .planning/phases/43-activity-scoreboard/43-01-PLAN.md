    ```typescript
      totalStudents: number;
    }
    ```

    **getPersonalBests(userId: string): Promise<PersonalBests>**

    Run parallel queries using Promise.all:
    1. Longest streak: query `users.longestStreak` where `users.id = userId`
    2. Highest daily XP: `SELECT MAX(total_xp) FROM daily_activity WHERE user_id = userId`. Also fetch the date of the highest day using a separate subquery or ORDER BY ... LIMIT 1
    3. Best practice score: `MAX(score)` from `practice_attempts` WHERE `user_id = userId AND completed_at IS NOT NULL` -- CRITICAL: filter for completed attempts only per Pitfall 7
    4. Total lessons completed: `COUNT(*)` from `lesson_progress` WHERE `user_id = userId AND completed_at IS NOT NULL`
    5. Total practice sets completed: `COUNT(DISTINCT id)` from `practice_attempts` WHERE `user_id = userId AND completed_at IS NOT NULL`
    6. Total conversations: `SUM(conversation_count)` from `daily_activity` WHERE `user_id = userId`

    Follow the existing `getBadgeStats()` pattern for structure and COALESCE handling.

    **getCohortRankings(userId: string): Promise<CohortRanking[] | null>**

    Returns null if fewer than 5 active students exist (minimum cohort threshold per research).

    Steps:
    1. Count active students: `COUNT(DISTINCT user_id)` from `daily_activity` joined with `users` WHERE `users.role = 'student' AND users.deleted_at IS NULL`
    2. If < 5, return null
    3. Get user's own values: total XP (SUM of daily_activity.total_xp), longest streak (users.longestStreak), average practice score (AVG of completed practice_attempts.score -- only include if user has >= 3 completed attempts per research recommendation)
    4. For each dimension, count students with a LOWER value than the user (only active students with >= 1 day in daily_activity)
    5. Calculate percentile: `percentileRank = (studentsBelow / totalActiveStudents) * 100`
    6. Bucket into tiers using helper function:
       - >= 95 -> "Top 5%"
       - >= 90 -> "Top 10%"
       - >= 75 -> "Top 25%"
       - >= 50 -> "Top 50%"
       - >= 25 -> "Top 75%"
       - >= 10 -> "Top 90%"
       - < 10 -> "Bottom half"

    For the average practice score dimension: only include students who have >= 3 completed practice attempts in the comparison cohort. If the user has < 3 completed attempts, omit this dimension from the results.

    Use `db.execute(sql`...`)` for the cohort counting queries (raw SQL with template literals) as shown in the research examples. Guard against division by zero.

    Import `isNull` from drizzle-orm if not already imported (it is already imported in the file).
  </action>
