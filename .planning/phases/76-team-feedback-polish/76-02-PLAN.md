# Plan 76-02: Coaching Enhancements

<objective>
Add three coaching workflow improvements: Mando↔Canto copy-over translation using GPT-4o-mini, per-entry notes/explanations field, and GHL tracking form embed in 1:1 lesson notes.
</objective>

## Tasks

### Task 1: Copy-Over Translation + Per-Entry Notes

**1a. Copy-Over Translation API:**
- Create `src/app/api/coaching/translate/route.ts`
- POST endpoint accepts `{ text, fromLang: "mandarin" | "cantonese", toLang: "mandarin" | "cantonese" }`
- Uses existing OpenAI client (check `src/lib/` for existing OpenAI setup, or use `fetch` to OpenAI API directly with `OPENAI_API_KEY` env var)
- System prompt: "Translate this {fromLang} Chinese text to {toLang} Chinese. For Cantonese use traditional characters with Cantonese-specific vocabulary (嘅 instead of 的, 喺 instead of 在, 冇 instead of 没有, etc.). For Mandarin use simplified characters with standard Mandarin vocabulary. Return ONLY the translated text, nothing else."
- Model: `gpt-4o-mini` (fast, cheap, sufficient for short phrases)
- Requires `hasMinimumRole("coach")`

**1b. Copy-Over UI:**
- In `CoachingMaterialClient.tsx`, add a "Copy Over" button on each NoteCard
- Button shows an icon (e.g. `ArrowRightLeft` from lucide) with tooltip "Translate to {otherLanguage}"
- On click: call the translate API, then create a new note in the opposite pane with the translated text
- POST to `/api/coaching/sessions/[sessionId]/notes` with `{ text: translatedText, pane: targetPane }`
- Show loading spinner during translation
- Auto-populate romanizationOverride and translationOverride if possible (or leave empty for coach to fill)

**1c. Per-Entry Notes Field:**
- Add `explanation` text column to `coachingNotes` table in `src/db/schema/coaching.ts`
- Generate migration: `npx drizzle-kit generate`
- Update PATCH endpoint at `/api/coaching/notes/[noteId]/route.ts` to accept `explanation` field
- In `NoteCard` component in `CoachingMaterialClient.tsx`:
  - Add a small "Add Notes" button (pencil + text icon) below each note entry
  - Clicking it expands an inline textarea for explanation/notes
  - Auto-saves on blur or debounced input (PATCH to API)
  - Display saved explanation in a subtle muted text block below the note
  - Students see the explanation (read-only), coaches can edit

### Task 2: GHL Tracking Form Embed

**What:**
- Embed the GHL tracking form in the 1:1 coaching session page
- Form URL: `https://api.leadconnectorhq.com/widget/form/Vy75BI6BJuB4ibQlYA8P`
- Place it in a collapsible section below the Coaching Goals area in `CoachingMaterialClient.tsx`
- Use an `<iframe>` embed with the `?notrack=true` parameter
- Add a toggle/accordion: "Session Tracking Form" with a chevron to expand/collapse
- Default collapsed to keep the page clean
- Only show for coaches (check role), not students

**Implementation:**
- In `CoachingMaterialClient.tsx`, after the Coaching Goals section (~line 1605):
  - Add a collapsible `<details>` or custom accordion section
  - Title: "Session Tracking Form"
  - Content: `<iframe src="https://api.leadconnectorhq.com/widget/form/Vy75BI6BJuB4ibQlYA8P?notrack=true" width="100%" height="600" frameBorder="0" />`
  - Pass `isCoach` prop to conditionally render
