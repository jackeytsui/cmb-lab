# Plan 76-03: Tone-Colored Characters (Pleco Style)

<objective>
Implement Pleco-style tone-colored Chinese characters across the platform. Characters of the same tone share the same color. Primary implementation in WordSpan.tsx (Reader component) which cascades to coaching notes, accelerator reader, and any page using ReaderTextArea. Secondary implementation in flashcards and vocabulary list.
</objective>

## Tasks

### Task 1: Tone Color Utility + WordSpan Integration

**1a. Create Tone Color Utility:**
- Create `src/lib/tone-colors.ts` with:
  ```typescript
  // Pleco standard tone colors (works in light and dark mode)
  export const MANDARIN_TONE_COLORS: Record<number, string> = {
    1: "text-red-500",           // First tone (ā) — red
    2: "text-green-500",         // Second tone (á) — green
    3: "text-blue-500",          // Third tone (ǎ) — blue
    4: "text-purple-500",        // Fourth tone (à) — purple
    0: "text-muted-foreground",  // Neutral tone — grey
  };

  export const CANTONESE_TONE_COLORS: Record<number, string> = {
    1: "text-red-500",           // High level
    2: "text-green-500",         // High rising
    3: "text-blue-500",          // Mid level
    4: "text-orange-500",        // Low falling
    5: "text-purple-500",        // Low rising
    6: "text-teal-500",          // Low level
    0: "text-muted-foreground",  // Unknown
  };

  // Extract tone number from pinyin syllable ("ni3" → 3, "nǐ" → 3)
  export function extractToneFromPinyin(syllable: string): number

  // Extract tone number from jyutping syllable ("nei5" → 5)
  export function extractToneFromJyutping(syllable: string): number

  // Get tone color class for a character
  export function getToneColorClass(tone: number, lang: "mandarin" | "cantonese"): string
  ```

- Tone extraction from diacritics: map ā→1, á→2, ǎ→3, à→4, and same for e/i/o/u/ü
- Tone extraction from numbers: parse trailing digit from syllable

**1b. WordSpan Integration:**
- In `src/components/reader/WordSpan.tsx`, the core character rendering (~line 175):
  ```typescript
  // Current: <span className="text-center">{char}</span>
  // New: <span className={cn("text-center", toneColorEnabled && toneColorClass)}>{char}</span>
  ```
- `getPinyinArray(text)` already returns per-character pinyin — extract tone from each
- Accept a `toneColorsEnabled` prop (default false, controlled by parent)
- When enabled, each character gets a color class based on its tone
- Use Mandarin tones when `showPinyin` is active, Cantonese tones when `showJyutping` is active
- If both are shown, prefer Mandarin tones (user can toggle)

**1c. Reader Toolbar Toggle:**
- In `src/components/reader/ReaderToolbar.tsx` (or wherever annotation mode toggles live), add a "Tone Colors" toggle button
- Icon: `Palette` from lucide
- Stores preference in localStorage (`cmb.reader.toneColors`)
- Pass the boolean down through ReaderTextArea → WordSpan

### Task 2: Flashcards + Vocabulary Tone Coloring

**2a. Flashcards:**
- In `FlashcardsClient.tsx`, the card front displays `{displayChinese}` as plain text
- Split the Chinese text into characters, look up pinyin tone for each character
- Use the `pinyin` npm package (already installed via tone-sandhi.ts dependency) to get tone numbers
- Apply tone color classes to each character span
- Both grid view and study mode card fronts

**2b. Vocabulary List:**
- In `VocabularyClient.tsx`, apply tone coloring to the Chinese characters displayed in the vocabulary list
- Each vocab item already has pinyin/jyutping fields — extract tones from those

**Key architectural decision:** WordSpan changes automatically cascade to:
- AI Passage Reader (uses ReaderClient → ReaderTextArea → WordSpan)
- Coaching notes (uses ReaderTextArea → WordSpan)
- Accelerator curated passages (uses ReaderClient → ReaderTextArea → WordSpan)
- YouTube Listening Lab transcripts (uses WordSpan)

So the primary change in WordSpan.tsx covers ~6 features at once.
