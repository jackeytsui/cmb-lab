# Plan 76-01: Quick Fixes & Polish

<objective>
Fix 5 quick issues from team feedback: differentiate sidebar icons for coaching types, fix TTS language bug on jyutping edit, show export button to students, add fathom link to CSV exports, and fix assigned coach display in admin.
</objective>

## Tasks

### Task 1: Sidebar Icon Differentiation + TTS Bug Fix + Export Access

**What:** Three small UI fixes.

**1a. Sidebar Icons:**
- In `src/components/layout/AppSidebar.tsx`, change the Inner Circle Coaching icon from `FileText` to `Users` (or `UsersRound`) so it's visually distinct from 1:1 Coaching when the sidebar is collapsed
- 1:1 keeps `FileText`, Inner Circle gets `UsersRound`

**1b. TTS Language Bug:**
- In `src/app/(dashboard)/dashboard/coaching/CoachingMaterialClient.tsx`, find the `handleSpeakSentence` callback (~line 270)
- The bug: `language` in the closure comes from the NoteCard's initialization and may not match the note's actual `pane` when editing
- Fix: pass `pane` directly from the note data when calling speak, not from the parent language prop
- The speak call should use `pane === "cantonese" ? "zh-HK" : "zh-CN"` derived from the individual note's pane field

**1c. Student Export Access:**
- In `/api/coaching/export/route.ts`, the export endpoint requires `hasMinimumRole("coach")` — change to `hasMinimumRole("student")`
- Add a check: students can only export their OWN sessions (filter by their email)
- Coaches/admins can export any student's sessions (existing behavior)

### Task 2: Fathom Link in CSV Export + Assigned Coach Fix

- Include the session's fathomLink URL in each row of the exported CSV/Excel
- In the coaching admin/coach UI, ensure there's a field to set the fathom link per session

**2b. Assigned Coach Display:**
