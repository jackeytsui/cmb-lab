  <action>
    1. Install dev dependencies:
       ```bash
       npm install -D @playwright/test @clerk/testing
       npx playwright install --with-deps chromium webkit
       ```

    2. Create `playwright.config.ts` at project root following the research Pattern 2. Configure:
       - testDir: `./e2e/tests`
       - fullyParallel: true
       - forbidOnly: `!!process.env.CI`
       - retries: `process.env.CI ? 2 : 0`
       - workers: `process.env.CI ? 1 : undefined`
       - reporter: CI uses `[['html', { open: 'never' }], ['github']]`, local uses `[['html', { open: 'on-failure' }]]`
       - baseURL: `http://localhost:3000`
       - trace: `on-first-retry`
       - screenshot: `only-on-failure`
       - webServer: command `npm run dev`, url `http://localhost:3000`, reuseExistingServer `!process.env.CI`, timeout 120000
       - Projects:
         - `setup`: testMatch `/global\.setup\.ts/`, testDir `./e2e`
         - `chromium`: Desktop Chrome device, storageState `e2e/playwright/.clerk/student.json`, depends on setup
         - `webkit`: Desktop Safari device, storageState `e2e/playwright/.clerk/student.json`, depends on setup
         - `mobile`: iPhone 14 device, storageState `e2e/playwright/.clerk/student.json`, depends on setup
         - `coach-chromium`: Desktop Chrome device, storageState `e2e/playwright/.clerk/coach.json`, testMatch `/coach\.spec\.ts/`, depends on setup

    3. Add npm scripts to package.json:
       ```json
       "test:e2e": "playwright test",
       "test:e2e:ui": "playwright test --ui",
       "test:e2e:debug": "playwright test --debug",
       "test:e2e:headed": "playwright test --headed",
       "test:e2e:chromium": "playwright test --project=chromium"
       ```

    4. Add to `.gitignore`:
       ```

       /e2e/playwright/.clerk/
       ```

  <action>
    1. Create `e2e/global.setup.ts` following research Pattern 1:
       - Import `clerk`, `clerkSetup` from `@clerk/testing/playwright`
       - Import `test as setup` from `@playwright/test`
       - Configure serial mode: `setup.describe.configure({ mode: 'serial' })`
       - First test: `global setup` -- calls `await clerkSetup()`
       - Second test: `authenticate student` -- navigates to `/`, calls `clerk.signIn()` with strategy `password`, identifier from `process.env.E2E_CLERK_STUDENT_EMAIL`, password from `process.env.E2E_CLERK_STUDENT_PASSWORD`. Then navigates to `/dashboard`, waits for page load, saves storageState to `e2e/playwright/.clerk/student.json`
       - Third test: `authenticate coach` -- same pattern with `E2E_CLERK_COACH_EMAIL` / `E2E_CLERK_COACH_PASSWORD`, navigates to `/coach`, saves to `e2e/playwright/.clerk/coach.json`
       - Ensure the `e2e/playwright/.clerk/` directory is created (use `fs.mkdirSync` with recursive: true)

    2. Create `e2e/fixtures/auth.ts`:
       - Export custom `test` and `expect` extending Playwright's base test
       - Add custom fixture `studentPage` that uses student storageState
       - Add custom fixture `coachPage` that uses coach storageState
       - This enables tests to get pre-authenticated pages via fixture injection

    3. Create `e2e/helpers/composition.ts`:
       - Export `typeChineseText(page, selector, finalText, intermediateSteps[])` function
       - Follows research Pattern 4 exactly: dispatches compositionstart, compositionupdate (for each step), compositionend, input events with isComposing flag, and sets value via native setter + change event
       - Include JSDoc with usage example

    4. Create `e2e/helpers/mux-mock.ts`:
       - Export `mockMuxPlayer(page)` function that:
         - Blocks `**/stream.mux.com/**` requests (route.abort)
         - Blocks `**/image.mux.com/**` with minimal placeholder
         - Returns cleanup function
       - Export `simulateVideoProgress(page, progressPercent)` function that calls the progress API directly via `page.request.post()`
       - Do NOT try to dispatch events on mux-player shadow DOM (uncertain, per research open question 1). Instead mock at the API level.

    5. Create `e2e/helpers/webhook-mock.ts`:
       - Export `mockGradingResponse(page, response)` that intercepts `**/api/grade` via page.route() and fulfills with provided response object
       - Export `mockAudioGradingResponse(page, response)` that intercepts `**/api/grade-audio`
       - Export `mockCoachFeedbackNotification(page)` that intercepts `**/api/notify/coach-feedback`

    6. Create Page Object Models:

       `e2e/pages/lesson.page.ts` - LessonPage class:
       - Locators: videoPlayer (`mux-player`), interactionArea (`[data-testid="interaction-area"]`), submitButton (`[data-testid="submit-answer"]`), chineseInput (`[data-testid="chinese-input"]`), feedbackArea (`[data-testid="feedback"]`), completionBadge (`[data-testid="lesson-complete"]`)
       - Methods: goto(lessonId), submitAnswer(text), expectCompleted(), expectFeedbackVisible()

       `e2e/pages/dashboard.page.ts` - DashboardPage class:
       - Locators: courseGrid, courseCards, progressBars
       - Methods: goto(), getCourseCount(), clickCourse(index)

       `e2e/pages/coach.page.ts` - CoachPage class:
       - Locators: submissionQueue, submissionCards, feedbackInput, sendFeedbackButton, noteInput
       - Methods: goto(), getSubmissionCount(), openSubmission(index), sendFeedback(text), addNote(text)

    IMPORTANT: Use `data-testid` attributes consistently. The test specs (in later plans) will rely on these selectors. If the actual app components do not have these data-testid attributes yet, that is fine -- the later plans or the tests themselves will add them as needed.
  </action>
