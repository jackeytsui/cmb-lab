2. **`buildWhereConditions(filters: PracticeResultsFilters)`** — internal helper that builds an array of Drizzle `SQL` conditions. Always includes `isNotNull(practiceAttempts.completedAt)`. Conditionally adds:
   - `ilike(users.name, ...)` OR `ilike(users.email, ...)` when `studentName` is set (use `or()` from drizzle-orm to search BOTH name and email, matching the research recommendation)
   - `eq(practiceAttempts.practiceSetId, ...)` when `practiceSetId` is set
   - `gte(practiceAttempts.completedAt, ...)` when `dateFrom` is set
   - `lte(practiceAttempts.completedAt, ...)` when `dateTo` is set
   - `gte(practiceAttempts.score, ...)` when `scoreMin` is set
   - `lte(practiceAttempts.score, ...)` when `scoreMax` is set

3. **`getPracticeResults(filters: PracticeResultsFilters): Promise<PracticeResultsResponse>`** — main exported function that:

   a) Builds shared WHERE conditions via `buildWhereConditions(filters)`

   b) Runs three queries in parallel with `Promise.all`:
      - **Attempts query:** SELECT attemptId, practiceSetId, score, totalExercises, correctCount, startedAt, completedAt, results, studentName (from users), studentEmail (from users), setTitle (from practiceSets) FROM practiceAttempts INNER JOIN users ON userId INNER JOIN practiceSets ON practiceSetId WHERE conditions ORDER BY completedAt DESC LIMIT 200
      - **Per-set aggregates query:** SELECT setId, setTitle, AVG(score) as avgScore, COUNT(*) as attemptCount FROM practiceAttempts JOIN practiceSets JOIN users WHERE conditions GROUP BY setId, setTitle ORDER BY AVG(score) ASC
      - **Overall stats query:** SELECT COUNT(*) as totalAttempts, COUNT(DISTINCT userId) as totalStudents, AVG(score) as overallAvgScore FROM practiceAttempts JOIN users WHERE conditions

   c) After attempts query returns, extract unique practiceSetIds, batch-load exercise definitions from `practiceExercises` WHERE practiceSetId IN (...). Build a `Map<string, { type: string; practiceSetTitle: string }>` for exercise lookup. Follow the exact pattern from `coach/pronunciation/page.tsx` lines 139-153.

   d) Transform raw attempts into `AttemptDetail[]`:
      - Compute `timeTakenSeconds`: if both `completedAt` and `startedAt` exist, calculate `(completedAt.getTime() - startedAt.getTime()) / 1000`. Cap at 1800 (30 minutes) — show null if > 1800 per research pitfall #1.
      - Parse `results` JSONB (guard against null per research pitfall #2): for each entry in results, look up exercise type from the exercise map, produce `{ exerciseId, exerciseType, isCorrect, score }`.

   e) Call `computeHardestExercises()` with the raw attempts and exercise map.

   f) Also query all published practice sets (SELECT id, title FROM practiceSets WHERE status='published' AND deletedAt IS NULL ORDER BY title) for the filter dropdown metadata.

   g) Compute `overallCompletionRate` from the per-set data (total completed / total started, but since we filter for completedAt IS NOT NULL, this is effectively 100% in the filtered set — instead, run a separate count of ALL attempts including incomplete ones matching the non-completion filters, then compute rate). SIMPLER APPROACH: just return the overall avg score and counts. The completion rate is computed as: (attempts with completedAt not null) / (ALL attempts matching other filters). Run one additional count query for total attempts (including incomplete) with the same filters minus the `isNotNull(completedAt)` condition.

4. **`computeHardestExercises(attempts, exerciseMap)`** — exported pure function. Follow the exact algorithm from the research code example:
   - Iterate all attempts, parse `results` JSONB, accumulate per-exercise stats (correct count, total count, total score)
   - Filter for exercises with >= 3 attempts (per research pitfall #5)
   - Sort by avgScore ascending (lowest = hardest)
   - Return top 10
