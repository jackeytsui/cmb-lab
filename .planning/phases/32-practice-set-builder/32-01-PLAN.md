  <action>
    1. Install dependencies:
       ```bash
       npm install @dnd-kit/react @dnd-kit/helpers
       ```

    2. Create `src/hooks/useBuilderState.ts` with the following:

    **BuilderExercise interface:**
    - id: string (server UUID or 'temp-' + nanoid for new exercises)
    - type: ExerciseType string
    - language: 'cantonese' | 'mandarin' | 'both'
    - definition: ExerciseDefinition (from @/types/exercises)
    - sortOrder: number
    - practiceSetId: string
    - isNew: boolean (true for palette-added exercises, false for server-loaded)
    - isConfigured: boolean (false when just dragged from palette with skeleton definition, true after editing)

    **BuilderState interface:**
    - exercises: BuilderExercise[]
    - title: string
    - description: string
    - status: 'draft' | 'published' | 'archived'

    **HistoryState interface:**
    - past: BuilderState[] (capped at 50)
    - present: BuilderState
    - future: BuilderState[]

    **BuilderAction discriminated union:**
    - ADD_EXERCISE: { type, exerciseType: string, language: 'both', atIndex?: number } — creates a new BuilderExercise with `temp-${nanoid()}` id, skeleton definition based on exerciseType, isNew: true, isConfigured: false
    - REMOVE_EXERCISE: { exerciseId: string }
    - REORDER_EXERCISES: { fromIndex: number, toIndex: number } — uses array splice to move
    - UPDATE_EXERCISE: { exerciseId: string, data: Partial<BuilderExercise> } — for inline editing, sets isConfigured: true
    - UPDATE_SET_META: { title?: string, description?: string }
    - SET_STATUS: { status: 'draft' | 'published' | 'archived' }
    - UNDO / REDO
    - MARK_SAVED: { idMap: Map<string, string> } — replaces temp IDs with server IDs, sets isNew: false on all

    **builderReducer function:**
    - UNDO: pop from past, push current to future
    - REDO: pop from future, push current to past
    - MARK_SAVED: does NOT push to history (it's a sync action, not an edit)
    - All other actions: push structuredClone(state.present) to past, compute new present, clear future. Cap past at 50 entries (slice from end).

    **Skeleton definitions per exercise type** (minimal valid structure so Zod won't crash on preview):
    - multiple_choice: { type: 'multiple_choice', question: '', options: [{ id: nanoid(), text: '' }, { id: nanoid(), text: '' }], correctOptionId: '' }
    - fill_in_blank: { type: 'fill_in_blank', sentence: '', blanks: [{ id: nanoid(), correctAnswer: '' }] }
    - matching: { type: 'matching', pairs: [{ id: nanoid(), left: '', right: '' }, { id: nanoid(), left: '', right: '' }] }
    - ordering: { type: 'ordering', items: [{ id: nanoid(), text: '', correctPosition: 0 }, { id: nanoid(), text: '', correctPosition: 1 }] }
    - audio_recording: { type: 'audio_recording', targetPhrase: '' }
    - free_text: { type: 'free_text', prompt: '' }

    **useBuilderState hook:**
    - Takes initialState: BuilderState
    - Returns: { state: BuilderState, dispatch, canUndo, canRedo, isDirty }
    - isDirty: compare JSON.stringify of present vs the initial state passed at mount (store initial in a ref)
    - Keyboard shortcuts in useEffect:
      - Ctrl+Z / Cmd+Z (no Shift) = UNDO
      - Ctrl+Shift+Z / Cmd+Shift+Z = REDO
      - Ctrl+Y / Cmd+Y = REDO (Windows convention)
      - e.preventDefault() on all

    Export BuilderExercise type and BuilderAction type along with the hook.
  </action>

  </verify>

    Add an optional `onLocalSave` callback to ExerciseForm and all 6 sub-forms. When `onLocalSave` is provided, sub-forms skip the API fetch and instead call `onLocalSave` with the constructed definition + language. This is a backward-compatible change — existing callers don't pass onLocalSave and behavior is unchanged.

    **ExerciseForm changes:**
    - Add to ExerciseFormProps: `onLocalSave?: (data: { type: string; language: string; definition: ExerciseDefinition }) => void`
    - Pass `onLocalSave` through to each sub-form component

    **Each sub-form (MultipleChoiceForm, FillInBlankForm, MatchingPairsForm, OrderingForm, AudioRecordingForm, FreeTextForm) changes:**
    - Add to props interface: `onLocalSave?: (data: { type: string; language: string; definition: ExerciseDefinition }) => void`
    - In the `onSubmit` handler, add a check at the TOP (before the try/catch with fetch):
      ```typescript
      if (onLocalSave) {
        const definition = { type: '...', ...constructedDefinition };
        onLocalSave({ type: definition.type, language, definition });
        return;
      }
      ```
    - The rest of the onSubmit (fetch, onSave call) remains unchanged for when onLocalSave is NOT provided.
    - Import ExerciseDefinition type if not already imported.

    **IMPORTANT:** The existing behavior when onLocalSave is NOT provided must be IDENTICAL to the current code. Only add the early-return path. Do not restructure the existing onSubmit logic.

    **Pattern for each sub-form onSubmit:**
    ```typescript
    const onSubmit = async (data: FormData) => {
      // Construct the typed definition from form data (same as existing code)
      const definition = { type: 'exercise_type', ...fields };

      // NEW: local save path for builder
      if (onLocalSave) {
        onLocalSave({ type: definition.type, language, definition });
        return;
      }

      // EXISTING: API save path (unchanged)
      setIsSaving(true);
      setApiError(null);
      try {
        // ... existing fetch logic ...
      }
    };
    ```
  </action>

    - ExerciseForm passes onLocalSave to all 6 sub-forms
