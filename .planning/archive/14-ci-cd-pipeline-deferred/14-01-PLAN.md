  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/ci.yml` with the following configuration:

- **Trigger:** `pull_request` targeting `main` branch
- **Concurrency:** Cancel in-progress runs for the same PR (`ci-${{ github.ref }}` group, `cancel-in-progress: true`)
- **Job name:** "Lint, Typecheck & Build"
- **Runner:** `ubuntu-latest`
- **Steps:**
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: '20'` and `cache: 'npm'`
  3. `npm ci` (clean install)
  4. `npm run lint` (ESLint)
  5. `npx tsc --noEmit` (TypeScript typecheck without emitting)
  6. `npm run build` (Next.js production build)

- **Environment variables for build step** (all from GitHub Secrets):
  - `DATABASE_URL: ${{ secrets.DATABASE_URL_PREVIEW }}`
  - `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY: ${{ secrets.NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY_PREVIEW }}`
  - `CLERK_SECRET_KEY: ${{ secrets.CLERK_SECRET_KEY_PREVIEW }}`
  - `CLERK_WEBHOOK_SECRET: ${{ secrets.CLERK_WEBHOOK_SECRET_PREVIEW }}`
  - `MUX_TOKEN_ID: ${{ secrets.MUX_TOKEN_ID }}`
  - `MUX_TOKEN_SECRET: ${{ secrets.MUX_TOKEN_SECRET }}`
  - `MUX_WEBHOOK_SECRET: ${{ secrets.MUX_WEBHOOK_SECRET }}`
  - `ENROLLMENT_WEBHOOK_SECRET: ${{ secrets.ENROLLMENT_WEBHOOK_SECRET_PREVIEW }}`

Use the exact YAML from the research document's "Complete CI Workflow" section. Ensure the `env` block is ONLY on the build step (lint and typecheck don't need env vars).

DO NOT add `OPENAI_API_KEY` or `N8N_*` URLs to the build env — these are runtime-only and not needed for `next build`.
  </action>
