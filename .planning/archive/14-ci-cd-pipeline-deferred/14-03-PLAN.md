<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Enable GitHub branch protection on main</name>
  <action>
Go to GitHub -> Repository -> Settings -> Branches -> Add branch protection rule:

1. **Branch name pattern:** `main`
2. **Enable:** Require status checks to pass before merging
3. **Required status checks:** Search and add:
   - `Lint, Typecheck & Build` (from ci.yml)
   - `E2E Tests` (from e2e.yml)
4. **Enable:** Require branches to be up to date before merging
5. Save the rule

NOTE: The status checks will only appear in the search after the workflows have run at least once. If they don't appear yet, create a test PR first to trigger the workflows, then come back and add the required checks.
  </action>
  <resume-signal>Type "branch protection configured" when the rule is saved on GitHub.</resume-signal>
</task>
