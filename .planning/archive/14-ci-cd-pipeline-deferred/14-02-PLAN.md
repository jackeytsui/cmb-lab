Create GitHub Actions workflows for Neon database branch lifecycle and schema validation on PRs.

Purpose: CICD-06 requires that database migrations are validated in CI before deployment. This plan creates three workflows: (1) create an isolated Neon DB branch per PR, (2) clean up on PR close, (3) run migrations and post schema diff on PRs that touch `src/db/**`.

Output: Three workflow YAML files for database branch management and schema safety.

<task type="auto">
  <name>Task 1: Create Neon DB branch create and cleanup workflows</name>
  <files>
.github/workflows/db-branch-create.yml
.github/workflows/db-branch-cleanup.yml
  </files>
  <action>
**Create `.github/workflows/db-branch-create.yml`:**

- **Trigger:** `pull_request` types `[opened, reopened]` targeting `main`
- **Job:** `create-branch` on `ubuntu-latest`
- **Steps:**
  1. `actions/checkout@v4`
  2. `neondatabase/create-branch-action@v6` with:
     - `project_id: ${{ vars.NEON_PROJECT_ID }}`
     - `branch_name: preview/pr-${{ github.event.pull_request.number }}`
     - `api_key: ${{ secrets.NEON_API_KEY }}`
     - id: `create-branch`
  3. `actions/setup-node@v4` with `node-version: '20'`, `cache: 'npm'`
  4. `npm ci`
  5. Run `npx drizzle-kit migrate` with `DATABASE_URL: ${{ steps.create-branch.outputs.db_url }}`

This ensures the preview branch has the latest schema applied immediately.

**Create `.github/workflows/db-branch-cleanup.yml`:**

- **Trigger:** `pull_request` types `[closed]`
- **Job:** `delete-branch` on `ubuntu-latest`
- **Steps:**
  1. `neondatabase/delete-branch-action@v3` with:
     - `project_id: ${{ vars.NEON_PROJECT_ID }}`
     - `branch: preview/pr-${{ github.event.pull_request.number }}`
     - `api_key: ${{ secrets.NEON_API_KEY }}`

The branch naming convention `preview/pr-{number}` must match between create and cleanup workflows exactly.

DO NOT add `continue-on-error` to the delete action — if cleanup fails, we want it visible so branches don't accumulate silently.
  </action>
  <verify>
Confirm both files exist and contain correct action versions:
- `grep 'create-branch-action@v6' .github/workflows/db-branch-create.yml`
- `grep 'delete-branch-action@v3' .github/workflows/db-branch-cleanup.yml`
- Both reference `preview/pr-${{ github.event.pull_request.number }}`
- Create workflow includes `drizzle-kit migrate` step
  </verify>
  <done>DB branch create workflow creates a Neon branch and runs migrations on PR open. DB branch cleanup workflow deletes the Neon branch on PR close. Both use matching branch naming convention.</done>
</task>

<task type="auto">
  <name>Task 2: Create database schema diff workflow</name>
  <files>.github/workflows/db-schema-check.yml</files>
  <action>
Create `.github/workflows/db-schema-check.yml` with:

- **Trigger:** `pull_request` targeting `main` with `paths: ['src/db/**']`
- **Job:** `schema-diff` on `ubuntu-latest`
- **Permissions:** `pull-requests: write`, `contents: read`
- **Steps:**
  1. `neondatabase/create-branch-action@v6` (id: `create-branch`) with:
     - `project_id: ${{ vars.NEON_PROJECT_ID }}`
     - `branch_name: preview/pr-${{ github.event.pull_request.number }}`
     - `api_key: ${{ secrets.NEON_API_KEY }}`
     (This is idempotent — if branch already exists from db-branch-create.yml, it returns the existing one)
  2. `actions/checkout@v4`
  3. `actions/setup-node@v4` with `node-version: '20'`, `cache: 'npm'`
  4. Install and run migrations:
     ```
     npm ci
     npx drizzle-kit migrate
     ```
     with `DATABASE_URL: ${{ steps.create-branch.outputs.db_url }}`
  5. `neondatabase/schema-diff-action@v1` with:
     - `project_id: ${{ vars.NEON_PROJECT_ID }}`
     - `compare_branch: preview/pr-${{ github.event.pull_request.number }}`
     - `api_key: ${{ secrets.NEON_API_KEY }}`

The `paths` filter ensures this workflow only runs when database files change, keeping CI fast for non-DB PRs.

The permissions block is REQUIRED for the schema-diff-action to post PR comments.
  </action>
  <verify>
Confirm:
- `grep 'src/db' .github/workflows/db-schema-check.yml` shows paths filter
- `grep 'schema-diff-action@v1' .github/workflows/db-schema-check.yml`
- `grep 'pull-requests: write' .github/workflows/db-schema-check.yml` shows permissions
- `grep 'drizzle-kit migrate' .github/workflows/db-schema-check.yml` shows migration step
  </verify>
  <done>Schema diff workflow triggers on PRs that modify `src/db/**`, creates/reuses a Neon branch, runs migrations against it, and posts a schema diff comment on the PR.</done>
</task>
